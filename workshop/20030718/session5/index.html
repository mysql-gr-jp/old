• MySQLの運用
    　最低限の運用知識
    1. オンラインマニュアル
      • まずは一通り目を通しましょう。ローカルにダウンロードしておくと便利
    2. 管理ツール
      • コマンドライン
        • 長所
          ・何でもできる
          ・telnet(ssh) が使えれば端末を選ばない
        • 短所
          すべてキー入力（当然）
        • ツール
          ・mysqladmin サーバの終了、設定、状態取得
          ・mysql SQL コマンド発行
          ・mysqldump データベースバックアップ
      • ＧＵＩ
        • 長所
          マウスクリックのみでもある程度のことができる
        • 短所
          ・インストールが別に必要
          ・ツールにより機能制限あり
        • ツール紹介
          • mysqlcc
            ・MySQL AB 純正
            ・ほとんどの操作が可能
          • phpmyadmin
            ・クライアントの制限がゆるい
            ・各種漢字コードに対応
            ・サーバ管理はバックアップ程度で基本的にはテーブル管理
            ・Web server + php が必要
            　（サーバ側が許していれば、mysql サーバと別も可、ただし遅い）
          • Eclipse + DbEdit
            ・Eclipse を常用している人には便利
            ・これもテーブル管理
          • その他
            http://www.softagency.co.jp/MySQL/portal/software/index.html
    3. バックアップ
      • 日常のバックアップ
        • mysqldump
          　 CREATE TABLE, INSERT クエリテキストが標準出力へ吐き出される
          mysqldump [Database] [Table] >backup.sql
          mysqldump -A>backup_all.sql
        • サーバをとめてコピー
          mysqladmin shutdown;tar zcvf db.tgz /usr/local/mysql/data
          mysqladmin shutdown;cp -rp /usr/local/mysql/data ~/backups
        • select into outfile
          select * into outfile 'table1.bakup' from table1;
        • log-update
          safe_mysqld --log-update (4.0以降は mysqld_safe --log-update) で自動的に変更クエリのみがログに記録される
      • マシン移行
        ・基本的にはバックアップ、リストア(mysql < [dump ファイル])
        ・サーバが動いていない状態で、バージョン差が小さいならコピーでも可
        ・バージョン差が大きい場合は若干の修正が必要になることも
        ・詳しくは http://www.mysql.gr.jp/jpdoc/4.0/manual.ja_Installing.html#Upgrade

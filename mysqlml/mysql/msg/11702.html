佐々木です。

MySQL への接続直後に
　mysql_query( "SET NAMES ujis" );
を実行してみてください。
Access から正常に見れるのなら、文字コードの自動変換が正しく行われている
証拠ですから、PHP 内部までは EUC-JP (ujis) でデータが受け取れるはずです。

尚、phpMyAdmin の方は、SET NAMES ではなく、
　SET CHARACTER SET utf8;
が実行されているようです。（phpMyAdmin の内部処理）
正常に表示されるのは MySQL が PHP へデータを送信するとき、
ujis -> utf8 への文字コード変換を行うからで、
ここらへんは荒木さんの説明通りですね。

この手の問題については、日本MySQLユーザ会が F.A.Q に
纏めておられるようなので、一読されることをお勧めします。
http://www.mysql.gr.jp/frame/modules/bwiki/?FAQ#content_1_40

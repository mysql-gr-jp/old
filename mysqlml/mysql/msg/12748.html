| 当初、私も
| 1. mysql 関連を全て削除
| 2. MySQL AB で配付されているパッケージをインストール
| までは問題なかったんですが、確か RHEL 付属の PHP にて
| MySQL を認識出来ず、PHP をソースから入れなおしても認識
| されないという流れでした。
| 
| 2 番でうまくいかなかったというのは PHP が認識しなかった
| が「再起動」する事によって認識したという事でしょうか？

そうなんです。
エラーの詳細は忘れましたが、
削除、そして
find / -name "mysql*"
あたりで検索して、
mysqlと名前の付く物は殆ど削除しました。

PHPのPear内やPerlのライブラリっぽいものは削除しませんでした。

そして、その状態で、インストールすると、
途中でエラーがでて止まってしまいました。

「初期化できない」か、「開始できない」のような感じのエラーでした。

同じ事を繰り返し行った後、（mysqlという名前の付く物すべて削除）

サーバ自体の再起動
reboot（若しくは shutdown -r now）
して、その後、新しくインストールすると、
問題なく進めました。

その際に使ったパッケージは、
http://dev.mysql.com/downloads/mysql/4.1.html
の
RHEL3 (x86) - Server
RHEL3 (x86) - Client
RHEL3 (x86) - Headers and libraries
RHEL3 (x86) - Shared libraries
です。


以上、参考になれば幸いです。

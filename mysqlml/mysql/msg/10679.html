バイナリ版はまだ作成されていませんが、動作としては
ソール版をujis/sjisでコンパイルした場合と同じに
なるはずです。

binaryキャラクタセットを使用した時のようにマルチバイトの
データベース名・テーブル名・コラム名が

mysql> create table `あ`(a int);
ERROR 1300 (HY000): Invalid utf8 character string: 'あ'

という形でエラーになることはなくなります。ただし、
マルチバイトのデータベース名・テーブル名・コラム名を
使うことに伴う既知の問題があることはご留意下さい。
http://bugs.mysql.com/bug.php?id=3906

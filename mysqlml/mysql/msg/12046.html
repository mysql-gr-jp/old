想像ですが、負荷はさほど変わらないか、MySQL が若干大きいかも。
スピードは４系なら断然有利のはずです。５だとどうかわかりませ
んが。

ただ、負荷率については、OS の種類、設定でも大分違うようなの
で、何ともいえないのではないでしょうか。例えばプラットフォ
ームに Windows を使うと、さまざまなプロセスが結構じゃまを
します。かといって切ることができないプロセスも多いです。
GUI も切れませんしね。

> > Oracleは、契約によって
> > 「オラクルの事前の書面による承諾なく、対象プログラムの
> > ベンチマークテストの結果を開示すること」
> > ができません。

ということなので、結局は Oeacle のライセンスを購入して比較
することしかできないんでしょう。残念ですが。

ただ、一つ言えるのは、ブログ／ＣＭＳツールのバックエンドで
Oracle はあまり見掛けませんが、MySQL はかなり多いです。
Oracle はライセンスの高さもあるのでしょうが、PostgreSQL
など、他の RDBMS より多いところをみると、高速性が評価されて
の事と考えてもいいのではないでしょうか。

ただ、先にもかきましたように、Oracle と MySQL の方向性は
大分違うと思いますので、用途に依っては Oracle の性能が上回ら
ないとも限りませんけど。

どちらも　SQL92/99 はほぼ満たしているでしょうから、独自拡張
部分を使わなければどっちでもいいはずです。あとはコストの
問題でしょう。Oracle のランセンス料が気になるとしても、多少
の性能差は PC の CPU で XEON を使うとか、デュアルコアを使う
とかすると体感差はほぼ見られないのではという気もしますので、
金はそっちに振り向ける方法で検討するのもありでしょうね。

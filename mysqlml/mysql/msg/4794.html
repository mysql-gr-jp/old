たてやんっす

mysql 4.0.0 (alpha)から ssl 接続をサポートしたんで
なんとか使ってみたいなぁ　とおもう今日この頃

１回目は　rootユーザでもつながらなくって　断念したんだけど

もういっちょ　とおもって再チャレンジはじめたんっす

openssl-0.9.6b はあらかじめ入ってるんで

とりあえずいつもの　configure に--with-vio --with-openssl
をくっつけて　コンパイル　＆　インストール

それで　INSTALL-SOURCE に説明あるように
mysql_fix_privilege_tables.sh
を実行してみたんっすけど　なんかうまく　テーブル更新できない

ええい　もうしょうがない　ってわけで　mysql テーブル
(/usr/local/var/mysql) をリネームして
mysql_install_db　で新規に接続パスワードファイルを作ったら
とりあえずは　root でも使えるようになったっす


show variables like 'have_openssl';
+---------------+-------+
| Variable_name | Value |
+---------------+-------+
| have_openssl  | YES   |
+---------------+-------+
1 row in set (0.01 sec)

ちゃんとサポートされてるっす

さて　これから　SSL の　GRANT options　だぁ
でも　Basics　項を読み返さないとだめだろうなぁ　^^;

パスワードテーブル(mysqlデータベース) は　user テーブルに以下の
項目が追加されてました

+-----------------+---------------------------------------+------+-----+---------+-------+
| Field           | Type                                  | Null | Key | Default | Extra |
+-----------------+---------------------------------------+------+-----+---------+-------+
| ssl_type        | enum('NONE','ANY','X509','SPECIFIED') |      |     | NONE    |       |
| ssl_cipher      | blob                                  |      |     |         |       |
| x509_issuer     | blob                                  |      |     |         |       |
| x509_subject    | blob                                  |      |     |         |       |
+-----------------+---------------------------------------+------+-----+---------+-------+


ここまでやって　本業がめちゃ忙しくなってるしぃ　(;_;)

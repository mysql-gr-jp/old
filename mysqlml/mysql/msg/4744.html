Subject: [mysql 04744] ODBC ドライバ経由での CREATE TABLE でエラー

初めまして、山内と申します。最近、MySQL を利用して、データベースを使った
Web の構築を思い立ち、まじめに勉強し始めました。

うちでは、MySQL は FreeBSD で動作中の個人サーバにインストールしたので、
データを作成したり編集したりするクライアントに Microsoft ACCESS 97 を使い、
ODBC ドライバを経由して MySQL とのやりとりを始めたのですが、以下のような具
合で失敗します。どなたか、お知恵をお貸し下さい。

１．やりたいこと
　ACCESS 97 上で作ったテーブルを、MySQL 内のデータベースにエクスポートする。

２．使用しているソフト
  ACCESS 97(non SP) on WindowsXP professional 又は Windows2000 professional
  FreeBSD 4.4R + MySQL3.23.45
  ODBC ドライバ 2.50.39
  MySQL は Ports を利用せずに、ソースからコンパイル

３．現象
　ACCESS -> MySQL の接続には成功する。
　既存のテーブル（MySQL で作ったもの）のデータ内容は変更できる。
　ただし、テーブル構造は変更できない。
　エクスポートをすると、不正なＳＱＬ文となる。(syntax error #1064)

４．MySQL 側で見た、アクセスからのＳＱＬ文
　CREATE TABLE `member`(`call` set,`name1` set,`name2` set,`age` smallint)
　（ログに残っていたもの）
　この set が Syntax Error の原因と思われる。
　ちなみに、全てを smallint ,Double 等、数値にすれば エクスポートでテーブル
　が作成できた。

５．知りたいこと
　上記エラーの回避策
　また、テーブル構造を変更する方法があるか？

どうも MS-ACCESS が ODBC 経由で MySQL に投げた SQL 文がおかしいと言うこと
はおぼろげながら分かるのですが、ACCESS を見ても、SQL 文を異なる書き方にする
方法など、見つけることが出来ませんでした。

どなたかお力添え頂ければ幸いです。宜しくお願いします。

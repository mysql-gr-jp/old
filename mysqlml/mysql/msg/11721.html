MySQL 5.0 - FAQ
2005年6月23日
http://dev.mysql.com/tech-resources/faq.html
の一部一般の部分のみ

一般

GEN 1. 何時MySQL 5.0 はプロダクション・製品リリースになるのですか？
品質がリリーススケジュールの第一プライオリティです。現在 MySQL 5.0は
ベータですが、ガンマになりかけです。MySQL 5.0 をQA して頂ければ
幸いです。詳細は
http://dev.mysql.com/tech-resources/articles/evaluating-mysql-5.0.html

GEN 2. MySQL 5.0ではsubqueryはサポートされていますか。
MySQLはsubqueryを4.1版からサポートしています。
http://dev.mysql.com/doc/mysql/en/subqueries.html

GEN 3. MySQL 5.0 は複数のテーブル挿入とアップデートをサポート
しますか？複数のテーブル挿入とアップデートは4.0からサポートされて
おり4.1で強化されました。
http://dev.mysql.com/doc/mysql/en/UPDATE.html
http://dev.mysql.com/doc/mysql/en/DELETE.html

GEN 4. MySQL 5.0にはクエリ・キャッシュがありますか？
サーバー、インスタンスまたはデータベースで動作しますか？
クエリ・キャッシュはMySQL 4.0で紹介されサーバーのレヴェル
で動作します。これは結果セットを完全にキャッシュ
しオリジナルのクエリストリングにマッチします。
もし全く同じのクエリがなされた場合（これはウエブの
アプリで良く見られます）パーシングや実行は必要ありません。
結果は直接キャッシュに送られます。種々のチューニング
のオプションがあります。
http://dev.mysql.com/doc/mysql/en/query-cache.html

GEN 5. MySQL 5.0 にはSequenceがありますか？
ありません。しかし、MySQLにはAUTO_INCREMENTシステムがあり
MySQL 5.0では複数のマスターが存在するリプリケーション
の状況で挿入を処理できます。 --auto-increment-increment と
-auto-increment-offset のスタートアップ・オプションで
それぞれのサーバーが他のサーバーと重ならないauto-incrementの値を生成
することができます。auto-increment-incrementはサーバーの
数と同じか多くなければなりません。それぞれのサーバー
はユニークなオフセットが必要です。

GEN 6. MySQL 5.0はNOW()を秒単位で表示しますか？
これはMySQL製品予定表では"rolling feature"と呼ばれ、
キーの機能でないことを示します。時間があれば
実装されます。特定のカスタマーからのリクエスト
があれば変更することがあります。
MySQLは時間のストリングを秒単位でパースはします。
http://dev.mysql.com/doc/mysql/en/time.html

GEN 7. MySQL 5.0はディユアル・コアのOpteron上で動作しますか？
MySQLは完全にマルチスレッドなので、OSがサポートするので
あればマルチCPUを利用します。

GEN 8. "InnoDB Hot-Backup"のような"MyISAM Hot-Backup"がありますか？
これは現在開発中でMySQL 5.1で提供されます。

GEN 9. Foreign Keyが誤動作した際のエラーの報告が
改善されましたか？（現在はどの列とレファレンス
が誤動作したかレポートしない。）
InnoDBでのForeign Keyのサポートは版を追うごとに
改善されました。しかし、グローバルなforeign key
のサポート (全てのストレッジエンジンに通用する) は
MySQL 5.2 で予定されており、現在のストレッジエンジンの
特定の実装の不十分な点を解決します。

GEN 10. MySQL 5.0 はACIDトランザクションをサポートしますか?
MySQLは何年にも渡ってトランザクションをサポートしてきました。これは
3.23-MAX版依頼で、4.0版とそれ以降の全てです。InnoDB
ストレッジ・エンジンは完全なトランザクションで行レベル
のロックと複数版、ロックなしの繰り返し可能なリードと
SQLの標準のアイソレーションレベルを持ちます。

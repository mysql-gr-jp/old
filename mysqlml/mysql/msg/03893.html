On Wed, 04 Jul 2001 18:06:37 +0900
moeru@xxxxxxxxxx (moeru) wrote:

> 「MTAが直接リストにアクセスできて送信する仕組み」
> 
> この「リスト」の意味は、MySQLに登録した会員ＩＤ1番〜1000番までのユーザーテーブルの
> emailリストのことですか？
> 
> postfixを使うと、
> MySQLに登録した会員ＩＤ1番〜1000番までのユーザーにメールを送る場合
> プログラムでどういう処理をしているんですか？

例えば、次のようなテーブルがあって、

	create table hoge (id int, mail char(30), grp char(10));

/etc/postfix/hoge.cf を次の内容で作って、

	user = mysql_user
	password = mysql_pw
	dbname = mysql_db
	table = hoge
	select_field = mail
	where_field = grp

/etc/postfix/main.cf に次の記述をして、

	alias_maps = mysql:/etc/postfix/hoge.cf

んで、group1 宛にメールを送ると、hoge テーブルの grp フィールドが 
group1 である全エントリの mail 宛にメールが送られます。

> >最小限のコストでメールを送信できる
> というのは
> メールのキューに貯まるデータは、１通で済んだりするわけですか？

キューに溜まるのは１通です。SMTP セッションは宛先ごとにまとめられて
行なわれます。例えば、すべての宛先が同じサーバにリレーされるので
あれば、SMTP セッションも１回で行なわれます。

# あまり多いと分割されるでしょうけど。例えば Postfix のデフォルトでは
# 最大 1000 宛先。

> postfixとMySQLに関して参考になるようなＵＲＬをご存じでしたら
> 教えてください。

あまり見当たらないんですよね。でも、Postfix のドキュメント読んでやって

> mysql-3.22.27からmysql-3.23.28-gammaにバ－ジョンアップしたら
> mysqldumpがエラーするようになりました。
> キャラクタセットはSJISです
> 3.22.27の時は正常に動作していました。
> どなたかご教授ください。
> 
> mysql> create table チェ (a int);
> Query OK, 0 rows affected (0.00 sec)
> mysql> Bye
> # mysqldump test2 > /d/test2
> mysqldump: Can't get info about table: 'チェ'
> error: You have an error in your SQL syntax near 'ェ`' at line 1 

mysqldump が SJIS 用にコンパイルされていないとか。

SJIS で「チェ」は 83 60 83 46 で、60 == 「`」なので、「`チェ`」が
「`(80)`ェ`」とみなされて、「ェ`」が文法的におかしい。と言われている
ような気がします。

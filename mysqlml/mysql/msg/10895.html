「クエリ」の意味はこちらを参照いただくとして
http://e-words.jp/w/E382AFE382A8E383AAE383BC.html

上記を読んでいただければ分かると思いますが「クエリ」は別に
SQL で記述する必要がある訳ではありませんし、GUI な SQL でも
ありません。

例えば Access の MDB を VB から使う場合は SQL を使わなくて
も Jet データベースエンジン(DAO)から、或いは ActiveX データ
アクセスエンジン(ADO)から操作できます(但し内部で相当する
SQL に変換しているようです)。

MySQL ではクエリに相当する動作を SQL でやることが殆どである
と思われるし、またそういった事は既に基本として覚えている人
が対象なので、明確に記述していないだけだと思われます。

もう一つ、MySQL と直接関係ないのですが．．．

顧客に住所等のデータを入力してもらうとき、SSL 等で暗号化する
べきかな、と思います。顧客の方からすれば個人情報がネット上で
盗み見されるのは嫌でしょう？
立場を変えてみて、その辺の事も考えて作る必要があると思います。

本を見れば簡単に出来るように見えますが、本に記載されていない
問題、注意点も結構有るものです。こういうのがノウハウです。
書籍も千差万別、Web の情報も同様ですが、とにかく経験を積むこと、

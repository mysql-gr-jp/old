MySQL コミュニティ版(4.1.11)のリリース
これは4.1版の保守版で主に最近発見されたバグの修正で、以下を
含みます。

ONLY_FULL_GROUP_BY はANS　composite SQL　モードに含みません。
mysqld_safeはディレクトリーがない場合はUNIXソケット・ファイル
が格納されるディレクトリーを生成します。

これはディレクトリーパスの名前の一番最後の
コンポーネントのみに適用されます。
USER()やVERSION()などの関数から返還される値は``implicitで
はなく`system constant'に設定されます。
変更のリストは
http://lists.mysql.com/announce/273
ソースとバイナリのダウンロード
http://dev.mysql.com/downloads/mysql/4.1.html

コンサルティング、トレーニングと認定
トレーニング: MySQL Clusterによるハイアベイラビリティ (3日間)
複数日 San Francisco, London, New York City

MySQL Clusterのインストールとコンフィギュレーションを学びましょう。
この新製品でMySQLサーバーを全てクラスターでつなぐことが出来ます。
１つのサーバーでトランザクションを実行すれば、同じクラスター
のどのMySQLのサーバーでも結果を得ることができます。 MySQL Cluster
用のストレッジ・エンジンで実装されているハイアベイラビリティ
のについて学びましょう。それはストレッジ・ノード間のフェイル・オーバー、
ネットワークのパティション・プロトコル、２フェーズの
コミットやその他です。
詳細は
http://www.mysql.com/training/courses/mysql_cluster.html

Subject: [mysql 03080] Mysql 　における boolean 型についてお尋ねします

マニュアルを見ても、 FAQ でも Mysql ではブール型はどうも定義され
ていないようです。
例によって MS-ACCESS ９７で MS-ACCESS の３つのブール型（ 
true/false yes/no on/off) を Mysql へエクスポートして dump リス
トを取ったところいずれも tinyint(4)― true yes on は　１　 false 
no off 　は０としてデータ化されていまして。(MS-ACCESS 側のコント
ロールの持ち方によっては　１　が　－１　として入力されることもあ
ります）

  そこでお尋ねですが Mysql のでブーリアン型を定義するときどのよ
  うな型定義をするのがもっとも妥当なのかどなたかお教えいただける
  と大変ありがたいです。上記のような場合 とりうる値が　０、１　
  なら tinyint(1) で定義してもよさそうですが・・・ PostgresSql 
  には４バイトのブール型があります。互換性やら拡張性（よくわかっ
  ていっているわけではありません）を考えれば tinyint(4)が正しい
  のでしょうか？それにそもそも Mysql にブール型がないのはなぜで
  しょうか？（マニュアルにはもちろん FAQ にも それとも何か見落と
  してるのか・・・）

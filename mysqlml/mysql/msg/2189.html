「Delete と ＵＰＤＡＴＥがひどく遅い」とちょっと刺激的な題名の
質問をした鈴木です。


週末にも関わらず、いろいろと教えていただきましてありがとう
ございます。

ご指摘頂いた点として：
（１） ＳＱＬの問題があるかもしれないこと。
    つまり、クオートで囲むということ。
       ×     delete from testtable where id = 000000000001;
       ○     delete from testtable where id = '000000000001';

（２） ＭｙＳＱＬはまじめにコンパイルして使った方がいいこと。
        当該マシンの開発環境がいまいちなので、バイナリを拾ってしまいまし
たが、
        開発環境から整えてソースからＭｙＳＱＬを作ってみます。

    ＃ しかし、ＬＩＮＵＸでも試してみるために、ソースからコンパイルし、
    ＃ 上のＳＱＬを走らせたところやはり＊遅かった＊ので、
    ＃ 質問したという経緯があります。


ということですが、そうですね、多分（１）なのでしょう。
残念ながら当該環境（マシンとＤＢ）は勤務先にありますので、
月曜日に（１）、（２）についてさっそく試して、
結果をご報告させていただきます。


いろいろありがとうございました。

うーむ。

a. --skip-grant-tables で 一旦とにかく mysql 5.0 起動して 4.1 から引き
   継いだ mysql.xxx そのものずばりを mysql_fix_privilege_tables にかける。

b. mysql 4.1 環境下で mysqldump -c -t して、mysql 5.0 で新たにつくった
   mysql.xxx に食べさせる、

c. mysql 5.0 で新たにつくった mysql.xxx を起点に、清く正しくcreate
   user とか grant なんたら on *.* ... を しこしこ 発行、

いま見ましたら、tables_priv や columns_priv のスキマも 4.1 と 5.0 では
微妙に変わってます (null = YES/NO とか、Table_priv の種類)。そのためも
含めて、 mysql_upgrade であるとかmysql_fix_privilege_tables があるんで
しょうけど。そう考えたら、c が後腐れの心配が少ないのでは、と言われれば
そうですね。ただ、c ですと私が入力するときの間違いの危険性が最大だった
りいたします。

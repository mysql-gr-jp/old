次の手順でコンパイル＆リンクできました。
皆様ありがとう御座いました。

＜開発マシン＞
・Redhat9をMySQL-Server（MySQL-3.23.58-1.i386.rpm）をインストールしない
状態にする。
＃MySQL-client-3.23.58-1.i386.rpmをインストールするとき、以下のメッセー
ジが出てインストールできない。
 file /usr/bin/mysql from install of MySQL-client-3.23.58-1 conflicts with file from package mysql-3.23.58-1.9
＃よって、他にMySQL-Serverをインストールしていないマシンを用意しました。

・以下のパッケージをインストールする
  MySQL-client-3.23.58-1.i386.rpm
  MySQL-devel-3.23.58-1.i386.rpm
  MySQL-shared-3.23.58-1.i386.rpm

・コンパイル＆リンク
  gcc -I/usr/include/mysql -L/usr/lib:/usr/lib/mysql -lmysqlclient -lz a.c

本来（？）の手順とは異なるのかもしれませんが、現在のところ、順調にプログラミングできてい
ます。
C言語で開発するときはMySQLのソースをインストールするのが順当なのでしょうか？
上記、手順に間違い等ございましたらご指摘いただければ幸いです。

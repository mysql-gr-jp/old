鶴長です。

>% mysql vpopmail
>ERROR 1044: Access denied for user: '@localhost' to database 'vpopmail';
>
>と、接続できませんでした。
> mysql -uvpopmail vpopmail
ではどうでしょうか。

>で、./vadddomain は通ったのですが、
>今度は、以下のように怒られてしまいました。
>MySQL に接続できてないようなのですが何故なのでしょうか？
>
># ./vadddomain hogehoge.or.jp
>Please enter password for postmaster:
>enter password again:
>could not connect to mysql server
ユーザー名・パスワードをconfファイルで変更できればいいの
ですが、そうは出来ないので、makeする前に、vmysql.hの
#define MYSQL_SERVER        "ＸＸＸＸＸ"
#define MYSQL_USER          "ＸＸＸＸＸ"
#define MYSQL_PASSWD       "ＸＸＸＸＸ"
が山本様の環境と合っているかどうかなのですが。もしあって
いれば、同じくvmysql.hで指定されている、それぞれのテーブ
ル名が一致していない可能性があると思います。
メッセージを見る限りではやはりパスワードのあたりだと思い
ます。私はこのあたりが面倒で、root@localhostで権限を与え
てしまいました（それもno password）。どうやっても動作が出
来ないときはパスワードを無くすのもありかもしれません。
それで動作できるようなら、1連の作業が終わった後で立ち返
るのも手かと。

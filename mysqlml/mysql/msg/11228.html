MySQL Community版(4.1.10a) の新しいリリース
これは4.1版の保守リリースで、主に最近発見された
バグを修正しています。このリリースは

セキュリティ改善: サーバーはすでに同じ名前が
存在していないのであれば .frm, .MYD, .MYI,.MRG,
.ISDと ISM テーブルファイルを生成します。
セキュリティ改善: User-defined 関数(UDF)はメインの
`xxx()'関数に対応する`xxx'に１つのシンボルを
追加する必要があります。この変更はUDFがロードされる方法に
影響がありますので、ご注意ください。マニュアル
http://dev.mysql.com/doc/mysql/en/udf-security.html
の"User-defined Function Security Precautions"の章を
参照してください。

新情報に関しては
http://lists.mysql.com/announce/268
ソースやバイナリのダウンロードは
http://dev.mysql.com/downloads/mysql/4.1.html

MySQL (4.0.24)のリリース

このリリースは

'InnoDB': テーブルがフルになった場合'InnoDB' tablespaces
を拡張するためのサイズをメガバイトでセットできる
'innodb_autoextend_increment'変数とコンフィギュレーションの
オプションを追加。デフォルトは8で、これは以前のMySQL.Schemas
の版で8MBごとしか増加できなかった機能を置き換え。
InnoDB: 'AUTOCOMMIT=1'であれば'LOCK TABLES'に
'InnoDB'テーブルロックを掛けない。こうすることで
古い'MyISAM'アプリを`InnoDB'にポートしやすくなる。
'InnoDB' テーブルロックはデッドロックを簡単に引き起こす
可能性あり。

新情報に関しては
http://lists.mysql.com/announce/269
ソースやバイナリのダウンロードは
http://dev.mysql.com/downloads/mysql/4.0.html

User-defined Function (UDF) セキュリティの注意
MySQLはUDFが利用されて問題を引き起こさないないように注意。

CREATE FUNCTIONにはINSERTプリビレッジが必要。DROP FUNCTION
にはDELETE　プリビレッジが必要。UDFはメインのxxx()関数
に対応するxxxシンボルの他少なくとも１つのシンボルを追加
すべき。UDFのオブジェジェクトファイルはどこのディレクトリー
に格納しても良いということではない。

詳細は
http://dev.mysql.com/doc/mysql/en/udf-security.html

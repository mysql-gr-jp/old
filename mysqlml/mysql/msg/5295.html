From: じょん <じょん <john@xxxxxxxxxx>>

Subject: [mysql 05295] MySQL+qmail+ezmlm-idx

初めまして。前田（MySQL初心者）と申します。

TurboLinux 7 Server（ftp版）でqmail+ezmlm-idx+mysqlを構築しようと
しているのですが、exmlm-idxのmakeでエラーが出てしまいます。

エラー内容は

/usr/lib/mysql/libmysqlclient.a(my_compress.o): In function 'my_compress_alloc':
my_compress.o(.text+0xb8): undefined reference to 'compress'
/usr/lib/mysql/libmysqlclient.a(my_compress.o): In function 'my_uncompress':
my_compress.o(.text+0x12a): undefined reference to 'uncompress'
collect2: ld returned 1 exit status
make: *** [ezmlm-gate] エラー 1

です。
mysql側？のエラーのようなのですが、素人なもので対処がわかりません。
mysql自体はTurboLinuxのrpmで
 MySQL-3.23.39-4
 MySQL-bench-3.23.39-4
 MySQL-shared-3.23.39-4
 MySQL-client-3.23.39-4
 MySQL-devel-3.23.39-4
zlibは同じくTurboLinuxのrpmで
 zlib-1.1.4-2
 zlib-devel-1.1.4-2
が入っています。
qmailはqmail-1.03、ezmlmはezmlm-0.53とezmlm-idx-0.40です。
あとvpopmail-5.2、qmailadmin-1.0.2等が入ってます。
TurboLinux7S自体はTurbopkgによる管理で一応最新状態です。

手順としては
tar zxvf ezmlm-0.53.tar.gz
tar zxvf exmlm-idx-0.40.tar.gz
mv ezmlm-idx-0.40/* ezmlm-0.53/
（すべて上書き）
cd ezmlm-0.53
patch < idx-patch
make mysql
make clean
make
ここで上記のエラーが出ます。
make mysqlをしないとmakeできるのですが、mysql対応にはなりません（多分）。

make出来た方、また対処方法をお知りの方がいらっしゃいましたら
ご助力お願い致します。

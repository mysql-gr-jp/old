結局、MySQLのデータベースの復旧は、先週に出来ていたのですが、
WordPressの復旧に手間が掛かってしまい報告が遅く成ってしまいました。
で、本日無事復旧出来ました。
http://blog.nakasatsunai.jp/

frmとibdファイルから復元した時、wp_optionsとwp_postmetaに損傷が有ったようです。
その為、mysqldumpが途中で止まっていました。
復元後、この二つのテーブルを除いてmysqldumpを取りました。①
wp_optionsの単体でmysqldumpが取れたので、これはそのまま使い、②
wp_postmetaに関しては、20201212の時の物を使って、mysqldumpを取りました。③
この３個（①②③）えぃｔのdumpファイルで、データベースを復元したものは、スムーズのmysqldumpが
取れる様になり、データベースは、復元出来たと判断しました。

そのデータベースを使って、本日WordPressの復元が出来ました。

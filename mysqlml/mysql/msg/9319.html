MySQLユーザーコンファレンスの報告
今年のMySQLユーザーコンファレンスとエクスポは
600人以上が参加し、100以上のセッション、事例、ワークショップ、
とラボで成功裏に終わりました。
全体の要約を知りたい方は、 Mike Hillyerのユーザーコンファレンス
Blogコレクションを参照してください。50以上のBlogが
ありキーノートやセッションの多くをカバーしています。
http://www.vbmysql.com/uc2004/

このBlogを読めば、役に立つMySQLの真髄や
どのようにYahooやSabreがMySQLを内部で使用しているか
を含むベストプラクティスが分かります。

MySQL 事例: B2 がMySQL Clusterを使用
Bredbandsbolaget (B2) はスエーデン最大のブロードバンド
ISPです。 B2 は MySQL クラスターを使用してブロードバンド
でインターネットへのアクセスとVoIPサービスを提供します。
どの様にして Bredbandsbolaget (B2) がネットワークを
安定化させるのに何日もかかっていたのを数分でできるように
なったかの説明を読んでください。

今年のアプリケーションとパートナー賞がユーザコンファレンス
で発表されました。この賞は会社やユーザに大きな経費節減や
性能や信頼性の改善をもたらす大規模コンピュータシステム
でMySQLを使用したユーザやパートナーに与えられます。

詳細は
http://www.mysql.com/news-and-events/news/article_551.html

製品

MySQL ABはMySQL Clusterを発表ーオープンソースのクラスタ・
データベース初の解
今年のユーザコンファレンスでMySQL ABは高い信頼性
を求められるアプリケーションのための新しいオープンソース
データベースのクラスター技術であるMySQL Clusterを発表しました。
MySQLクラスタはMySQLとクラスター・アーキテクチャを
結合してミッション・クリティカルなデータベースのアプリケーション
用に99.999%のアベイラビリティを提供します。

技術ホワイトペーパ: MySQL クラスター・アーキテクチャの要約
このペーパでメモリー内でクラスター化する分散データベース
マネージメントシステムであるMySQLのクラスタのアベイラビリティ
と信頼性機能について述べます。MySQL クラスターは完全に独立
したアーキテクチャに基づき以下のような高度な機能を含みます、
フェイル・オーバー、ノードリカバリー、
シンクロナス・データ・リプリケーションとシングルーポイント
フェイリュアなし

ホワイトペーパはここから
http://www.mysql.com/it-resources/white-papers/cluster-technical.php

ヒント

MySQLストレッジ・エンジン・アーキテクチャーその１：要約
これはウエブの新しい開発ゾーンで述べられている
MySQLストレッジ・エンジン・アーキテクチャーに関する
３回シリーズの第一回めです。

MySQL のサーバーにはストレッジ・エンジンあるいはテーブル・タイプと
よばれる概念があります。サーバと開発者はどこにどのように
データベースのテーブルが格納されるかを決めることが出来ます。
開発者とデータベース管理者は通常データベースのサーバが
使用する実際のストレッジにはタッチしません。
MySQLのユーザはMySQLのストレッジ・エンジン・アーキテクチャ
を考慮すれば更にアプリケーションを最適化することが出来ます。

詳細は
http://dev.mysql.com/tech-resources/articles/storage-engine/part_1.html

PHPからMySQLを使用する

O'Reilly社の ONLamp.comのJohn Coggeshall氏がPHP と MySQL
を使用してデータベースに基づくウエブページのつくりかた
を記事にしています。この記事はデータベースに基づくアーキテクチャ
から始めて、データベースへの接続、クエリの方法、結果の
受けとり、接続の終了などをカバーします。

記事はここから
http://www.onlamp.com/pub/a/php/2004/02/19/php_foundations.html

> INSERT ... VALUES ..., ..., ... ON DUPLICATE KEY UPDATE ...のSQLは、
> 「VALUES ..., ..., ...」←ここで一度にバルクINSERT (or UPDATE)する行数の
> 二乗に比例した時間がかかります。
> 
> レコード長100バイト、13万レコードのテーブルで
> INSERT ... ON DUPLICATE KEY UPDATEをすると、

> 
> 125行：0.06秒
> 250行：0.26秒
> 500行：1.01秒
> 1,000行：4.04秒
> 2,000行：16.09秒
> 4,000行：64.00秒 (このときのテスト用SQL文は453KB)
>

Subject: [mysql 10738] MySQL 4.0.23aが正常にコンパイルできない (Was:MySQL 4.0.23の共有ライブラリの問題)

いつもお世話になっています。S-Lines Network Service の 上田 です。

先日、MySQL 4.0.23がリリースされたため当方のバージョンをアップを
行おうとしているのですがちょっと気になる点があるために投稿させてもらいます。

すでに、本家のMLでは話題になっているようなのですがMySQL 4.0.23で作成される
共有ライブラリの一部がおかしな形で生成されるバグが存在するようです。

実際に細かく確認をしたわけではないのですが、どうもファイル名が違って
作成されているような感じに見えます。

http://lists.mysql.com/mysql/177752 の引用元にあるように、MySQL 4.0.22と
MySQL 4.0.23では共有ライブラリの名前が変わっています。
すでにバグとして認識され、MySQL 4.0.23aというバージョンで修正版がリリース
されているようなのですが、そのMySQL 4.0.23aはドキュメント関係のmakeでこけてしまいます。

> Making all in Docs
> cd . && /bin/sh /usr/home/tatsuya/temp/mysql-4.0.23a/missing --run makeinfo --no-split -I .
> makeinfo: missing file argument.
> Try `makeinfo --help' for more information.
> *** Error code 1

と言うようなエラーが出てしまいます。

このエラーはMySQL 4.0.23では発生しないため何かが変わっているのだと思い
調べてみたところ、 http://lists.mysql.com/internals/8016 の記事が該当
するようなのですが返信で「投稿する場所が違う」と指摘されたためか結局、
解決の方法がありませんでした。

確認すべき点などもわからず、先に進めずに居ます。
お手数ではありますが、なにか情報をお持ちの方は教えていただければ幸いです。

藤原です

  普段はLinux使いなんですが、ちょっと訳あって Windows XP + Apache 2.0
+ MySQL 4.1.10a + PHP5 （WAMP）の構成で動かしています。
  まだ詳しく調べた訳ではありませんが、日本語も一応問題なく動いているよ
うです。データベース名、テーブル名にも漢字が使え、UTF-8のフォルダやファ
イルがちゃんとできました。

質問というか、Windows版を初めて使っていてちょっと奇異に感じたことなん
ですが、

mysql> show variables like 'char%';
+--------------------------+-----------------------------+
| Variable_name            | Value                       |
+--------------------------+-----------------------------+
| character_set_client     | sjis                        |
| character_set_connection | sjis                        |
| character_set_database   | sjis                        |
| character_set_results    | sjis                        |
| character_set_server     | sjis                        |
| character_set_system     | utf8                        |
| character_sets_dir       | c:＼mysql＼share＼charsets/ |
+--------------------------+-----------------------------+

のように表示されます。パス情報を表示させると、区切り記号が \ ではなく
＼ になってしまうのですが、これはこういうもの（？）なんでしょうか。

character_sets_dirの場合、途中は ＼ で、最後が / で混在してしまってい
るのも変なんですが。

character_sets_dir以外は、きちんと全部 ＼ で表示されるみたいなんです。

恐くてまだ試していないのですが、＼、\、/ を混在して記述してもちゃんと
動作するんでしょうか？

このあたりは、4.1の国際化以降変化したものなんでしょうか。

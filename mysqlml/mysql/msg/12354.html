MySQL 4.1.x は、クライアントとサーバ間で文字コードが異なっている場合、
自動変換を試みます。

今回は両方とも latin1 だったので変換はされてないと思いますが、おそらく
テーブルも latin1 で作られていると思うので、テーブルを作り直す必要があ
りますね。

「SHOW CREATE TABLE テーブル名」で、テーブルとフィールドの文字コードが
わかります。

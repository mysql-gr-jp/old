そういう商売で十分だったらそれはそれで役割分担なんでしょうね。

それぞれにスキルを上げればいい、というのは同意。
わたしも Oracle/MySQL/PostgreSQL/DB2/SQLServer と経験してきましたが、
適材適所というものはあります。

- 某携帯キャリアにて Oracle の OPS が高かったので MySQL 12台 で分散した
- ホストからのダウンサイジングで Oracle だとライセンス費用が億になったので
  DB2 になった
- 止められない決済系のシステムでは Oracle 10g RAC + DataGuard 

などなどをやってきました。

元質問者の「上司を説得できない」というのは、
まだまだ未熟だということでしょう。
いまのわたしなら Oracle が高いと言われたら
MySQL よりは DB2 になってしまいますが。

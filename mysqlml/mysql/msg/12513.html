Subject: [mysql 12513] Mike Zinnerのインタビュー

http://dev.mysql.com/tech-resources/interviews/mike-zinner.html


Mike ZinnerはMySQL GUI開発チームのリーダです。現在はMySQL Workbench
に集中しています。このツールはビジュアルにデータベースの
スキームをデザインしたり、閲覧するためのものです。
これはMikeの前のDBDesigner4を引き継ぐものです。

（聞き手はコミュニティ・リレーション・マネージャーのArjen Lentz ）

Q: GUIチームの仕事の内容を教えてください。

MySQL Administrator, MySQL Query Browser, MySQL Workbench,
MySQL Configuration Wizardなどのツールをデザインして実装することです。

私のチームには３人いて、 WindowMaker担当のAlfredo Kojima
はLinuxと OS X 版の仕事をしています。

Mike Lischkeはコンポーネントのグールーですが、私と一緒にとWindows版の
GUIツールの仕事をしています。

Vladimir KolesnikovはLinux, WindowsとOS X 版の仕事をしています。
メインはバグとビルドと新機能です。

Q: 今までで関わった仕事で一番誇りに思うのはなんですか。

Configuration Wizard付きのMySQL Server Windows Installerは
簡単だけど優秀なGUIが物事をスピードアップしたり、
ユーザにとって使い勝手がよくなるかの例です。

一般的に言って、GUIチームはサーバーチームが求める
レベルのツールを提供することになっています。そういう
ツールでユーザの生産性が更に上がるものです。

それぞれのプラットフォームの上で、ネイティブの開発が
可能なようにライブラリーやコンポーネントを開発しました。

Q: 素晴らしい。ツールのことはまた後で聞くとして、
まずあなたのことを教えてください。

３０歳でAustriaのViennaに住んでいます。オフィースは
Viennaから15kmほど離れた小さな村で、Viennの森林地帯の真ん中にあります。 
(Wienerwald).
ここの静かな環境が気にいっています。夏にはThinkPad とPowerBookを
持って庭で座って仕事をします。

小さなスタジオを持っており、歌を作曲したりレコーディングをするのが
好きです。ヴィデオの製作やグラフィカル・デザインなども行い
ます。スノーボード、バイク、ボーリングやダイビングも好きです。

Q: MySQLのようなバーチャル・カンパニーで仕事をするのはどうですか？

バーチャル・カンパニーのコンセプトが好きです。
世界中でどこにいようと、もっとも才能のある人々と一緒に仕事が
出来るからです。GUIの開発というのは分散しているチームにとって
はかなりのチャレンジです。VoIPとVNCを使えば可能になります。

Q: どういういきさつで、MySQL ABに参加するようになったのですか？

MySQL (3.23の初めの版から)を使い始めたのコマーシャル版
のデータベースにあきあきしていたからです。 主にウエブ
のアプリ、計算とログのアプリに使いました。

学生のうちに自分の会社を設立しました。ソフトの開発
のほかに、ラジオやTVのコマーシャルを製作していました。
断るには良すぎる仕事のオファーがあり、ローカルの
ソフトのディストリビュータのソフト開発部門の責任者
になりました。それからMySQLに来る前は Austriaの
一番大きな印刷会社のIT部門のマネージャをやりました。

暇なときに、fabFORCE DBDesigner4というオープンソース
のプロジェクトに関わっていました。それはMySQLの
ビジュアルなデータベースのデザイナーでした。

Georg RichterがKarlsruheで開催されたLinuxTagに招待して
くれました。そこで初めてMySQLの人々と会いました。何ヶ月
かして、Kaj Arn がViennaに休暇でやってきて、そのときの
ミーティングの後、そのときの仕事をやめてMySQLに
参加することにしました。それは2003年の10月でした。

オープンソースの開発をしたいのなら、情熱を持って
参加できるのはほんの一握りの会社しかありません。MySQLは
単に一番良い会社なんです。皆にオープンソースの
スピリットに参加して欲しいと思います。というのも
それがソフト開発の将来だと思うからです。ソフトの
特許と戦いましょう。

MySQLにとってはコミュニティは大変重要です。全てにスタッフ
がコミュニティに関わるように言われています。それで
今までに多くのコンファレンスで話をしていますし、フォーラム
にも書いています。 http://forums.mysql.com/.

Q: 素晴らしい。で今何を主にやってますか？

MySQL Workbenchはビジュアルにデータベースのスキーマを
デザインするツールです。これは私の以前のDBDesigner4の
次の版というところでしょうか。これは非常に興味のある
プロジェクトで、最新の技術を駆使しています。モジュール
ベースの実行環境を利用しています。これを使用すればユーザ
は自分のプラグインを追加でき、全てをスクリプト化できますし、
レイアウトのツールやその他 DBDesigner4でなかった機能
を使用できます。

先月何百というメールを受け取りました。皆プロジェクトの
進捗を知りたがっています。多くは励ましの言葉でした。
支持してくださっているコミュニティに感謝したいと思います。

２週間前に最初のアルファ版をリリースできてとても嬉しいです。
多くの人はこれを２年も待っていましたから。まだやり残しが
多くあります。しかし、一旦完成すれば今までのなかで、
一番パワフルなオープンソースのデータベースのデザイン
ツールとなると思います。

来年4月の MySQL User Conference に来てGUIツールの
プリゼンを見てください。[Arjenの注: 同感です。
マイクのデモを見た人は誰でも、すごいといいますよ。]

システム構築する際に、ブラウザに表示させる文字コードを EUC-JP にしない
ように事を徹底すれば、問題は起きないので MySQL で CP51932 を対応する必
要性はないとも言えます。

しかし、PHP や Perl だとスクリプトを EUC で書いて、文字コード変換をせ
ずに、HTTP の入出力、DB の入出力を行うという事をやると問題が起きること
になります。

まず、eucJP-ms と CP51932 は別物であるという認識を共有する事が必要ある
のだと思っています。

あと、MySQL 単独の問題として解決は困難なので、MySQL の DB にアクセスす
るソフトでの文字コードの取り扱いをどうすべきかという点にも言及する必要
があるでしょうね。

2.4.17.2. Upgrading from MySQL 4.1 to 5.0 より
---
Incompatible change: MySQL 5.0.3 and up uses precision math when
calculating with  DECIMAL and integer columns (64 decimal  digits) and
for rounding exact-value numbers. Rounding  behavior is well-defined,
not dependent on the  implementation of the underlying C library.
However, this  might result in incompatibilities for applications that
rely  on the old behavior. (For example, inserting .5 into an  INT
column results in 1 as of MySQL 5.0.3, but might be 0 in older
versions.) For more  information about rounding behavior, see  Section
21.4, "Rounding Behavior", and  Section 21.5, "Precision Math
Examples".
---

RENAME を試してみました。ある接続で、

	mysql> insert into hoge values (99);
	mysql> select * from hoge;
	+------+
	| i    |
	+------+
	|   99 |
	+------+

ここで別の接続から「rename table hoge to fuga」を実行。

	mysql> select * from hoge;
	ERROR 1146 (42S02): Table 'test.hoge' doesn't exist

この時点で別の接続から「select * from fuga」しても空なので、COMMIT さ
れた訳ではないみたいです。だけど、トランザクション中なのに、他からテー
ブル名が変更された影響を受けて、続くクエリがエラーになってしまいます。

ちなみに、リネーム後のテーブル名を指定すればちゃんとアクセスできます。

	mysql> select * from fuga;
	+------+
	| i    |
	+------+
	|   99 |
	+------+

COMMIT すれば、ちゃんと他の接続からも挿入されたデータが表示されます。

トランザクション中で RENAME するというのも試してみたんですが、その場合
はその時点で COMMIT されるようです。トランザクション中で挿入したデータ
は、RENAME した瞬間に他の接続からも見れるようになりました。

トランザクション中で RENAME すると COMMIT される…というのはともかく、
トランザクションで使用しているテーブルが他の接続から RENAME できるのは
バグのような気がします。なんとなく…。

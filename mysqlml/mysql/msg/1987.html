コンパイルしようとしているマシンに、アセンブラが
入っていないのかもしれません。

Linux の場合は、configure & コンパイルに以下のコマンドが必要です

as
as86
ld
ld86
cpp
gcc
automake
autoconf
m4
flex
bison
perl
make


( ./configure --disable-assembler
  で、アセンブラの利用をスキップすることも可能。)

開発に必要な include ファイルもインストールしておきます

コマンドや、開発用のファイルが、どのパッケージに入っているかは
知りません。
探して インストールしてみてください


また、3.22.15-gamma は古いので、
新しい 3.22.32 のソースをためしてください
3.22.15 の configure に問題があることも考えられます。

###
Linux の場合、配布物の名前やバージョンの情報よりも
重要な情報は、以下です

uname -a
ls -la /lib/libc*
cat /etc/ld.so.conf

gcc --version
gcc -v
gcc -print-libgcc-file-name
gcc -print-search-dirs

perl -V
env | grep -i path
env | grep -i library

単に redだのと言うよりは、より多くのヒントが得られます
配布物の名前やバージョンの情報は、パッケージ探すときのヒント
程度にしかならない事が多いです

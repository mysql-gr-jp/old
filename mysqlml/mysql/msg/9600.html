HDD は ATA100 なのでしょうから問題ないでしょうが、Memory は
幾ら積んで居ますか。128MB とかだとスワップが働くばかりで
パフォーマンスが出ないと思います
CPU のクロック云々よりそちらの方が問題です

それから CPU に関して言えばセレロンというのも気になります
セレロンはキャッシュの容量が少ないので、クロックが低下しても
pentiumIII にすること(できれば FSB133MHz 品)にすることをお薦め
します

あとせっかく Linux なのですから GUI (X Window)は落としておい
た方がいいですよね^^;

私も MySQL と Access の両用使いですが、ある場面においては
MySQL が、またとある場面では Access の方が有利です。
前者は RDBMS であり、Access は MSDE があろうとなかろうと
Personal DB の域を出ないと思います

そもそも比較すべきじゃないのでは？？？
苦言になりますが速度が云々よりも、データ量が無限に大きくなる
のであれば最初から Aceess を採用すべきではないです
月単位でバックアップ取ったら消す、なら別ですが

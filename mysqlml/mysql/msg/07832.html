　初めて投稿致します。nisi と申します。

　MySQLの日本語SELECTに関して気になる挙動がありましたので、
　ご連絡させていただきます。
# 過去ログは確認したつもりですが、見落としていましたらすみません。

　当方、http://plantl.org/l/munou/margarine/ の人工無脳の記
　憶部分にてMySQLを利用しています。

　確認したバージョンはWin版で4.0.12-ntですが、Linuxでも同様の
　現象が出ています。

  CREATE TABLE marg_words(
    tno INT NOT NULL AUTO_INCREMENT PRIMARY KEY,
    str_m1 VARCHAR(255),
    str_y1 VARCHAR(255),
    str_m2 VARCHAR(255),
    str_y2 VARCHAR(255),
    str_p VARCHAR(255),w_w INT );

　にて作成したテーブル str_m1 に 日本語一文字 ’は’ のレコード
　を挿入、引き続き 'わ' を挿入。

  insert into marg_words(str_m1) values('は');
  insert into marg_words(str_m1) values('わ');

　その後、'は' を select で検索したところ、

  select * from marg_words where str_m1 = 'は';

 +-----+--------+--------+--------+--------+-------+------+
 | tno | str_m1 | str_y1 | str_m2 | str_y2 | str_p | w_w  |
 +-----+--------+--------+--------+--------+-------+------+
 |   3 | は     | NULL   | NULL   | NULL   | NULL  | NULL |
 |   4 | わ     | NULL   | NULL   | NULL   | NULL  | NULL |
 +-----+--------+--------+--------+--------+-------+------+

　という結果になります。
　「わ」がなぜ select されるのかわからないのですが、設定か文字コード
　などの問題でしょうか？

　よろしくお願い致します。

MySQL ユーザコンファレンス2005 - 論文募集中
カリフォルニア州、シリコンバレーにて
2005年4月18-21日

MySQL ユーザコンファレンス2005はMySQLの開発者とユーザ
の集まりで最大規模のとなる予定です。個人と企業の方で
発表したり、チュートリアルを提供することを
望まれ方の提案を募集しております。今回のテーマは
「 MySQLはどこにでも。」です。提案書は参加者の
役に立つものに限ります。

ベストプラクティスを含むMySQLの力を最大限に引き出し
MySQL 5.0の新しい機能を学び使いましょう。
様々なアプリ(ウエブ、データウエアハウス、クラスタ、組み込み)
でMySQLを使った専門家から学びましょう。
プロプリエタリーのデータベースからMySQLへマイグレートしましょう。
詳細は
http://www.mysqluc.com/cs/mysqluc2005/create/e_sess

事例: ソニーがMySQLでデータベースのコストを削減
ソニーのコンテント管理システムであるFileNetはターンキーで
はありません。FileNetはMicrosoft SQL サーバーとASPを使い外部
のコンサルに依存してカスタマイズする必要があります。ソニーは
他のプロジェクトでMySQLを使用して成功していたので、コンテント管理システム
をLAMPスタックを使って書き直し始めました。
詳細は
http://www.mysql.com/it-resources/case-studies/mysql-sony-casestudy.pdf

事例: Fortum はMySQLを使用してリアルタイムで 発電所をモニター
スエーデンのストックフォルムの近くのFortumのHogdalan発電所 はヨーロッパ
のなかでも一番モダンなゴミ焼却地です。法的な規制と将来の
ビジネスチャンスのためにデータを毎分から毎秒に収集する必要
が出てきました。 FortumはMySQLを使ってデータウエアハウスの解
を作ることにしました。その性能、信頼性と柔軟性を評価した
からです。
詳細は
http://www.mysql.com/it-resources/case-studies/mysql-fortum-casestudy.pdf

製品
MySQL (4.1.6 Gamma)がリリース
これは4.1 ガンマの３度目のリリースです。大部分はプロダクション
リリースに向けてのバグ修理です。
 * InnoDB: パージがなかなか作動しないとき用に、スタートアップ
オプションを追加し、更にグローバル変数のinnodb_max_purge_lagも追加
してINSERT, UPDATE and DELETEのオペレーションを遅延させます。
I* InnoDB: 4.1.5で挿入されたinnodb_autoextend_increment のスタートアップ
オプションはセットされるグローバル変数として扱えるようになりました。
TIMESTAMPコラムはNULL値を格納できるようになりました。
日本語のリンクは
http://lists.mysql.com/mysql-ja/83
ダウンロードは
http://dev.mysql.com/downloads/mysql/4.1.html

事例: CanadaDrugs.com がEmicアプリケーション・クラスターと
MySQLを使用して安全でセキュアなプロセスで薬を提供
CanadaDrugs.com はMySQLとLinux上にEmic Application Cluster (EAC)を使用して
費用効果の高く、スケーラブルでハイアベイラビリティなシステム
を建設しました。この構造のおかげで、品質を落とすことなくオープンソース
とコモディティのハードを使用することでの節約されたコストを
カスタマにも提供しました。

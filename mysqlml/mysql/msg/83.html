漢字の対応は次のようなアプローチがあると思います
 1.client がわで対処
 2.server で変換。内部コードは一定
 3.server が内部から完全にそのコードになる(現在 ujis のみあり)


私は次の点で 2 でなんとかできないかと考えました。
（でも民斗さんの思索のけっか(#78)、この方法はあまりうまくないという
  報告がなされました。現在、内部コードを SJIS で思索中とのこと(#80) ）

 a. MSの開発環境を持っていない。また揃えるつもりもない。

 b. client で対処すると、違うクライアントが出たら
    それごとに書き直さなくてはならない。
      ( ただし MySQL のばあい、libmysqlclient.so を書き換えれば
        その作業は軽減されると考えています )

 c. server を 内部から完全に変えてしまうと、
    漢字コードの数だけサーバーを作るはめになるので、これも大変そう

 d. そういえば Oracle は 使用コード SJIS, EUC を指定できる、、、

 e. やっぱり理想は、どんな漢字コードの client がきても、
    それに自動で対処してくれる server がいいなぁ、、、

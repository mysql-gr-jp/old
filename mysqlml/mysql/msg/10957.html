ちょっと誤解が有るようなので(私の表現もまずかったかもしれま
せんが)、補足します。

> 1)データベースを組む必要がないかもしれない

組まないと、どう検索します？？
例えば namazu を使うにしても、キーワード検索を掛ける訳です
から、画像が相手ですから、キーワードをツリー形式でも DB でも
いいから引っ張ってこないとなりませんよ

> 5)HDD選びによってスピードが変わる。

これそのものは間違っていません。確かに現在の PC では HDD の
速度が重要です。しかし、DB を組んでしまうと結局は検索のとき
のパフォーマンスを出すために DB や SQL のチューニングが必要
になってきます。特に Oracle の場合、データ件数が多くなると
チューニングもかなり重要な要素で、その為の書籍も出てるくらい
です。もっとも、元々速い MySQL ではそこまで考えなくてもいい
かもしれませんが。チューニングしても満足できる速度が出ない
場合はクラスタリングを考えなければいけませんし、そう単純な
ものではないんです。

> 結局MySQLどころかデータベース以前の問題だったにも関わらず

そうでもないでしょう。
やはり「管理」を考えたら最初の手間はともかく、今後の事を
考えれば DB 化が普通の流れでしょう。ただ最初にじっくりテー
ブル構造とか、検討しないといけないという話です。

あと外注するにしてもどれくらいの予算を考えているのか。

しか考えてないのだと話にならないです。

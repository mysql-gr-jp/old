現在、
Red Hat Linux
Apache 2.0.46
PHP    4.3.2
mysql  3.23.58
という体制で開発を行っています。

先々週、サーバーを再起動したところ
mysqlが立ち上がらなくなってしまいました。
Can't connect to local MySQL server through socket '/var/lib/mysql/m
ysql.sock'
や
startと同時にendedしてしまうなどのエラーが出ていたのですが、
このMLの過去ログを参考にさせていただき、my.cnfの「mysqld」部に
port=3305を追加したところ、コマンドライン上からmysqldを実行する事で
SQL文の実行等、mysqlとしての動作は正常に行われるようになりました。

が、再び再起動するとまた前回と同じエラーが発生し、
my.cnfのportを3304などに変更すると、立ち上がるのです。
何故こんな事が起きてしまうのでしょうか。

また、PHPとの接続は正常に行われているようなのですが
（起動出来るまではPHPで作成した画面そのものがエラーで立ち上がらなかった)
データを参照することが全く出来なくなっているようなのです。
（テーブルが表示されない。入力を受け付けず画面がループする、等)
PHP側のソースで設定したパスワードなどを変更すると再びエラーになるので、
接続は行われていると推測しました。

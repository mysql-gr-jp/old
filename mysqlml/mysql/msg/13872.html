という事はサーバに同じ環境は用意出来るという事ですね。
まず簡単なテーブルをサーバと自分のPCに作成してテストしてみてはどうでしょうか。

１．自分のPCのデータをファイルに出力する
SELECT * INTO OUTFILE 'c:/test.txt' FROM table_a;

２．出来たファイルをサーバにアップロードする
アップロードする場所は分かりませんがMySQLからもアクセス出来る場所の必要があります。

３．ファイルをサーバのテーブルに読み込む
まず自分のPCと同じテーブルがOCNのMySQLにも作成されている必要があります。
ファイルの場所'test.txt'はアップロードした場所に合わせて変更して下さい。
LOAD DATA INFILE 'test.txt' INTO TABLE table_a;

こんな感じでいけないでしょうか？

ただ自分のPCとサーバのMySQLのバージョンが違う場合とか文字コードの問題があったりとか
他の問題が出てくるかも知れませんがそれはそれでまた別の問題なので
まず単純なテーブルでもいいので移動出来るかどうか試してみるのがいいかもしれません。

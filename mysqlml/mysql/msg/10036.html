>> 「mysql パスワード リセット」を Google に聞くと:
>>     http://dev.mysql.com/doc/mysql/ja/Resetting_permissions.html
>> こんなのが見つかります。試されました? これで root の
>> パスワードをリセットしたり、root を作り直せませんかね。

　私も何度か同じような失敗をしているので、参考にしているのは
さとうさんが指摘されたものです ^^;

> マニュアルの記述通り入力すると、
> # kill `cat /var/lib/mysql/ホスト名.pid`
> # /etc/rc.d/init.d/mysql restart --skip-grant-tables
>
> 以下のエラーが返ってきました。
> [1] 13519
> bash: --skip-grant-tables: commando not found
> # No mysql pid file found.Looked for /var/lib/mysql/ホスト名.pid.

  えっと、「ホスト名.pid」ってまさかこのまま入力されてないですよね。

　あと、"/etc/rc.d/init.d/mysql restart --skip-grant-tables"の記述
はおかしいですね。マニュアルのとおりmysqldに与えるパラメータなので
/etc/rc.d/init.d/mysqlのstartの部分(おそらくrestartはstop/startを
実行しているだけなので)で、mysqldに与えるパラメータを変更してくだ
さい。

# 終わったら元に戻してくださいね。

Subject: [mysql 12724] RHEL 3 付属の mysql-server に MySQL AB にて配付されている mysql-max をインストールする方法

中村＠東京です。

ホスティングにてサーバーを利用しております。

--- 環境 ---
Red Hat Enterprise Linux ES3
# uname -r
2.4.21-37.EL
---

開発中の WEB AP でトランザクションを利用する必要があり、InnoDBを
利用したいのですが、ホスティング会社に問い合せた所「mysql-max は
ディフォルトパッケージに含まれてないのでご自身の責任でインストール
して下さい」という事で追加インストールの方法を調査しております。

MySQL に関しては以下のパッケージがディフォルトで
インストールされております。
---
# rpm -qa | grep -i mysql
perl-DBD-MySQL-2.1021-3
mysql-devel-3.23.58-16.RHEL3.1
mod_auth_mysql-20030510-2.ent
php-mysql-4.3.2-25.ent
libdbi-dbd-mysql-0.6.5-5
mysql-server-3.23.58-16.RHEL3.1
mysql-bench-3.23.58-16.RHEL3.1
qt-MySQL-3.1.2-13.4
mysql-3.23.58-16.RHEL3.1
MySQL-python-0.9.1-6
---

MySQL AB にて配付されている mysql-max パッケージをインストール
しようと試みましたが、以下エラーの為に断念。

# rpm -ivh MySQL-Max-3.23.58-1.i386.rpm
警告: MySQL-Max-3.23.58-1.i386.rpm: V3 DSA signature: NOKEY, key ID 5072e1f5
エラー: Failed dependencies:
        MySQL = 3.23.58 is needed by MySQL-Max-3.23.58-1

現状、ソースからのインストールしかないのかな～と考えておりますが
インストール方法（RPM ＋ ソース）を解説している WEB ページ等ご存知
でしたら動作可能かどうかも含めてお知らせ頂けると幸いです。


以上、宜しくお願い致します。

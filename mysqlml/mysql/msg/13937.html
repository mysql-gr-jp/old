後藤さま、

--without-readline につきまして、

> ご指摘の通りここが一番問題ですよね。オプションを理解して選択していないので
> 雑誌等に載っているコンフィグのオプションをそのまま入力してエラーとか

$ mysql -u xxxxx -p
mysql> status

とすると、ビルドのとき使われた readline の version が見えます。

マニュアル見ますと、バイナリ版 MySQL 5 では --without-readline をあらわ
に使っているのは見当たらないです (--with-readline のものはちらほら見掛
けます)。ということで、readline 5.0 と出るかと思います。

また、日本語全文機能を組み込む、senna を入れてビルドするやりかたを説明
しているところ(http://qwik.jp/tritonn/userguide.html) で
は、--with-readline をあらわに入れてますね(うちではそれで作ったバイナリ
で問題なく走ってます)。http://qwik.jp/tritonn/userguide.html

のあたりから推測しますに、--without-readline は使わないで
--with-readline もしくはreadline について何も指定しない(そのときの効果
は --with-readline と同じと「思う」) で良いんじゃないでしょうか。

はじめまして。
hirohamaと言います。

> MySQL-5.1.19 で MySQL Cluster を導入してみました。
> 新規に create したデータベースは同期される事を確認致しましたが、デフォルトではmsqylデータベースに含まれるユーザー情報(table:user or db)は同期されないようですね。

MySQL Clusterはエンジンの一つとして機能するので、engine=ndbなテーブルで無いと同期しないのでは無いでしょうか。

mysqlデータベース内の各テーブルはMyISAMテーブルで構成されているので、そのままでは同期しないと思います。

> これらのデータを同期する事は可能でしょうか？

もし、"MySQL Cluster"で同期したいのであればalter table xxx
engine=ndb;でエンジンをひとつずつ変更することになるでしょう。

あるいはSQL Nodeが二つの場合には、マスター/マスターレプリケーションを構成してしまうのもありかも知れません。


が、mysqlデータベースは運用開始後はあまり更新しないことが多いので、私でしたら、設定完了後にコピペで同期して、後はそのまま同期しない手を選ぶと思います。

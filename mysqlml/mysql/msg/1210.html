> 海と顔がどうも判別できないようです。
> 具体的にはウェブ上からPHP・IFでselectのクエリーをつかってデータの検索をさせた
> いのですが、
> 海で検索すると、顔というkeywordも検索にひっかかってしまします。

漢字コードは Shift JIS と仮定して…。

海<8a43> 顔<8ae7> だから <43> と <e7> の区別がついていないんですね。
iso-8859 では <43> は "C" で <e7> は "c" の下に鬚がついたような文字です。
また MySQL は BINARY じゃない項目は、似たような文字を同じものとして解釈
する機能があります。つまり MySQL を iso-8859（デフォルト）で make してい
ると <43> と <e7> は区別されません。

ということで、MySQL を --with-charset=sjis で configure してから make す
ればいいと思います。

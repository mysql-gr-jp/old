Subject: [mysql 02972] check table にてエラーが頻発

はせがわです。

現在 3.23.29a-gammaを使用し、６台ほどでレプリケーションを行っていますが、

mysql> check table table1;
+---------------+-------+----------+---------------------------------------------------------------+
| Table         | Op    | Msg_type | Msg_text                                                      |
+---------------+-------+----------+---------------------------------------------------------------+
| table1        | check | error    | Size of datafile is: 1509469817         Should be: 1509470416 |
| db1.table1    | check | error    | Corrupt                                                       |
+---------------+-------+----------+---------------------------------------------------------------+

といったようなエラーが頻発します。
更新頻度の非常に高いテーブルなのですが、それが影響しているのでしょうか。
クエリーの結果が返る場合、エラーが即刻返ります。

このエラー、もう一度同じクエリーを送ると、正常に終わるのです。
しかも今度はきちんと処理に時間がかかります。

lock table をはさんだりして試しましたが、結果は同じでした。

エラーの意味もつかみかねているのですが、システムが把握しているファイルサ
イズと実際のファイルサイズが異なるといったことでしょうか。
たまに、2回目のクエリーでcrashedになり、さらに3回目のcheck tableで
正常に戻ったりもします。

lock して flush table してなんてことをしないとだめなんでしょうか。

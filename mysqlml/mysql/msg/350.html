> # 実験で CVS に commit したりして、今 revision が 1.3 になってます。
> # 1.1.3 とかにしてくれればいいのに。 どうするんだろ?

CVS(というかRCS)の場合はあまりそういうことにこだわらない方がいいと
思います。

	1.1 → 1.2 → 1.3 → 1.4

という revision がある状態で、1.3 を変更すると、

	1.1 → 1.2 → 1.3 → 1.4
	               └→ 1.3.1.1 → 1.3.1.2

となります。つまり revision 番号は2つか4つにしかできなかったと思い
ます(4つの場合は枝分かれしたことを示す)。

CVS が勝手につける revision 番号とは別にバージョンをつけたい場合は、
私は cvs tag コマンドでつけてます。例えば、現在の最新の revision に 
ver1-1-3 という名前をつけるには、cvs tag ver1-1-3 として。

複数の異なる revision のファイルに一括して同じタグをつけられるので、
便利です。ただし、タグは数字で始まったり、'.' を使ったりできません。

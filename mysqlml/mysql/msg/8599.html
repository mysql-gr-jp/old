いつもお世話になっております。濱中と申します。
前任者から引継ぎ、MYSQLを使用し始めて半年ほどになります。

現在、mysql3.23.58＋php.4.3.1を使用してシステムを作っています。
この間、Mysqlのバージョンを3.23.56からバージョンアップしたのですが、
バージョンアップしてから、PHPのコマンドライン版でスクリプトを実行
すると以下のエラーが出るようになりました。
--------------------------
File 'NONEXISTENT/charsets/?.conf' not found (Errcode: 2)
Character set '#12' is not a compiled character set and is not specified in
the 'NONEXISTENT/charsets/Index' file
--------------------------
PHPのスクリプト自体はうまく動作するのですが、エラー表示が
気持ち悪いので、何とかしたいと考えております。
MLの過去ログを見たところ、以下のような解決法がのっていました。
(本メール末尾に転載)

自分の環境を調べたところ、main/build-defs.hの中には、
#define PHP_MYSQL_LIBS     ""
となっており、モエルさんが書かれた内容とは、少し異なっていました。

この状況のもとでも、モエルさんが書かれた、
#define PHP_MYSQL_LIBS          "-L/usr/lib/mysql -lmysqlclient "
と書き直して、makeしてしまっても問題ないのでしょうか？

今、調べてみたところ、/usr/lib/mysqlというフォルダはないようで、
書き換えようにもどこのフォルダを指定してよいのやらわかりません。

どなたかご教授いただけると助かります。
PHPの再Configureなど、必要になってくるのでしょうか？
よろしくお願い致します。

--------------------------

モエルです。

河田さんにいろいろメールでおつき合いいただいて

php-4.0.6以降のＣＧＩ版で以下のエラーが出る件
解決できました。

>File 'NONEXISTENT/charsets/?.conf' not found (Errcode: 2)
>Character set '#13' is not a compiled character set and is
>not specified in the 'NONEXISTENT/charsets/Index' file


>php directoryの中でconfigure後、main/build-defs.hを見ると
> > #define PHP_MYSQL_LIBS          "-L/usr/lib -lmysqlclient "

なってるので

#define PHP_MYSQL_LIBS          "-L/usr/lib -lmysqlclient "
を
#define PHP_MYSQL_LIBS          "-L/usr/lib/mysql -lmysqlclient "

に書き換えてから
make
すればＯＫとの情報をいただき見事解決でした(^^)

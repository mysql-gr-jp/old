> 自動変換するのはいけないことではないと思います。

sjis<->cp932 変換、したくない者としたい人と。
じゃ、両方実現する。
こういうのはどうでしょう。

mysql> SET SESSION convert_string=0;

これで自動変換停止。

mysql> SET SESSION convert_string=1;

これで自動変換動作。

SET GLOBAL は実装しない。(悲惨なことになりそうだから)
そのかわり、サーバー管理者だけが使用できる

mysqld --skip-convert-string

を実装。

もし全体の自動変換を停止するのがだめなら、
sjis<->cp932, ujis<->eucjpms の変換のみを止めるオプション

mysql> SET SESSION convert_nihongo=0;

mysqld --skip-convert-nihongo

を実装。(convert_nihongo は命名がいまいち)


現場によって状況は多種多様なので、
それに対して解決策を多く用意しておいた方がいいでしょう。

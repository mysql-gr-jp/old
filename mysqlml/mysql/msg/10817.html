2005年1月1日：開発チームは新しい年に多分ここ
数年でいちばん大きなスケールで、
新しいツール、技術そしてプラットフォームを得るでしょう。

2004年の最後の数週間の間にBEAとオーラクルはプラットフォーム
のアップデートをアナウンスしました。それぞれWebLogic 9
とOracle 10g リリース 2で、どちらもビジネス・プロセス管理
でSOAを実装することに焦点が置かれています。両社とも
この方向に力をいれています。昨年10月に発表された
IBMのWebSphere 6とSunとJBossからの新しくなったJ2EEアップ・サーバ
など、Javaコミュニティはあたらしいセットから選択できます。

Java ツールのグループはIBMの2004年末に発表されたRationalツール
を含むEclipse 3.0を中心に展開します。オープン・ソースツールの
プラットフォームはまだ勝ったわけではありません。Oracleの
JDeveloper 10g リリース 2, BEAのWebLogic Workshop, Sunの
Java StudioとSun Studio,そしてBorlandのJBuilderは
Eclipseコミュニティの外にあります。

Microsoftのコミュニティは2005年の2つの大きなリリースを
待ち望んでいます。それはリリースが遅れているSQL Server 2005と
Visual Studio チーム・システムです。新しいデータベースで
Microsoft　の.NET プラットフォームは完璧になります。アップデート
されたツールで開発者はコラボレーションとテストとQAの機能を
期待できます。どちらのリリースも.NET側の大きな期待を膨らませる
ものです。

OSではLonghornが2006年まで延期されたため、Solaris 10 が一番期待されて
います。AppleのMac OS X 10.4 「タイガー」は2005年の前期に
リリースされます。これには修正されたXcode IDEを含みます。

オープンソースの世界では2005年にはLinuxのコミュニティにとっては
大きなOSの変更はありません。しかし、動きとしては利益を上げる
開発プロジェクトに向かっています。例えば、JBoss、MySQL、
Berkeley DBなどです。また他の動きはLinuxカーネルや
Apache Webサーバーよりももっと注目を集めています。

オープンソースの動きには何時も２つの大きなグループがありました。
最初の勝者は「言論の自由」で言われるフリーの
グループで、オープンソースをAT&TやMicrosoftのような大きな
会社やプロプリエタリーのシステムに対するものとみなしました。
他のグループは違った立場をとり、「White Box」ソフトウエアの
恩恵に重きをおきます。ビジネスモデルに関わらず、オープンソース
の恩恵はコードがリビューされ、テストされ、拡張される
ことです。2005年には利益が上がるNetBeansや Eclipseプロジェクト
が前進すると思います。

こんばんは、広津です。

> あるテーブルに同時に二つのPCからアクセスする可能性がある場合
> ロックが必要なのはどういう時でしょうか？

通常、SQL 文が１つの場合は、INSERT だろうが、UPDATE だろうが、
DELETE だろうが、ロックの必要はありません。

ロックが必要なのは、同時に処理すべき SQL 文が複数の場合です。
（それぞれの SQL 文の間に、他の SQL 文が入るとまずい場合）
このとき、一連の SQL 文の前後を、ロックとアンロックで挟みます。

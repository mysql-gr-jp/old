Subject: [mysql 12973] ApolloでMySQLに接続するには

「Rubyを256倍使うための本 黄道騙」、「標準MySQL」を読んで、ApolloからMySQLを使え
ないかと思い立ち格闘しておりますが、大海を漂流する小船のような状況でして、こちら
で質問させて下さい。
apollo's Wikiを調べていたら、データアクセスソフトを介して物理的データベースに接
続するものだということがわかりましたが何が良いかわからないので、とりあえず
dbExpressを使ってみようと思い、
  http://wiki.fdiary.net/apollo/?RDB%3A%3ASQLConnection
  http://wiki.fdiary.net/apollo/?InterBase
を参考にrubyスクリプト(mysql_connect.rb)を書いて、

  c:\ruby_ap mysql_connect.rb
とすると、
  「'table_names':LIBMYSQL.dllを起動できませんでした。(EDatabaseError)」

というエラーになります。
LibMySQL.dllは、c:\program files\mysql\mysql server 4.1\binの中にありパスが通っ
ていますし、c:\winnt\system32の下にコピーしても同じ結果でした。

  Windows2000 5.00.2195 Service Pack4
  MySQL server version: 4.1.12a-nt
  Apolloは、ap-841a.msiをインストールしました。
  libmySQL.dll 2005/05/25 11:20

ちなみに、ruby-mysql-0.2.6のAPIを使って、MySQLのテーブルを作ることはできていま
す。これで作ったテーブルをApolloから操作したいと考えています。
お気付きの点を教えていただければと思います。また、そもそもこのような形態に
dbExpressが有用なのかどうかご意見をお聞かせいただければ助かります。

こんにちは、グレイペンギンです。
uedaさん、すいません。include_pathは、特に関係ないか見たいです。
irieさんがいわれていた通りのようです。php.ini の　include_pathをコメント
にしても問題なく動作しました。
uedaさんの環境がよくわからないので、同じで認識するのかわからないですが、
私の環境を参考までに書いておきます。

OS：                         windowsXP pro
Apache:                 2.0.49
PHP:                    　5.0.0
MySQL:                  4.0.20a-nt

php.ini                    C:\WINDOWS\php.ini
php本体:                C:\php5

php.iniでの変更点
include_path = ".;C:\php5"
extension_dir = "C:\php5\ext\"
 (階層は、前回 / にしていましたが、\ でも認識するみたいです。ただ、extの後ろ
にも、 / もしくは、 \ がいります。）

extension=php_mysql.dll    を　コメントアウト( ; を削除　別に必要なものがあ
れば、随時コメントアウト)
ちなみに、このextension_dir自体をコメント化すると、
C:\php5\php_mysql.dll がありません。とでませんか？たぶんここが　
extension_dirのデフォルトです。

私の場合、doc_rootは、記述していません。

また、apacheの設定ファイルhttp.conf には、
LoadModule php5_module module/php5apache2.dll   を　LoadModuleの最後に追記
apacheのmoduleディレクトリーに　php本体にある、php5apache2.dll をコピー
AddType application/x-httpd-php .php　を　774行目当り（私のconfファイル）に
追記

前回同様、libmysql.dll は、C:\WINDOWS\SYSTEM32にコピー

以上で、動作しています。phpmyadminにて、確認済み

あくまで、私の環境なので、なんともいえませんが、php.ini が、もし、別の場所に
もあるなら、削除はするべきでしょう。
以上、参考になれば幸いです。

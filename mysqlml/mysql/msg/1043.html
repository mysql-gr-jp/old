丸井さんとのやりとりで、やはり、non-free と、free (売ってもよい)
を分けてみようと思いました。
Debian は既に分かれているので、今回は RPM の SPEC ファイルを
ちょっと変えてみました。

ftp://ftp.softagency.co.jp/mysql/japan/rpm/test/free/
ftp://ftp.softagency.co.jp/mysql/japan/rpm/test/non-free/

free/ の方のパッチと、.spec ファイルで、
商用配布 制限なしの クライアントパッケージとソースができ上がります。
使用しているソースは
http://www.debian.org/debian/dists/potato/main/source/misc/mysql-gpl_3.22.25.orig.tar.gz
で、これと、free/*.patch を /usr/src/redhat/SOURCES/ 以下において
build すれば free 版ができあがります。
これらは、例え Vine の製品版に含まれていようが、ライセンスは発生しないこ
とになります

non-free/ の方はサーバーとnon-free の作成 spec で、こちらは
通常のソース mysql-3.22.25.tar.gz を使用しています。
今使用されている .spec のサーバー以外を削っただけ

ちょっと rpm のパッケージングが可能な方で
お暇であればテストをお願いしたいです。
OK ならば、TCX にでも上げてみようかとおもいます。
# 却下されるかもしれませんが、
# そうなったらそうなったで、日本国内だけでも配布します

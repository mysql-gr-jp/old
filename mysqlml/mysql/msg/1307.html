Subject: [mysql 01307] MySQL+Apache+PHP3 の環境作り

はじめまして。澤田と申します。
MySQL+Apache+PHP3でDBと連動したWebサイト
をつくるための環境作りに挑戦しています。、
MySQLをインストールした時点でつまずいてしまいました。

（環境）
サーバ：
RedHatLinux5.2 kernel2.0.36
Apache1.3.3

クライアント：
PowerBuilder6.5(win95)
myodbc-2.50.24-win95(ODBCドライバ)

まずダウンロードしたRPMファイル

MySQL-3.22.25.ujis-1.i386.rpm
MySQL-client-3.22.25.ujis-1.i386.rpm
MySQL-devel-3.22.25.ujis-1.i386.rpm

をサーバにインストールした後、root権限で
#mysql とすると
ERROR 1045: Access denied for user: 'root@localhost' (Using password: NO)
とでました。これは、
#mysql -h ホスト名で
Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 5 to server version: 3.22.25

Type 'help' for help.
を得ました。

しかし一般ユーザ権限(sawada)で
$ mysql
Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 48 to server version: 3.22.25

Type 'help' for help.
と容易にホスト名の引数を渡さなくても、接続されます。
これはどういうことなのでしょうか？

その後
http://www.softagency.co.jp/mysql/TODO/begin.html
を参考にtestデータベース内にt1、t2テーブルを作成し
$ mysql test

mysql> show tables;
+----------------+
| Tables in test |
+----------------+
| t1             |
| t2             |
+----------------+
2 rows in set (0.00 sec)

テーブルの中身は以下のように確認をし、

mysql> select * from t1;
+------+
| a    |
+------+
| aaa  |
| bbb  |
| ccc  |
+------+
3 rows in set (0.00 sec)

このテーブルをODBC経由で、アプリケーション側（PowerBuilder）のGUI
を用いて操作しようと、ODBCの設定をおこない接続を試みると、

SQLSTATE = S1000
[TCX][MyODBC]Host " is not allowed to connect to this MySQL server

と返ってきます。ODBCの設定は
root (パスワードなし)
sawada(********パスワードあり)
Port 3306(あり、なし)

の組み合わせをすべて行ないましたがだめでした。

このMySQLが使うPortはサーバの/etc/services
に記述する必要があるのでしょうか？


また現在RPMからインストールしたApache1.3.3が
動いているのですが、PHPを動かすためには、
ソースからコンパイルして設定し直すしかないのでしょうか？
デフォルトでインストールした状態のhttpd.confは
設定部分の関連項目と思われる部分、

#LoadModule php_module         modules/mod_php.so
#LoadModule php3_module        modules/libphp3.so

#AddModule mod_php.c
#AddModule mod_php3.c

がコメントアウトされています。
以上長々と一度にたくさん質問しますがよろしくお願い致します。

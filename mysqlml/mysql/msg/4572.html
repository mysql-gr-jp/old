# 一度送ったのですが飛んでこなかったんで再送します。

最近になってやっと Linux(RedHat6.2) + MySQL でもってシス
テムを作り安定に稼働しております。
このシステムはサーバー・クライアント型のシステムなのです
が（クライアントはWindows2000Client）デバッグするために
ノートにMySQLのWindows版を入れてテストを行いたいのですが
これが「データベース接続時にエラーが発生しました」と言っ
て接続してくれません。

「環境」
・MySQL  ... MySQL For Win 3.23.37
・Client ... MyODBC Ver. 2.25.29
・IPアドレス ... 192.168.0.10

「現象」

・ネットワークに接続しているときならODBCの設定「MySQL Host」
　に192.168.0.10を設定すれば接続できます。

・ネットワークに接続してないときにODBCの設定「MySQL Host」に
　192.168.0.10を設定しても接続できません（当たり前ですが）

・ネットワークに接続してないときにODBCの設定「MySQL Host」に
　127.0.0.1を設定しても接続できません（これが不思議）

# エラメッセージは「データベース接続時にエラーが発生しました」

・ネットワークに接続しているときにODBCの設定「MySQL Host」に
　127.0.0.1を設定しても接続できません（これが不思議）

# エラメッセージは「データベース接続時にエラーが発生しました」

# とにかく127.0.0.1またはlocalhostを設定すると接続には失敗し
# てしまうのです。

しかし、mysql.exe を起動してselect * from xxxx などは問題なく
動いてくれます。

何か設定が悪いのでしょうか？
ご存じの方が有ればよろしくお願いいたします。

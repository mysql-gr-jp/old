アクセスが相当数集中するまでは私も問題なく動いてました。それまでは
テーブル定義も変えていませんでした。
今でも、更新の集中していないものは、全く問題ありません。
エラーが出ていた時も、行の追加などは出来るんです・・

ヒントをいただいたので、プライマリキーの設定が必須ということから、
マニュアルを一生懸命拾ってみたら、以下の項目があったので、なるほどと
思い今回実施してみたのでした。

You should have a primary key in the table. 
You should have a timestamp in all tables you want to be able to update. 

更新が起こった場合にも、今まででなかった適切な再読み込みを促す
メッセージがでるようになりました。

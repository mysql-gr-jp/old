Subject: [mysql 10720] 4.0.22→4.1.7 mysqldumpが文字化け

いつもお世話になります。

4.0.22から4.1.7へmysqldumpでデータを持ってくると
データが？？？のように文字化けしてしまいます。

現在2つの異なるFreeBSDサーバでmysqlを走らせていて
片方は4.0.22・・もう片方は4.1.7です。
両サーバともujisです。

4.1.7のサーバの方から
mysqldump -h 4.0.22server.com -u root -p パスワード 
--all-databases > dump.sql としてデータを持ってきて、
mysql -u root -p < dump.spl
としてデータを吸い込んでいるのですが・・文字化けが起こります。

/etc/my.cnfは両サーバとも
[mysqld]
language = Japanese
default-character-set = ujis
と設定されています。

もし同じ問題に遭遇した方がいらっしゃればアドバイス頂けますと幸い
です。よろしくお願いいたします。

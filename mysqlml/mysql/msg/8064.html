mysql-4.0.13 on FreeBSD環境で、日夜悪戦苦闘しております。
簡単なカウントを行うためのテーブルを作って、テストも終わり、
delete from table_nameで、データを削除して、本番と思ったの
ですが、新しくデータをinsertすると、auto_incrementの数値が、
前のdeleteしたはずの値+1からスタートしてしまいます。

このMLのアーカイブから
myisamchk -A table_name
というのを見つけて実行してみたのですが、やはりだめです。

結局、dropして、新たにcreate tableするしかなかったのですが、
今後の為に、他に方法があれば、ご教授お願いします。

show create table table_nameで書き出した、このテーブルのsql文は、
以下のとおりです。

CREATE TABLE `count_mm` (
   `number` int(6) NOT NULL auto_increment,
   `mod_time` timestamp(14) NOT NULL,
   `name_l` varchar(20) default NULL,
   `name_f` varchar(20) default NULL,
   `email` varchar(60) default NULL,
   PRIMARY KEY  (`number`)
) TYPE=MyISAM

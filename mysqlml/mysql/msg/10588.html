MySQLユーザコンファレンス
カリフォルニア州シリコンバレー
April 18-21, 2005年4月18-21日
MySQLの2005年ユーザコンファレンスは最も大きいMySQLの開発者とユーザの
集まりです。個人か企業で発表をするかチュートリアルを行い方は
プロポーザルを提供してください。今回のテーマは「MySQLはどこにでもある」
です。提案は他の参加者に役に立つものでなければなりません。

MySQLの力を引き出しましょう。ベストプラクティスも含みます。
MySQL 5.0の新しい機能を学んで使ってみましょう。
様々な環境(ウエブ、データウエアハウス、クラスタ、組み込みなど)
でMySQLを使ったエキスパートから学びましょう。
プロプリエタリのデータベースからMySQL にマイグレートしましょう。
参加する5つの重大な理由（http://www.mysqluc.com/pub/w/35/why.html）


事例: T-Systems はMySQL に頼っています。
T-Systems Integration はMySQLを使用してウエブフロントとバックエンド
の注文プロセスのアプリをWestfalia Wekzeugco. GmbH社のために
構築しました。この会社はツール、エレクトロニクス、家庭のこまごましたもの
などを販売しています。その上T-Systems はカスタマー管理システムの裏側
にあるデータベースをOracleからMySQLに変更しました。
www.westfalia.deのウエブサイトは11個の MySQL サーバーを使用しています。
これらのサーバーは4つのコンピュータに搭載され毎月役40万の
人にアクセスされています。

事例のダウンロード
http://www.mysql.com/it-resources/case-studies/mysql-tsystems-casestudy.pdf

事例: 郡政府はMySQLを使って市民にサービスします。
アイオワ州のPottawattamie郡はLAMPソフトスタック（Linux, Apache, MySQL, 
Perl/PHP）
を使用して郡の政府のアプリケーションの信頼性と機能アップを行っています。
全ての新しいデーターベースの展開にMySQLを使用することにしました。
これにより古いInfomixやMS SQLサーバーのアプリは全部取り替えられました。

事例のダウンロード
http://www.mysql.com/it-resources/case-studies/mysql-pcit-casestudy.pdf

ヨーロッパのオープンソースの指導者たちはEUの政府にソフトの特許
を求めないようにアピール　（ドイツ　ミュンヘン）

ヨーロッパで一番有名な３人のオープンソースの開発者がNoSoftwarePatents.com
上でソフトの特許に反対のアピールを載せました。Linus Torvalds (Linux),
Michael "Monty" Widenius (MySQL)とRasmus Lerdorf (PHP)はEUの政府に
対して（今週に会議を開きますが）ソフトの特許に関するドラフトを採択
しないようにアピールしました。彼らによるとこれは「虚偽、危険、民主的に
非合法」だということです。彼らは更に、多くのウエブサイトに
NoSoftwarePatents.comへのリンクとバナーを張ることをアピールしています。
http://www.nosoftwarepatents.com/

http://www.mysql.com/news-and-events/press-release/release_2004_33.html

製品
MySQL (4.1.7 プロダクション)の新しいリリース
これは4.1プロダクト版の最初のリリースで主に最近発見された
バグの修理版です。このリリースには

InnoDB: スタートアップのオプションとセットできるシステム変数を追加
LOCK TABLEがInnoDBのロックも入手できるようにinnodb_table_locks
数多くのバグ修理

詳細は
http://lists.mysql.com/announce/226

ソースとバイナリのダウンロードは
http://dev.mysql.com/downloads/mysql/4.1.html

MySQLクエリブラウザー(1.1.0 ガンマ)とMySQL Administrator 1.0.14 のリリース
これはグラフィカルなクエリ・シェルとSQLのスクリプト・エディター
であるQuery Browserのガンマリリースです。このリリースには多くの
改善が含まれています。一番目につくのはMySQL5.0版の機能である
Stored Procedure, Functions とViewです。

複数の結果セットを返還するstored procedureへのサポート
Stored Procedures, Functions and Views を形成してエディットするサポート
多くの UI の改善

LinuxとWindows 版のダウンロードは
http://dev.mysql.com/downloads/query-browser/

MaxDB (7.5.00.19)の新しいリリース
このリリースには

チュートリアルのデータをロードする新しいDBM Server コマンドload_tutorial
ストラティジー・データ用のメモリーを割り当てる新しいパラメータの 
_MB_STRAT_SIZE
クエリの性能アップ
Database Analyzer への数個の改善
MaxDB Loaderが使用するIsolation レベルが可変に

新しい版のダウンロードは
http://dev.mysql.com/downloads/maxdb/7.5.00.html

新しいクライアントのパッケージは
http://dev.mysql.com/downloads/maxdb/clients.html

変更のリストは
http://dev.mysql.com/doc/maxdb/changes/changes_7.5.00.19.html

Connector/Net (1.0.2ガンマ) のリリース
これはMySQL-4.1とMySQL-5.0を含む全てに適用される最初の
ガンマリリースです。このリリースには:

TableEditor CSとVB のサンプルを追加
stored procedureを呼ぶと「Illegal mix of collations」が
生じる問題を解決
MySqlParameter(string parameterName, object value) での
ダブルのタイプを修理　

ソースとバイナリのダウンロードは
http://dev.mysql.com/downloads/connector/net/1.0.html

変更のリストは
http://lists.mysql.com/announce/235

Connector/MXJ (1.0.7 ガンマ)のリリース
MySQL Connector/MXJ はJMX MBean でMySQLデータベースを展開
したり管理するのに使います。これがあればJavaの開発者がエンドユーザ
のためにインストレーションの問題を軽減してデータベースを
必要とするアプリの展開を容易にします。

ソースとバイナリは
http://dev.mysql.com/downloads/connector/mxj/1.0.html

Connector/J (3.0.16)の新しいリリース
3.0.16版はMySQL-4.1と MySQL-5.0を含むどの版にも適用できる
ステーブルなツリーのバグ修正版です。

ソースとバイナリは
http://dev.mysql.com/downloads/connector/j/3.0.html

変更のログは
http://lists.mysql.com/announce/239

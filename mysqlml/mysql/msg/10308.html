> com.mysql.jdbc.CharsetMapping.javaの
> tempMap.put("sjis", "SJIS");
> という行を、
> tempMap.put("sjis","MS932");

早速ソースコードを入手し、ビルドし直しました。
調査の結果下記のことがわかりました。

ビルドし直し版のドライバで行う場合

①DB登録
String url = "jdbc:mysql:///test?useUnicode=true&characterEncoding=MS932";
にすると㈱㈲の文字化けは回避できる。

②DB取得→jsp表示
<%@ page contentType="text/html; charset=MS932" %>
を記述しておけば

String url = "jdbc:mysql:///test?useUnicode=true&characterEncoding=sjis";
でも
String url = "jdbc:mysql:///test?useUnicode=true&characterEncoding=MS932";
でも文字化けせず表示可能。

上記のように①と②の条件を満たせば
回避できることができました。

ただし、オラクルなどDBの移植性とかも考慮すると
PG中はsjisで統一して処理したいと思うのですが、
その場合はやはりmysqlは4.0にするのが
ベターなのでしょうか?

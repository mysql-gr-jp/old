On Wed, 29 Nov 2000 13:51:14 +0900
Takahiro Iwatani <iwatani@xxxxxxxxxx> wrote:

> 　ところで、nobodyの場合は、パスワードは必要なのでしょうか？

パスワードはあってもなくても接続できます。
ただしnobody@localhostというユーザーのアクセスが許可されて無ければなりません。

もしパスワードがある場合は、MySQLサーバーへ接続するときにパスワードを
指定する必要があります。
私の場合はネットワーク経由でのアクセスを禁止しているので、パスワードなしに
しています。

nobody@localユーザーの設定方法についてですが、
とみたさんも書かれていますが、とりあえずmysqlコマンドでインタラクティブに
SQLを実行できますので、"mysql"データベース中の"user"テーブルを見てみたら
なんとなく雰囲気が分かると思います。

テーブル一覧は
mysql> show tables;
+-----------------+
| Tables_in_mysql |
+-----------------+
| columns_priv    |
| db              |
| func            |
| host            |
| tables_priv     |
| user            |
+-----------------+
6 rows in set (0.01 sec)

で見れます。上記の６つのテーブルでいろんな階層でアクセス制限を設けれます。
結構柔軟なアクセス制限ができるみたいです。

userアクセス許可テーブルは
mysql> select * from user;

で見ることができます。出力結果は１行が長いためメールには書きませんでしたが、
是非とも一度ご覧になられると、雰囲気が分かると思います。
それからドキュメントを読まれると入りやすいと思います。

＃サーブレットのことは全く存じませんので、はずしてるかもしれません

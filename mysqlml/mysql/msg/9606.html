Windows と Linux の大きな違い、というか利点に挙げられるのが
ココです。Windows では 3.1 以前は別として GUI を外そうにも
外せません。しかし Linux は X Window を落としてしまえば、その
リソースは DB に廻せます。これは大きな利点かと思います
それをしないで、というのはチト何でも酷いかと。

GUI は LAN 接続した Windows マシンで遠隔操作すればいいんじゃ
ないのでしょうか。何も MySQL のマシンで GUI を直接使用する
必要は無いかと思います

CUI については telnel で入ればデスクトップコンソールと同じ
操作を行えますし．．．

> 弱小企業では、充分に耐えるDBだと、シロウト的には思います。

私も小さいところでは Access で入れてますよ

ただ Access は所詮 Personal DB である故か、データロックが完全
ではありません。これがよく言われる MDB の破壊です
データ一件のみをロックできればいいのですが、何故かブロック単位
でのロックをしているようです

従って Access を単に MDB としてみてクライアントソフトウェア
を作る時には、その回避のコーディングをしないと完全なものに
なりません

まぁ RDBMS でもそれを当然考慮して作るべきですが、SQL を出せば
済む話なので、こっちのほうが安全で楽なように思います

> 当然保守は必要ですが。

ですが OS の方が保守が終わってしまったらどうするのでしょう

＃まぁその時は「新しくしないといけない」と受注するネタになる
＃かもしれないですけど

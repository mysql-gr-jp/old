On Fri, 06 Apr 2001 10:19:42 +0900
Akihiko Shinohara <sino@xxxxxxxxxx> wrote:

> この前、気が付いたのですが、CHAR,VARCHAR も
> 結局同じな感じです。
> 
> | mysql> create table a ( c char(128),vc varchar(128) );
> | Query OK, 0 rows affected (0.03 sec)
> |
> |
> | mysql> desc a;
> | +-------+--------------+------+-----+---------+-------+
> | | Field | Type         | Null | Key | Default | Extra |
> | +-------+--------------+------+-----+---------+-------+
> | | c     | varchar(128) | YES  |     | NULL    |       |
> | | vc    | varchar(128) | YES  |     | NULL    |       |
> | +-------+--------------+------+-----+---------+-------+
> |2 rows in set (0.00 sec)

同一のテーブルに CHAR と VARCHAR が混在すると、VARCHAR に統一されます。
# マニュアル 7.7.1 参照

１個でも VARCHAR があると、結局レコード長が固定じゃなくなるので、
CHAR にしておく意味がないんでしょう。きっと。

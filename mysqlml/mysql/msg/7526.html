原因が分からない…じゃなくて、MyISAM テーブルはそういうものです :-)

マニュアル「6.5.3 CREATE TABLE」より…

  削除したレコードが、AUTO_INCREMENTフィールドの最大値を含んでいた場合、
  その値は、ISAM と BDB テーブルでは、次に再利用されますが、 MyISAM と 
  InnoDB テーブルでは再利用されません。また、テーブル内の全てのレコー
  ドを DELETE FROM table_name (WHERE節無しで) を AUTOCOMMIT モードで使
  用して削除した場合は、すべてのテーブル型で始めから採番し直します。

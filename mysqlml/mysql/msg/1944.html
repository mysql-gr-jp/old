藤沼＠イイガです。

レンタルサーバーが MySQL ベースのサービスを提供している関係で、
MySQL には注目しています。PostgreSQL ユーザー会会員になってしまって
いる関係上、MyNA は注目しつつも、両方のサイドから裏切り者と
いわれそうで入会を躊躇しています。(^^;

moeru wrote:
> よって、よくある、「だれでも簡単ＣＧＩ」みたいな本ができると
> そうとう売れるんじゃないでしょうか。
> それに、MySQLやこのユーザ会の存在も知名度を上げると思います。

MySQL のインターフェースを直接たたくような例レベル処理を「誰にでも、
簡単に使える」ように説明するのは結構大変ですね。(^^;
そういう要望は常にあるのですが（会社の中でも）、言うはた易し、
行うは難しの典型例となってしまっています。

PostgreSQL のシーラカンス本のように、各言語用の便利なクラスライブラリを
提供して、初心者はそっちを使ってください、という方が便利かも
しれませんね。

今までの経験から、初心者は以下の点でつまずくことが多いです。
・SQL とは何か。どう書くのか。どうやって抽出するのか。
・Fetch とは何か。なぜあのように複雑な手続きが必要なのか。
　（Connect -> Parse -> Exec -> Fetch -> GetColumnValue という
　　流れを把握するのが結構大変）
・その値をどう HTML の中に埋め込むか

この辺を解決するには、
・SQL を隠蔽する。検索条件となるキーワードを与えるだけでよい。
・問い合わせや結果セットといった概念に触れずに、直接 HTML として
　出力する。
ことが必要だと思います。PHP だとこの辺を自動的にやってくれる
PHPLib（広川さんとおおたさんと共同でマニュアルを翻訳しました。
http://www.php.gr.jp/php/phplib/ をご覧下さい）がありますが、
他の言語についても一通り用意できると便利だと思います。また、
初心者でも詳細な点に触れることなく、MySQL のパワーの恩恵を受ける
ことができるようになるのではないかと思います。

で、MySQL をもっと細かく制御したい人は、オライリー本へ移行する形に
なれば、それぞれの本の個性が際立って、仲良く共存できるようになると
思います。

> しかし、出版時期：2000年度第2四半期後半（9月ころ）ですか(;o;)
> 今すぐ欲しいんです、そういう本(;o;)

オープンソース方式で、MyNA の Documentation Workgroup みたいなグループを
立ち上げて、そこでみんなでレビューしながらフィードバックする形式にする
と、
今すぐ読みたい人には便利かもしれませんね。で、読んだ人は、先立って情報を
入手した見返りとして、必ず実行結果や不満・要望を述べると。

自分自身は最初に述べた理由から、MyNA に入っていませんが、もし可能であれ
ば
何らかの形でお手伝いしたいです。PHPLib のチュートリアルとかなら
できるかな（広川さんと共同で？）。(^^

はじめまして。

日頃はLAMP環境下で社内システムを開発している植村と申します。
今回、Zend Core for oracleを社内システムに導入することになり、
同一サーバにMySQLと共存させることになりました。
Zend Core for oracleとMySQLを共存させるにあたっての質問です。

共存サーバ環境は下記の通りです。

Zend Core Version 1 (Zend Core for oracle)
PHP Version 5.0.5
Zend Engine Version 2.0.5
Apache Version 2.0.52
OS CentOS 4.6

上記の環境で、MySQL5をインストールして使用したいと考えています。

実施したインストール手順は下記のとおりです。

１．CentOS4.6をGUIインストーラを使用してインストールしました。
このとき、選択したパッケージ群は開発ツールのみです。

２．Zend Core for oracleのパッケージをZend社のサイトにあったインストール
手順書に
沿ってインストールしました。
インストール後、WEB管理画面 (Zend Core for oracleと一緒にインストールさ
れたGUIツール）から
上記環境を確認しました。

３．次にMySQLを、yumでインストールしました。
インストールしたMySQLのVersionは、5.0.54 です。

４．その後、WEB管理画面から「設定」−「エクステンション」の画面でMySQLを
確認すると、
エクステンションが無効な状態で未ロード状態でしたので、エクステンションを
有効にし、
「設定を保存」−「サーバを再起動」を実施しました。

ここまでが一連の作業内容なのですが、サーバ再起動後、先ほどのエクステン
ション確認画面で
確認すると、MySQLの状態が「エクステンションが無効な状態で未ロード状態」
に戻っていました。
その後、何度やっても同じ結果にしかなりません。

ほかに何かをインストールしたりしないといけないのでしょうか？

Zend Core for oracleのオンラインヘルプ等も確認しましたが、デフォルトでは
MySQLは無効になっており
configureオプションで--with-mysqlを付けないとダメというような記述があっ
たのですが、Zend Core for oracleの
ようなパッケージ(？)でconfigureオプションをつけてリコンパイル(？)する方
法が分かりません。


今まではLAMPの環境でしか構築経験がないため、Zend環境は初心者です。

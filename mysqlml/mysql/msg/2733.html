下記のようなテーブルを作成し、MAX_ROWSのテストを行ったのですが、
うまくいきません。
１０件INSERTされた時点で、Table FULLになる事を期待していたので
すが、実際は９６件まで登録されてしまいます。
何か考え違いでもしているのでしょうか？
ちなみにAVG_ROW_LENGTHの値はレコードを一件INSERTした時点で増えた、
ファイルのサイズを指定し、INSERTされるレコード長も型自体には可変
長のものが含まれていますが、全て同じレコードサイズになるようにし
ています。

create table test (
  ida char(15) not null,
  idb char(15),
  idc char(15) not null,
  idd char(15) not null,
  title text not null,
  text text not null,
  regist_date timestamp(14) not null,
  primary key (ida),
  index B_KEY(idb),
  index C_KEY(idc),
  index D_KEY(idd)
)type=myisam raid_type=1 avg_row_length=676 max_rows=10 raid_chunks=4 raid_chunksize=256;

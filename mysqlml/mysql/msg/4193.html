さてさて、これから MySQL + Web のページを作ろうと思っていま
すが、お勧めの組み合わせは以下のうちどれでしょうか？

・MySQL + sh ^^;
・MySQL + csh ^^;
・MySQL + PHP3
・MySQL + PHP4
・MySQL + Perl
・MySQL + Ruby

出来れば、
・保守性が良く、
・デザインとプログラマブルな部分を切り離せて、
・プログラミングしやすいこと
を重視したいと思います。

私は PHP も Perl も Ruby も超初心者です。^^;
# 友達に「ごとけん」がいますが、あいつは Ruby を懇切丁寧
# に教えてくれる玉ではありません（笑）
# 私が賢くなさ過ぎるっちゅう話もあります^^;

Web+DB PRESS Vol.3 には「HTML デザインと PHP プログラム分離
の試み」な～んていうタイムリーな記事もあります。

現在のサーバの構成は
・FreeBSD 4.3R
・Apache + mod_ssl-1.3.19+2.8.2
・MySQL 3.23.41
です。

私的には
・PHP が今のところ Web 業界標準なようだし
・PHP4 はなんかいい機能（オブジェクト指向が導入されているとか）
　があるみたいだけど
・PHP4 はまだ不安定だって話もあるし、
・Perl は難しそうだし、CPAN もよくわからんし、
・Ruby にも興味あるけど、すぐにバージョンアップして使い勝手が
　変わるから（昔の事かな？）保守性がちょっと心配だし
・sh とか csh で DB いじるのスーパー嫌っぽいし。。。^^;
ってな曖昧な情報しか無くて判断に困ってます。
やっぱり PHP3 でやるのがいいんでしょうか？

ご意見よろしくお願いいたします。m(_ _)m

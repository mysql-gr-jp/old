>アップグレートが不可だったようで、
>rpm -e php-mysql-3.0.15_i18n_ja-4jaJP
>を打ってしまったのが原因かもしれません。

　　そしたら当然ですね。'rpm -e' はアンインストールですから。『何をしたらどうなった』
　ということは端折らずに全部書きましょう。

　　次のテは以下のようなところでしょうか。

　　(1) 今入れてある MySQL に対応した PHP のパッケージを探してくる

　　　　存在するならこれが一番簡単ですね。

# tarball から install するより探すのが大変だった、というオチはあり得ますが（爆）。

　　(2) SRPM から rebuild

　　　　build  に必要なものを揃えるのが大変かも。

　　(3) source tarball から ./configure; make; make install

　　　　あまりおススメしません。


>（言い訳になり恐縮ですが、TurboLinux6.1の環境を
>使用していたためパッケージインストールに固執しすぎていた
>面もありました。）

　　固執する必要はありませんが、後々破綻しないために極力 RPM で入れるようにして、
　何か問題があったときにはすぐ思い出して対処できるように、パッケージ管理外で行っ
　たことは記録しておきましょう。

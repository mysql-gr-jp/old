「ビ」= A5 D3, 「ン」= A5 F3 ですね。MySQL デフォルトの文字セットの Latin1 
では、D3 と F3 は同じ文字とみなします。

	D3 = 「O」の上に「'」
	F3 = 「o」の上に「'」

文字比較では大文字小文字を区別しないので、一致するとみなされているの
でしょう。文字じゃなくてバイナリで比較してみてはいかがでしょうか。

	select * from easy_cart_item where binary name like '%ビ%';

…ってな感じ。

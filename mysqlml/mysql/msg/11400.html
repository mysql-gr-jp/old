直接 MySQL の話でなくて恐縮なのですが．．．

CPU とかメモリ搭載量、HDD のスペックなどの環境がわかりません
が、Windows2000 という OS はサーバとして使う場合、負荷を減ら
すのはサーバソフトウェアや SQL 文の最適化よりも、HDD を高速
なものに変えるとか、メモリを積めるだけ積むとか、ぺージング
ファイルの HDD を OS や MySQL の入っている HDD と別に用意する
とか、で大分動作速度の変わる OS です。

実際に数値データを取った訳ではないですが、メモリを 768MB
以上にするかしないか、ATA100 以上の HDD が使えるかどうかで
随分変わりますね。

CPU が Dual でも Intel のチップセットを使っていないマシン
は案外遅いとか、そういう特徴もあります。

バックエンドとして使う場合は CPU が pentium4 HT でなくとも
メモリさえ積んでいて HDD が ATA100 以上ならば pentiumIII で
十分、という感じです。
まぁ CPU が pentiumII/III の頃に開発された OS だから当然と
いえばそうなのですが。

kuyo@xxxxxxxxxx様の<87psglyl1h.wl@xxxxxxxxxx>から
>--skip-opt オプションに関して、mysqldump(1) に下記の記述があります。
>
>> If you are using a recent copy of the mysqldump program to generate a
>> dump to be reloaded into a very old MySQL server, you should not use
>> the --opt or --extended-insert option. Use --skip-opt instead.
>
>古い MySQL へデータをリストアしなければ、このオプションを
>付ける必要はないでしょう。
>おそらくですが、古い MySQL では属性として auto_increment が
>利用できないため、--opt-skip ではそれが省かれるのでしょう。
>
>ささきさんは、どのような意図で --skip-opt を付加しているのでしょうか。
>
　"--opt"を指定すると"--add-locks"が自動出来に指定されますが、すでに
"--lock-all-tables --flush-logs"を指定しているので、ロックの指定が重なる
のが嫌だな、などと考えて"--skip-opt"としました。いま考えると、どうでもよ
いことではありますが(苦笑。

　以上です。失礼します。

こんにちは、広津と申します。

MySQL に関わらず、データベースサーバーにおいては、
単一の SQL 文で追加・更新・削除などの処理が完結する場合は
ロックの必要はないと思います。

複数のクライアントから一斉に SQL 文が投げられても、サーバーが
１つずつ、順次、処理するからです。

ロックが必要なのは、追加・更新・削除の処理に複数の SQL 文を
投げる必要がある場合です。
（追加・更新・削除レコードを特定するための SELECT なども含む）

> localhostのIPアドレスをレンタルサーバのIPアドレスにする方法はあるのでしょう
> か。また、こういう場合の解決策はあるのでしょうか。

自分のパソコンに入っている mysql クライアントから、localhost に接続しようと
すると、当然自分のパソコンにつながってしまいます。


そうではなくて、今回アクセス権を変更しようとしているレンタルサーバにtelnetや
sshでログインして、そこから localhost に (つまり、レンタルサーバ自身に)接続
してみてください。

名執さんが「まるごとサーバ」と書いていらっしゃいましたが、多分
http://www.marugoto-server.com/
のサービスだと思います。
http://www.marugoto-server.com/service/faq/answer_server.html#3-26
この辺りを見ると、telnetでサーバにログインできるようなので、これでサーバに
接続すればよいと思います。
# ただし、telnetで接続するのは安全な方法とは言えません。
# 可能ならSSL-telnetやsshを使えるように設定する(してもらう)ほうがよいでしょう


その上で、アクセス権の定義を変更して localhost 以外から MySQL に接続できるように
すれば目的は達することができると思います。


ただし、セキュリティには十分に注意してください。
MySQL のプロトコルは暗号化がされていないため、通信経路の途中で覗き見をされると
やり取りしているデータが丸見えになります。
[mysql 06636]のメールを拝見したところ、プロバイダのアクセスポイントを経由して
MySQLに接続しようとされているようですが、セキュリティ上の危険性が高いため、
普通はこのようなことは行いません。(よね？)

どのようなデータをやり取りされるのかは存じ上げませんが、最近社内情報や個人情報
が漏洩する問題が各所で起きていますので、十分に、十分に注意されることを強く
お勧めします。

どうしてもリモートからMySQLを利用したいのなら、何らかの暗号化を行うことは
必須でしょう。
また、上にも書きましたが、素のtelnetも危険なので、暗号化されたSSL-telnetや
sshを使うべきです。

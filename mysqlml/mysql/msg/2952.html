とみたです。

[上西　太 <upw@xxxxxxxxxx>さんが]
["[mysql 02949] create 時のエラーについて" で曰く]

> #  mysqladmin create teishokuを実行した所、以下のメッセージがリターンさ
> れました。

> mysqladmin:connect to server at 'localhost' failed.
> error:Can't connect to local mysql server through socket
> '/var/lib/mysql/mysql.sock'(111).

エラーの意味は、/var/lib/mysql/mysql.sock 経由で通信できる MySQL サーバ
(mysqld)がいないってことなので、

> 過去のメーリングリスト検索を参考にし、以下の対応を行った上、再度#
> mysqladmin create teishokuを
> 実行しましたが、上記と同様の結果となっています。尚、
> /var/lib/mysql/mysql.sockを確認した所、容量が
> 0ＫＢとなっていました。
> ・mysql_install_db
> ・ｓａｆｅ_mysqld

対処方法としてはこれでいいはずですけどね…。ソケットファイルなので、容量
が0なのは問題ありません。ソケットファイルかどうかは
	ls -l /var/lib/mysql/mysql.sock
の結果の左端の文字が s になっているかどうかで確認できます。

もし mysqladmin -h 127.0.0.1 create teishoku でうまくいくようだったら、
ソケットファイルの状態がおかしいことが考えられます。

mysqladmin -u root -h 127.0.0.1 shutdown でサーバを一旦停止してから、再
度 safe_mysqld でサーバを起動すれば良いかもしれません。

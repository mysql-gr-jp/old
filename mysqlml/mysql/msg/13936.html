ご指摘の通りここが一番問題ですよね。オプションを理解して選択していないので
雑誌等に載っているコンフィグのオプションをそのまま入力してエラーとか

つけた場合とつけない場合では、どのようなMySQLの使用方法に違いが
でてくるのか不明なので教えていただければ幸いと思った次第です。。

> >--without-readline      Use system readline instead of bundled copy.
> と出てくるところを見ると、何も指定しなければmysqlにバンドルされている
> readlineを使うような気がします。

了解しました。

develをインストールして同様のコンフィグオプションでも同様のエラーが
でましたのでそのオプションなして使ってみようと思います。

# yum install readline-devel
# rpm -qa | grep readline
readline-4.3-13
readline-devel-4.3-13

佐々木です。

「コネクト失敗」というのは恐らく「かねやんMySQLAdmin」が
表示するエラーダイアログのメッセージです。

mysql_real_connect() というオリジナル関数の実行に失敗したときに
返されるエラーがトリガーの一つになっており、関数仕様は下記に
掲載されています。

http://www.mmdb.net/m_kaneko/mysql/kansu/mysql_real_connect.html

telnet で 3306 ポートを叩いて OK でしたから、
TCP/IPベースでは通信できています。
（最初は駄目でしたから何か環境を変えて出来るようになったんでしょう）
MySQL をソースからビルドして、何にも弄っていない状態のなら、
パスワード設定と権限設定を正しく行えば接続できます。
ですから、その他の要因を除けば、既に「かねやんMySQLAdmin」で
接続できている可能性があるので確認の問合せをした次第です。

もし、接続できていないなら「かねやんMySQLAdmin」ではなく、
 Windows XP へ MySQL クライアントをインストールして直に
mysql コマンドを操作してもらうとサポートしやすくなるんですが・・・。

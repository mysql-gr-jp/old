On Sun, 15 Jul 2001 23:09:04 +0900
"Ken Ide" <ken.ide@xxxxxxxxxx> wrote:

> 今mysqlが動いているhost上で
> host:~/$ mysqldump -u***** -p***** -T./ database table
> を実行し、tableのデータをテキストファイルに落とそうとしているのですが、
> mysqldump: Got error: 1: Can't create/write to file '/home/user/table.txt'
> (Errcode: 13)
> なエラーが出てうまくいきません。

mysqldump の -T オプションは、実は SELECT INTO OUTFILE なので、mysqld が
ファイルを作るのです。で、安全のために、パーミッションが 777 のディレクトリ
でないと mysqld はファイルを作らない仕様になっています。

ということで、-T/tmp とかだとうまくいくと思うのですが、いかがでしょうか。

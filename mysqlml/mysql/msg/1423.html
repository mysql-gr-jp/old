TurboLinuxになると出てくる小岩です。
rpmですが、4.2日本語版のCD-ROMにはMySQLは入ってなかったと思うんで、どこから
持ってきた何をインストールされたんでしょうか。TurboLinux社のFTPサイトにある
MySQLのrpmはたぶんSJIS/EUCオプションをつけてあるものではないと思いますが、
それを入れました? 日本語を扱う予定があるならソースから起こした方がいいと思
います。
ftp://ftp.softagency.co.jp/mysql/tcx/Downloads/MySQL-3.22/mysql-3.22.27.tar
.gz
を取得してから
     ソースを展開してそのディレクトリに降りて
     ./configure --with-charset=sjis
     make
     make install
     scripts/mysql_install_db
     scripts/safe_mysqld &
というここまででサーバ用もクライアント用も一通りインストールされます。もう
この段階でmysqlと叩くと
     [root@foo mysql-3.22.27]$ mysql
     Welcome to the MySQL monitor.  Commands end with ; or \g.
     Your MySQL connection id is 36 to server version: 3.22.27

     Type 'help' for help.

     mysql>
というようにmysqlプロンプトが出ます。

余談ですが先ほどTurboLinux SERVER 日本語版6.0 SOHO Edition(これが正式名称な
のかなぁ)をインストールしてMySQLを入れました。RPMSにMySQLが入っていたんです
がやはりcharsetは指定されていないようなのでrpm -eでスッパリ消してからやっぱ
りソースから起こしました。今のところ特に問題はないです。TurboLinuxにも慣れ
てきたのかもしれない。私はPerlを使うので、DBIもDBD::mysqlもperl -MCPAN -e 
shellを起こしてから入れましたが、特に問題なし。製品版はスッキリしていていい
なぁ、6.0β10で週末を潰したことを思うと天国のようだ。cpan shellはv1.48だっ
たけど。
手元にOracle8i Workgroup Server for Linux R8.1.5β版というOracle社から送ら
れてきたCD-ROMがあるので気が向いてたらMySQLと同居させてみます。でもメモリは
128MBしかないしClassic Pentiumの120MHzだからダメかもしれないなぁ。名簿管理
で動かしているMySQLなんか486DX 66MHzにメモリ40MBだけれど何にも問題ない。

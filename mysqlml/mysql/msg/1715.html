わたしも PAM はさっぱりだめです
mysql の pam 用のソフトがあります。
むかし、wu-ftpd に MySQL による認証を考えたとき、
最初に、wu-ftpd <-> pam <-> MySQL のアクセスを考えました。
しかし、pam は一体何をもらって何が返されるのかわからず、
/etc/passwd で使用されている以外の値を pam が返すようにするにはどうしたらよいのか
検討もつきませんでした。
さらにその時は(使用や開発の)ドキュメントが英語のみで、
pam-mysql もうまく動かすこともできず、断念しました。
結局、独自の値がMySQLから wu-ftpd, qmail, radius 等が取得できるように
pam はすてて、mysql client を組み込みました。

他に私が思い付いた組み込みは、
samba+MySQL, netatalk+MySQL
dnscache+MySQL (dnscache は qmail の作者が作った DNS サーバー)
です。
だれかトライしてみませんか？

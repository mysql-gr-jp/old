MySQLのCEOはソフトウエア特許を非難し、オープンソースを推奨

http://story.news.yahoo.com/news?tmpl=story&cid=2265&ncid=738&e=9&u=/infoworld/20050427/tc_infoworld/59083
4月26日　Paul Krill 著

サンフランシスコ (InfoWorld) - MySQLは多分一番目立つオープンソースのデータベース 

のプレーヤーとなりました。年間売り上げ$20Ｍで、MySQL5.0でエンタープライズ
市場に参入を予定。 MySQL5.0はtriggerなどの機能を追加しました。フィンランド
出身の Marten MickosはCEOですが、MySQLは隣のスエーデンから始まりました。
InfoWorld のエディターのPaul Krillが先週サンタクララで開催された
MySQLユーザー・コンファレンスの最中にMickosにインタビューしてMySQLの
データベース市場での位置やソフトの特許まで話を聞きました。

InfoWorld: 明日のキーノートでは何を話しますか。

Mickos: 私のキーノートの主なトピックは「Scale Out」です。
それは現在MySQLの活動の共通点です。もっとも有名な
MySQLのカスタマ（Sabre, Cox Communications, Google, Yahoo）
の例を見ると、共通点はビジネスが大きくなるに従って全てスケール
をあげて容量を増加して行くことです。5.0版ではサービスと
共に我々が提供するツールでこれが可能となります。たくさんの
サーバーを隣あわせに配置してどのような負荷にも耐えられるよう
にする方がはるかに簡単ですが。

InfoWorld: MySQL 5.0 はいつ出ますか。

Mickos: 6月末を予定しています。しかし、前にも言ったように
品質が一番大切なので、２週間前に言ったことと同じです。

InfoWorld: では6月に出ないかも。

Mickos: まだなんとも言えません。あらゆる努力をして時期を
厳守します。しかし、品質が一番大切ですから。時期を守れない
という理由もありませんが、保証できるのは品質で時期ではありません。

InfoWorld: 去年の末に出る予定だったのでは。

Mickos: ずっと前にはそう思っていました。

InfoWorld: どうしたんですか。

Mickos: 予想があまり良くなかったのです。去年の6月にエンジニアリング
担当の副社長を雇いました。彼がその件の責任者です。それ以降は
時期の予想はよくなっています。

InfoWorld:  MySQL 5.0はどのくらい重要なんですか。

Mickos: 新しいエンタープライズのユーザを獲得するのに
とても重要だと思っています。これにはエンタープライズが
要求する機能があります。 Stored procedure, trigger, viewや
XA [拡張アーキテクチャ]等です。

InfoWorld: ソフトの特許に関してはどう思いますか。

Mickos: ソフトの特許はソフトウエア産業にとっては害になります。

InfoWorld: どうしてですか。

Mickos: 革新や発明を妨げ、発明者を守ることがないからです。
ハードにとって良いからと言ってソフトにも良いというのは
誤りです。

InfoWorld: ソフトには特許はいらないと。

Mickos: そうです。

InfoWorld: 一番の競合はどこですか。Oracle、 PostgreSQL
か Microsoft またはそのほかに。

Mickos: ビジネスの分野ではPostgresは競合だと思っていません。
ビジネス分野では半分以上が新しいアプリや新しいカスタマなので、
全く最初からです。競合するところではMicrosoft,IBM, Oracleと
Sybaseです。

InfoWorld: PostgresはMySQLと似かよっているのでは。

Mickos: それは違います。まづ最初にPostgres は会社では
ありません。ビジネスモデルから言うと、全く違います。ソフトの
スタイルやアーキテクチャから言っても違います。 前にも言った
ように我々の３つのプライオリティは性能、信頼性
と使い勝手のよさです。機能は特別なプライオリティということ
ではないのです。そこが他のデータベースと違うところです。

InfoWorld: 機能がプライオリティに入っていないんですか。

Mickos: 言い方が悪かったです。機能は必要です。誤解しないでください。
でもプライオリティからいうと性能です。それが一番です。
もし今朝のファウンダのキーノートを聞かれたら
ソフトが速くて、バグが迅速に修正されて使いやすいことがいかに
大切か語っています。

InfoWorld: MySQLは何時からあるのですか。

Mickos: 10年前からです。

InfoWorld: オープンソースではコマーシャルのソフトの
ように金が儲かるのですか。ライセンスを只で提供したら
儲からないように思うのですが。

Mickos: それは皆が聞く質問です。しかし、我々の売り上げ
はインストレーション・ベースよりも早く伸びています。

InfoWorld: 売り上げはどのくらいですか。

Mickos: 昨年は$20Ｍでした。

InfoWorld: 悪くないですね。しかし、Microsoft やBM やSun
と比べるとたいしたことはないですね。

Mickos: 4年前には$1Ｍでした。今は$20Ｍです。弊社は
まだ大きな会社とは言えません。でも成長率をみると
他にこんなに早く大きくなっているデータベースの会社は
ありません。

InfoWorld: 今日MySQL マイグレーション・スイートを発表
しました。これでどうするつもりですか。

Mickos: これはOracle, Microsoft, IBM, Sybaseなどのデータベース
からMySQLにマイグレートしたい人のためのものです。

InfoWorld: どうしてマイグレートしたいんでしょうか。

Mickos: まづ最初にテストするためです。条件付でマイグレート
します。マイグレートするけれど前のデータベースはそのまま
おいておきます。もし結果に満足すれば最終的に
マイグレートします。性能強化、コスト削減と簡素化
のためにマイグレートします。

InfoWorld: ヂュアル・ライセンスはどんな仕組みですか。

Mickos: ヂュアル・ライセンスはディストリビュータ
には提供していますが、エンドユーザには２月に違う
モデルを提供し始めました。今はMySQL Networkを
販売しています。これはサブスクリプションです。
エンドユーザであれ、ウエブ・ベースのグループで
あれ企業だであれ、MySQL Network を何年・サーバー何台と
言って、購入するだけでそれでお仕舞いです。これには
サーティファイド・バイナリ、アップデートとアラート、
知識ベース、テクニカルサポート
が含まれます。また保証や免責も含まれます。

InfoWorld: でもライセンス費はないのですか。

Mickos: ありません。

InfoWorld: どうしてこのモデルに変えたのですか。

Mickos: カスタマがこの方が良いと言ったからです。
「ライセンス費の問題は扱いたくない。プライスのモデル
は簡単な方が良い。そして安価なオプションが欲しい。」と
言っています。

InfoWorld: あなたの会社はMySQLで新しい会社です。周りに
OracleやIBMのような巨人がいます。本当に競争できる
んですか。それともニッチで行くのですか。

Mickos: これはIkeaのモデルと見ています。Ikeaはスカンディナビア
の簡単なデザインで新しい家具が欲しい人に売ります。しかし、
必ず骨董品の家具の市場もあります。そしてそういうもの
にはたくさんのお金を払います。我々は競合から
ビジネスを奪っていると思っておりません。新しいビジネス
の分野は違うのですよ。

InfoWorld: ５年後、MySQLはどうなっていると思いますか。

Mickos: 今はまだ小さな会社ですがデータベースのビジネスは
大きいと思います。５年後には大きな会社になっていると期待
しています。

InfoWorld: MySQLはデータベースのコモディティ化を表して
いると思いますか。

Mickos: そう思います。そうなると思います。我々が最初です。
だからその恩恵を受けています。しかし、これは我々がいようが
いまいが必ず起こると思います。我々が仕掛けているのでは
なくそれに乗っているのです。

InfoWorld: Oracleのような会社にとってはこれはどういう
意味を持つのでしょうか。四半期ごとに１０億ドルの
売り上げというのはもうないんでしょうか。

Mickos: 初めからそんなふうであるべきでなかったので、
そういうビジネスはもう終わりになるでしょう。しかし、それなり
のビジネスは続けるでしょう。ある部分のビジネスに
はOracleは最適でしょう。Oracleは素晴らしい製品です。
それに関しては全く異議はありません。

InfoWorld: どんなビジネスに値しないんですか。

Mickos: 「我々はOracleショップでニーズに関わらず
いつもOracleを使うんだ。」という会社があります。
それは正しくありません。全く違う会社はこう言います。
「Oracleが一番適したところにOracleを使用し、MySQL
が一番適したところにMySQLを使用する。」

InfoWorld: オープンソースの市場へのインパクトをどう
みますか。例えば、Microsoftはどう関わってくるんでしょうか。

Mickos: オープンソースの流れは強くなりすぎて
もうだれにも止められないでしょう。Microsoftはこの流れ
には逆らわないと思います。それが証拠にすでに
オープンソースを持っています。Windowsのインストレーション
のスクリプトに使えるコンポーネントをオープンソース化
しました。MySQLはそれを使っています。誰が一番
オープンソースで金儲けをするかは興味深い質問だと思います。
現在だとオープンソースを使ってサービスを提供する人
が一番儲けています。 Yahoo, Google, Amazon, eBay, Salesforce.comを
みてください。現在、自分のインフラにオープンソースを
使用しています。それでウエブサービスとして売って
非常によく儲かっています。

InfoWorld: エンタプライズはオープンソースを短期
のはやりでなく、オープンソースをとおしてなにか
を入手できると思う時点に来たと思いますか。

Mickos: そう思います。私達にこういっています。
「貴方の製品は素晴らしいだけれど、代金を払うの
は嫌だ。」

InfoWorld: それに対してはどういいますか。

Mickos: それで結構ですが、サービスが欲しければ
お金を払ってくださいと。

InfoWorld: 無料で使っている人はどのくらいの数ですか。

Mickos: １人のお金を払うカスタマーに対して無料で使う
ユーザは千人くらいです。

InfoWorld: １人のお金を払うカスタマーに対して無料で使う
ユーザは千人くらいですか。

Mickos: そうです。

InfoWorld: 無料で使うユーザも含めると全部で
ユーザは何人くらいですか。

Mickos: 600万人くらいです。

InfoWorld: 殆どが無料で使っているのですね。

Mickos: そうです。しかし、データベース市場で我々を
みてください。既に無視できないプレーヤーに
なっていますし、皆ちゃんと受けとめてくれます。ここまで
に来るために投資したことは何だと思いますか。何も
在りません。他のデータベースの会社でここまで来るのに
どの位お金を使ったでしょうか。多額です。

InfoWorld: ２－３週間前に LAMP [Linux, Apache, MySQL, PHP/Perl/Python]
スタックに関して話しましたね。これは Java や.Netに対する
大きな競合でしょうか。

Mickos: 殆どの会社が複数のスタックをインフラに使用していると
思います。LAMPが３大スタックの１つになったのは事実です。
３大スタックは .Net, J2EEと LAMPです。

InfoWorld: LAMPに人気ある理由は。

Mickos: 使いやすく、拡張性があり水平にスケールするからです。

ありがとうございます、佐々木です。
そのテーブル構造とレコードデータなら、

select * from `diary` where `date` >= 1121298488 and `date` < 1121298743 order by date desc;

というSQLで１レコード目が取り出せると思います。
昨晩、柳町が書かれたSQLは `date` の比較値が文字列だった上、
２つめの `date` の比較値が１つめの比較値よりも小さいため、
検索条件で全てのレコードが除外されたものと思われます。

また date という列名については、SQL の規格では禁止されていますが、
MySQL では「一般的な単語である」という理由から使用が認められています。

あと、WHERE 句を使用すると問題の切り分けが出来なくなると書いたのは、
WHERE 句の質問をしているので、WHERE 句を利用しないケースで
きちんと MySQL からデータが取得できるか確認したかったからです。

以上、よろしくお願いします。

MySQL Network - 認定ソフト、アドバイザーとサポート
MySQL Networkはエンタプライズの開発者とDBAに時間と
手間を省く包括的なサービスです。これには
エンタープライズでMySQLをうまく使用するのに
必要な全てを含んでいます。

例えば
*アプリの信頼性を改善する認定ソフト
*問題が起こる前にアラートやアップデートを受け取ることの
できるサービス・アドバイザー
*何百という技術の記事の知識ベース
*問題解決のためのMySQLプロダクション・サポート
*その他。。。
MySQL Networkの詳細は http://www.mysql.com/network/

エンタープライズのソフトとハードのベンダー
はMySQL Networkを支持

HP, Intel, Novell, Red Hat と主な ISVは新しいMySQL パートナー
プログラムに参加してIT部門が容易にこのオープンソースの
データベースを統合できるようにしました。複数レベルの
MySQL Network 認定パートナー・プログラムはISV のパートナー
をパートナーのソフトが認定されていることとMySQLプロ認定
サーバーや関連ツールとコンパチであることを保障します。

ツールや技術が認定されていることを確かめてください。
http://solutions.mysql.com/program/

MySQLのコミュニティ版(4.1.10)のリリース
これは4.1版の保守リリースで主に最近発見されたバグの修理
版です。これには

*'lower_case_table_names=2' の際にはサーバーが警告を発生し
データ・ディクショナリはケース・センシティブな・ファイルシステムを使用
*異なった列の長さを持つより新しいサーバーにインターラプト
される文字の列を含む古いテーブルに遭遇すると警告を発生。
*InnoDB: 共有されるレコードのロック('LOCK_REC_NOT_GAP')はギャップ
にインサートが認められるためフォーリン・キー・チェックの
レコードとマッチするかどうか 確かめられる。

Windows上でMySQL サーバーをセキュアーにする
2005年1月の末Forbotと呼ばれるワームのようなマルウエアーが
駆け巡りました。これは正しくインストールされていないMySQL
を探し、それを利用してWindowsのマシンに侵入します。 Windows
上のMySQLに簡単なステップを踏めばMySQLやデータを守ることが
出来ます。一番大切なステップはデフォルトのユーザ
のアカウントをセキュアーにして外部からのアクセスを制限
してパスワードを強固にすることです。

詳細は
http://dev.mysql.com/tech-resources/articles/securing_mysql_windows.html
http://lists.mysql.com/mysql-ja/144

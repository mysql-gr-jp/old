お世話になります。横山ともうします。
環境は
サーバ      : Debian(woody): mysql-3.23.51
クライアント: Vine         : mysql-3.23.53

サーバ側で外部からもアクセスして操作できるmasterというユーザを
作りたくて下記のように入力してmasterユーザを作成しました。
$ mysql -uroot -p
> grant all privileges on *.* to master@localhost identifi
ed by 'password' with grant option;
> grant all privileges on *.* to master@"%" identified by 
'password' with grant option;

クライアントのVineからサーバ機に下記のようにmasterでアクセス
したところ
$ mysql -hDebian -umaster -ppassword
ERROR 2003: Can't connect to MySQL server on 'Debian' (111)
となりました。

サーバのDebianで下記のようにしたらうまくいきます。
$ mysql -hlocalhost -umaster -ppassword

外部のマシンからサーバのmysqlにアクセスするのに
masterユーザの権限の与えかたが間違っているのでしょうか？
それともサーバ側の何かの設定が外部からのアクセスを
拒否しているのでしょうか？
前者にしろ後者にしろどのように確認を行ったらよいのか
分かりません。
FAQのような内容で誠に恐縮ですが、アドバイスを頂けましたら
幸いです。

インストールを行いました。

WinXpにmysqlサーバをインストールした、ということですよね。
そして、そこで何をしたらどこでどう文字化けが起きたのですか？

例えば、そのWinXpのコマンドプロンプトでmysqlクライアントを
実行し、create database → create table → insert → selectと
順に行ったところ、selectした結果が文字化けして、
文字で表現されている罫線のレイアウトが崩れてしまっている、とか。

あるいは、テーブル名やカラム名に「表」の字を使ったところ、
show tableやshow fieldsで表示すると文字化けしている、とか。

> しかし、知人の環境(Win98)では、特に何もしなくて大丈夫(「\」
> を含む文字もOK)だそうです。

文字化けは、状況によって発現したりしなかったりします。
サーバ上で動くPHPスクリプトでは文字化けして表示されるデータでも、
ODBC接続で読んでみると全く文字化けしない、というケースもあります。

状況の違うものを並べても、原因はなかなか見えてこないのでは
ないでしょうか。

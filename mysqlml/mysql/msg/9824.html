前に報告いたしましたSELECT文、UPDATE文、INSERT文、REPLACE文実行中のエ
ラーですが、コードは長くなりすぎてかけませんが、
1.いつも一定の位置で起こる（プログラム上の）

2.エラー内容はユーザーIDとパスワードが一致しませんといったような内容でし
た。これが、プログラム実行中それも、同じテーブルを処理中にでるというのが
不思議なんですが。

3.プログラムは、PARADOXテーブルをMySQLテーブルに変換するものです。これに
はテーブル名、項目名の変換も含まれます

4.この時のdbexpressドライバーはCodeCentralのID: 20483 dbExpress MySQL 4.
0.x Beta driverでLibMySQL.dllは4.0.13に付属のものだったと思います
。
5.ただしdbexpressドライバー、LibMySQL.dllともにDelphiのBinディレクトリー
と生成されるEXEファイルの存在するディレクトリーの両方に入れました。そう
しないとSQLCONNECTIONのConnectedプロパティでTrueに出来ても実行するとコネ
クションエラーがでました

6.Delohiの7.1UpdateではMySQL4.0.18まで対応となっておりますので4.0.20でも
たぶん大丈夫と思うのですが

7.MySQL4.1および5.0ではCoreLab社のドライバーしか試していませんがテーブル
名および項目名に日本語を使うとテーブルの認識や項目の認識が出来ないような
ので4のプログラムを改良して英数アルファベットに変更しました。この現象は
ODBC経由でも一緒です。
今のところCoreLab社のドライバーでうまく動きます。
しかし約８０００円は高いBorland社の方でdbexpressドライバーへの対応（バー
ジョンアップ）を早くできないのでしょうか
dbexpressドライバーの作り方がわかれば

telnet などでlogon してレンタルサーバーのmysql にアクセスしてみましたか
cd /usr/local/mysql
bin/mysql -p *******
パスワードを入力すれば起動します　私のサーバーの場合は
あとは参考書かなんかでみてテーブルの作成をすればいいのではないですか
私も敷居が高いかなと思ったけどsqlの知識は少し有ったのでそんなにむづかし
くないと思いますが。
システム上の情報ぐらいはレンタルサーバーが教えてくれます。
私はじゃまくさいとアクセスからodbc でつないで、テーブルを作成します。
sql文を送るだけですが。
CREATE TABLE kenkyuin(age INT  NOT NULL ,email VARCHAR(50)  NOT NULL ,etc TEXT  NOT NULL ,handle VARCHAR(50)  NOT NULL ,id INT  NOT NULL ,ken VARCHAR(20)  NOT NULL ,kenkyuin_no INT  NOT NULL ,namae VARCHAR(50)  NOT NULL ,sex VARCHAR(10)  NOT NULL ,sukinasyocyu TEXT  NOT NULL ,UNIQUE id_index(id))
みたいに書いてます。

今は便利なツールがいっぱいあるので、便利です。
Myaccess とか　mysqlodbc とか、
「Common SQL Environment」は、
http://www.hi-ho.ne.jp/tsumiki/
にありますよ。私はこれらのツールにお世話になってます。

当方にて環境
VB6.0
Mysql　4.0.12
MyODBC　3.51.10
にて確認を行ったところ、
MyODBCを使用しAOD接続でSQLを発行した場合
VB6では
dim rs as new adodb.recordset
on error resume next
err.clear
rs.open SQL文,接続先
if err.number <>0 then
 msgbox err.number & chr(10) err.Description
 exit sub
end if
の様に明示的にエラートラップを咬ませないと
エラーが上がらず正常に表示出来ないみたいです。
＊SQLの構文　と　接続設定にて不具合が出るようにしたところ
エラートラップを咬ませないと空のデータグリッドが出力されました。

鶴長　様

以下,それぞれの設定で「mysql TEST_JDBC < make_sample_table.sql」を実行した結果です。

1.winmysqladminで,「my.ini Setup」タブの[mysqld]及び[mysql]に
　default-character-set=ujisを設定し「mysqladmin reload」した場合

  ---> ERROR 1064 at line 47: You have an error in your SQL syntax near ???????

    (ただし "???????"の表記は,文字化けの意味です)

2.winmysqladminで,「my.ini Setup」タブの[mysqld]及び[mysql]に
　default-character-set=sjisを設定し「mysqladmin reload」した場合

  ----> ERROR at line 47: Unknown command '\z'.

となり, どちらでもERRORとなりました。

 (動作環境は
　　(1) mysqlサーバーは, WINDOW NT Server 上で起動
　　(2) TEST_JDBCの構築の実行するmysqlクライントも(1)と同一PC上　)

winmysqladminで, TEST_JDBCテーブルを見るとフィールドが生成されていたので
DBとしての雛型は生成されています。 Sampleグラフィックデータ?(バイナリーデ-タ)の部分
でERRORになっているようです。

 それで,http://www.shoeisha.com/book/pc/mysql/ よりダウンロード
したグラフィックデータを登録するプログラム「ExampleA.jar」を
win NT上でダブルクリックし起動,
  サーバー: localhost   user: test  パスワード:test2000 
で　画像選択ボタンを押し1kB程度の容量のgifアイコンを選択し
挿入ボタンで正常登録。何件か登録し,
グラフィックデータを検索表示プログラム「ExampleB.jar」を
起動し(サーバー,user,パスワードは,同じ)正常に検索されることを
確認し,mySQLサーバーとの接続が確認できました。
　
　ただし,容量の大きいグラフィツク(ex. 39kb gifアニメーション)を登録すると
「...SyntxError or access violation 」とエラーになります。

以上　長くなりましたが　windowsでの動作確認しました。
 (別PC(win95)からの「ExampleB.jar」の実行もOKでした)

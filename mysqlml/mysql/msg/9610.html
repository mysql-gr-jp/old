> Accessの性能評価はネットワークからのスピード+クライアントの

それには同意しますですが、

> それに対してMySQLだとサーバ上でガンガン動作するのでサーバ
> のCPUパワー+メモリーの利用状態が大きく影響してくると思います。

メモリには同意しますが CPU パワーには単純に同意できません
例えばの話ですが同じ 1GHz のセレロンと pentiumIII とでは能力
の差があります。前者は L2 Cache 128KB,FSB100MHz、後者は
L2 Cache 512KB と FSB133MHz で使えます。

これは結構思ったよりも大きな差があります
同じコード、データを連続して使うような場合は頭が良くないと
(L2 Cache の量の差)メモリや HDD から頻繁に読み出します
そうしたら当然遅くなります

ましてや手足に当たる Memory は CPU が速くても動作速度が遅い
のだとその名の通り足を引っ張ります

pentiumIII 以降の PC というのは、頭が良い(CPU が速い)のに、
手足(Memory、HDD、PCI ブリッジアクセス)が遅く、付いていって
居ない状態です。サーバ仕様で売って居るものはともかく、クラ
イアント型やコンシューマ向けのメーカ PC は標準実装のままでは
メモリは確実に不足です

何故そのようにわざわざ不足した状態で出荷するのかは、販売価格
を抑えたいというところからでしょう

> でも、やっぱり導入するとなると色々比較しちゃいますよね

そのお気持ちは分かるんですが．．．

発言内容から現時点では導入先の要望を入れて、何にするか迷って
居るようにお見受けしました

Access は確かに手軽で値段の割には高機能です
しかしクライアント PC の数が増えると果たして安いのか？という
疑問があります。VBA で何とかするんだったらむしろ Excel を
クライアントにした方が、メーカ PC ではプリインストールされ
ライセンスも付いてくるし安上がりに感じます

それから MDB に DAO でアクセスすると確かに ADO よりも格段に
速いのですが、確か致命的不具合があり、未だ改善されてなかった
と思います。その意味では ADO を使うべきですが今度は客の要望
(速度)を満たせないでしょう

何れにしても評価していらっしゃるマシンの能力にはかなりの不足
があります。新規導入するなら、メモリの容量はともかく CPU の
能力と Memory のアクセス速度は格段に速くなりますので評価の
意味がありません。
既存のものを流用するということで、それを使っていらっしゃるの
でしたら Windows だろうが Linux であろうが、サーバというのは
とにかく実メモリを積めるだけ積む、のが鉄則です。
HDD にスワップするより実メモリで動く方が遥かに速いのですからね

その点で Access と MySQL を速度だけの面で比較するのはどうか
と思います

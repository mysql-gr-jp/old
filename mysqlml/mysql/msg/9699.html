takeshi@xxxxxxxxxx wrote:

> また、クライアントに対する
> ./configure --with-client-charset=binary
> ですが、
> 以下が 4.1.3 前の(今はbitkeeerに入っている)ものに対する簡単なパッチです.
> 
> またこれが必要な場合ですが、どういう場面で使うでしょうか?
> 私は次のように考えています
> 1. 上の mysqld --init-connect をしなくても、設定無しで 4.0, 3.X までと同様に動く
> 2. デフォルトで文字が壊れるとかのトラブルに遭遇しなくなる
> 3. バイナリを作成する人が、サーバー用とクライアント用の2回コンパイルする必要が無い
> 他ありますか？

サーバーに、
--init-connect='set names binary'

を付けていると、当然以下のようになりますが、

% mysql --default-character-set=ujis test                                             
mysql> show variables like 'character%';
+--------------------------+----------------------------+
| Variable_name            | Value                      |
+--------------------------+----------------------------+
| character_set_client     | binary                     |
| character_set_connection | binary                     |
| character_set_database   | ujis                       |
| character_set_results    | binary                     |
| character_set_server     | ujis                       |
| character_set_system     | utf8                       |

--default-character-set=ujis が上書きされて binary になるので、
SET NAMES 文を実行せねばならないのか、という状況が生まれます。
サーバーは一人だけが使うものではないので、
binary が接続時に全て実行されるのもちょっと混乱しそう。

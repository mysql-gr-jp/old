> >FULLTEXT は、空白で区切られた単語しかインデックスしてくれないので、日

> >
> >単語がちゃんと空白で区切られているなら、速いかもしれません。

嘘でした (__) マニュアル読んでみたら、

  6.8 MySQL Full-text Search

  MySQL uses a very simple parser to split text into words. A ``word''
  is any sequence of letters, numbers, `'', and `_'. 

…と書かれてたので、空白で区切られてるかどうかじゃなくて、英数字と「'」
「_」の連続を単語と見なしているようです。

ということで日本語は全然アウトです。

  6.8.3 Full-text Search TODO

  ・ Support for multi-byte charsets.

という記述もありますね (^^;

バイナリデータの件も少しは参考になれば。

＞使用条件及び問題点
＞            バイナリのデータを登録したい
＞            特定の文字（ソなど）が化けたり、特定の漢字の登録でエラーになる
＞            日本語項目に \ マークを入力してもデータ登録されない

Visual Basic .NETとASP.NETで、全部できてますが、
使用しているものは、

            Windows XP
            MySQL 4.0.5（変更点はmy.iniのみ）
            MyODBC 3.51（英語版のまま）
            MDAC 2.7日本語版（MS）
            ODBC .NET DataProvider日本語版（MS）
            Visual Studio .NET

です。日本語変更機能版は今は使っていません。
バイナリデータでデータを登録・利用するだけなら、

            Windows XP
            MySQL 4.0.5（変更点はmy.iniのみ）
            MyODBC 3.51
            MDAC 2.7日本語版

でも、BLOB型のフィールドに、Access 2000でバイナリデータを
張ることも引っ張ることもできました。
ただ、貼られるデータがコンテンツタイプを含んだ形でOLEオブジェクトに
されてしまうため、Accessからのアクセスに限定されたフィールドにな
ってしまいそうです。

僕的には、Windows 2000ならASP.NETでADO.NET対応のマネージド
ドライバーを使うのがいいかと。.NET Frameworkを使うのがいやじゃな
ければ。
ODBC接続より高速ですし、日本語が通るものもありました。

以下、参考までに

□ 問題
・AサーバのMySQLをBサーバで読み出す

□ 前提
・AサーバにMySQLをインストールする



☆Bサーバ
■MySQLをインストールする
+------ tar解凍・インストール ------+
# tar xvf mysql-4.0.12.tar
# cd mysql-4.0.12
# ./configure -prefix=/usr/local/mysql --without-server
# make
# make install


■以下モジュールをインストール
　・DBI-1.35.tar.gz
　・DBD-mysql-2.1026.tar.gz
　・Msql-Mysql-modules-1.2219.tar.gz
　・Data-ShowTable-3.3.tar.gz

インストールの方法は全て
# gunzip XXXXXX.tar.gz
# tar xvf XXXXXX.tar
# cd XXXXXX
# perl Makefile.PL
# make
# make install


ただしMsql-Mysql-modulesのインストール時で、

perl Makefile.PL

をした場合、以下の質問をされるので答える

------------------------------------------------------------------ここから
Do you want to install the MysqlPerl emulation? You might keep your old
Mysql module (to be distinguished from DBD::mysql!) if you are concerned
about compatibility to existing applications! [n]　　　　　　　　←リターン
Where is your MySQL installed? Please tell me the directory that
contains the subdir 'include'. [/usr/local/mysql]　　　　　　　　←リターン
Your Perl configuration says you are using gcc, but your compiler (gcc) doesn't
look like gcc. There might be missing symbols in libmysqlclient.a, typically
'umoddi3' or something similar, if you have precompiled mysql binaries. If so,
try to compile your own binaries, perhaps the '--without-server' option
might help in the configure stage.
Which database should I use for testing the MySQL drivers? [test]　←リターン
On which host is database test running (hostname, ip address
or host:port) [localhost] 　　　　　　　　　　　　←Aのホスト名を入力しリターン
User name for connecting to database test? [undef]　　　　　　　　 ←リターン
Password for connecting to database test? [undef]　　　　　　　　　←リターン
------------------------------------------------------------------ここまで



■全てインストールされたら、AサーバのMySQLで使用できる権限を与える
☆AサーバのMySQLを起動

mysql > use データベース名
mysql > grant all on テーブル名 to Bサーバのホスト名;


これでBサーバからAサーバのMySQLを呼び出すことができました。
パスワードの設定とかは省略したのですが、こちらは順次していきたいと
思います。

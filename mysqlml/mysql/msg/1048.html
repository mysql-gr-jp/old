メーリングリストのアドレス変えました。

さて、ldap サーバをあげようと思っているのですが、バックエンドのデータベースを
MySQL でやろうと思っています。ここらへん、実装されたものの存在等ご存知の
方いらっしゃいませんか？

ldap は、http://www.openldap.org のものを利用します。

redhat,debian などでは、パッケージ化されていると思いますので、導入は簡単なのですが、
バックエンドで使えるデータベースが、gdbm なので、いろいろ実験やるのは不便なのです。

用途としては、ldap でシングルサインオン の環境をつくりたいと思っています。

状況としては、次のようなことを考えています。

いくつかのショッピングサイトが提携をしているので、顧客の基本的な情報
（本人認証のためのパスワード、住所など）は共有したい。というときに便利です。
物理的にバラバラのおのおののショッピングサイトが、ldap 経由で、本人認証
住所情報等の取得を行います。個別の取り引きデータは、各ショッピングサイトが
保持します。実質的に重要な顧客情報は、取り引き情報ですから、その管理は、
各サイトがしますが、パスワード、住所等は、別のldap server が受け持つわけです。
顧客にとっても、便利です。一度、情報を書きかえれば、他のサイトでも使えるわけ
ですから、

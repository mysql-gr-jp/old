> ところで、ソースの記述がおかしいのでしょうか、それとも。MysqlのGRANTの仕方が

そうです。
SQLException ですから Class.forName() は通過しています。
それから、例外のメッセージは階層的に出力されますので、メッセージを辿れば
MyJdbcTester.java 内のどの行で発生した例外なのかを見つけることができます。

また、権限設定は接続テストを行うだけなら、
GRANT ALL ON *.* で全てを許可した上で、最初から存在する test データベースへ
接続を試みる方が分かり易くないですか？

尚、MySQL の JDBC は
　org.gjt.mm.mysql.Driver
　com.mysql.jdbc.Driver
の両方をパッケージしてますので、どちらを指定しても構いません。
java.lang.Class.forName と java.sql.DriverManager.registerDriver
も内部的には同じ結果になりますから、これもどちらでも構いません。

とりあえず、他の方も言われてますけど、JDBC より先に java そのものへの
理解を深めた方がいいかと思います。
メジャーなもの以外は API リファレンスとソースコードしかない世界ですので・・・。

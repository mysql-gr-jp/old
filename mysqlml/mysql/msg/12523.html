事例 - NetQoS Delivers Distributed Network Management Solution

NetQoS ReporterAnalyzerはMySQLを使い、IT のグループに
どのアプリやユーザが帯域を消費しているかを示し、
ネットワークを改善し、サービスを改善します。
NetQoSによれば MySQL は性能、信頼性と使い勝手のよさを
提供するとのこと。更に、MySQLの手頃なライセンスモデル
のおかげで、 NetQoS はネットワーク・アプライアンス
に付き何千ドルもの経費を削減できます。

事例のダウンロード
http://www.mysql.com/why-mysql/case-studies/mysql-netqos-casestudy.pdf

製品
MySQL 5.0.16 コミュニティ版（ＧＡ）のリリース

これは製品版のMySQL コミュニティ版（5.0シリーズ）です。
追加は

* InnoDB, NDB, BDBと ARCHIVE ストレッジエンジンはspatial column

コメントのhexのダンプを表示。このアウトプットはリプリケーション
のデバッグをするのに役立ちます。
* 様々なバグ修理

記事: BIRTを使ってMySQLのなかで Bugzillaにレポート

この例ではBIRT listingレポートを
開発します。データソースとしてMySQL上にインストール
されたbugzillaデータベースを使います。このレポート
で示したいのはシステムにある全ての現在のバグのリスト
です。これは簡単ではあるけれどなかなかパワフルな
レポートです。これには合計つきの Bugzilla Products and Components
のグループを示します。これにはたかだか15分しか
掛かりません、BIRTをよく知っていれば。

記事を読む

サンプル: PHPのセキュリティへのphp|architectガイド
3章, SQL Injection

SQL injectionはインプットをきちんと検証しないため
に生ずる弱点です。究極的には貴方のサイトへ
のビジターに影響するクロスサイトのスクリプトの
弱点のようではなく、 SQL injectionはサイトそのもの
への攻撃です。特にデータベースです。

この章を読む
http://dev.mysql.com/tech-resources/articles/guide-to-php-security-ch3.pdf

Subject: [mysql 10351] 4.1.7 (これは4.1のプロダクション最初の版です。)



MySQL 4.1.7,がリリースされました。 http://dev.mysql.com/downloads/
またはミラーサイトから幾つものプラットフォーム用にソースまたはバイナリ形式で
ダウンロードできます。

全てのミラーサイトがアップデートされているわけではありません。
特定のミラーサイトにない場合、後ほどチェックし直すか他の
サイトを試してみてください。

これは4.1のプロダクション最初の版です。

バグの詳細はバグデータベースを参照してください。http://bugs.mysql.com/

4.1.7での変更

   追加機能と変更機能
     * InnoDB: LOCK TABLES の動作をデフォルトで4.0.20 と 4.1.2以前のように
しました。InnoDB のロックは取らない。スタートアップ・オプションを追加、LOCK 
TABLE
をInnoDBロックも入手するようにinnodb_table_locksというセットできるシステム変数 

も追加。参照：マニュアルの「Restrictions on InnoDB Tables」の章。
       (バグ番号 #3299, バグ番号 #5998)

   修理されたバグ:
     * prepared statement内でLIMITクローズと一緒に使用されたFOUND_ROWS()の
バグを修理。(バグ番号 #6088)
     * prepared statementのNATURAL JOIN のバグを修理(バグ番号 #6046).
     * 異なったデータベースで同一の名前を持つコラム(prepared statement）
のJoinのバグを修理。(バグ番号 #6050)
     * システム　タイムゾーン記述テーブルへの暗示的アクセスに特権
が必要とされない。(これはtime_zone変数をセットするかCONVERT_TZ()関数
を使用する場合。）(バグ番号 #6116)
     * 旧式にされたlibmysqlclient 関数のmysql_create_db()が呼ばれた際に
サーバがクラッシュするバグを修正。 (バグ番号 #6081)
     * REVOKE ALL PRIVILEGES, GRANT OPTION FROM ユーザを修理。全ての
特権が正しく取り消されるように。(バグ番号 #5831). これは4.1.6で見逃された
バグも修理。
     * キー・コードがBLANKの下の文字から始まる場合MyISAM インデクスの
コラプトを修正。(バグ番号 #6151)
     * 結果のセットを返還しないSELECTステートメント(例えば　SELECT ... INTO 
OUTFILE)
に間違ったメタデータが送られた場合のprepared statementのプロトコルのバグ
を修正 。(バグ番号 #6059)
     * プライマリー・キーを持ちINSERT とSELECT 特権をテーブルにもっている
際、UPDATE特権を迂回するバグを修正。(バグ番号 #6173)
     * libmysqlclientの間違った変換（負の時間値をストリングへ）を修理。(バグ番号 #6049).
     * libmysqlclient のゼロ(0000-00-00)の日にちの値をストリングに変換するときのバグ
を修正。(バグ番号 #6058)
     * RAND(?)でステートメントを用意しようとするとサーバーがクアッシュする
バグを修正。(バグ番号 #5985)
     * バイナリー・プロトコルでDATE, TIME, と DATETIME コラムを扱う
際のバグを修正。問題はコンパイラー特有で、HP-UX, AIX, Solaris9でその
プラットフォーム用のコンパイラで処理したときに見られる。(バグ番号 #6025)
     * バイナリー・プロトコルでTINYINTコラムを処理した際のバグ
を修正。これはプラットフォーム特有で、Cコンパイラがデフォルトでchar
タイプがunsignedの場合。 (バグ番号 #6024)
     * MySQL 4.0.21 で挿入されたバグを修正。接続がトランスアクションを開始、
アップデート、FLUSH TABLES WITH READ LOCK、COMMIT, した場合リプリケーション
のスレーブがerror 1223を吐いて停止。バグはInnoDB innobackup スクリプト
を使用した場合。(バグ番号 #5949)

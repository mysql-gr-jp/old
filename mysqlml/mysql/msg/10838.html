製品
MySQL (4.1.9 製品版)がリリース
これは4.1製品版の保守リリースで、主に最近発見された
バグの修理版です。リリースは以下を含みます。

InnoDB: AUTOCOMMIT=1であればLOCK TABLESにある内部
のInnoDB テーブル・ロックを取得しない。こうすれば古いMyISAM
のアプリをInnoDBにポートできます。
Seconds_Behind_MasterはもしスレーブのSQLスレッドが
実行されていないかスレーブI/Oスレッドがマスターに接続して
いないと NULL (``unknown'') となる。SQLスレッドがI/Oスレッド
に追いつくと0になります。マスターがアイドルだと際限なく大きくなる
ことはありません。
幾つかのバグ修理
変更に関しては
http://lists.mysql.com/announce/252
ソースとバイナリのダウンロードは
http://dev.mysql.com/downloads/mysql/4.1.html

MySQL Query Browser (1.1.5 Gamma)がリリース

ヒント
MySQLの新しいストレッジエンジン
MySQL データベースのサーバーのアーキテクチャーはモジュール化
形式で形成されています。サーバーのコアである上層部の機能
例えば、ネットワーク層、SQLパーサーやクエリのオプティマイザー
などははっきりとしたインタフェースでデータ・ストレッジと
は区切られています。MySQLデータベースはデータをMyISAM テーブル
またはトランズアクションのInnoDB データストアーに格納されています。
MySQLにストレッジエンジンを追加するのは比較的簡単です。
サンプルコードを含む記事を読んでください。
http://dev.mysql.com/downloads/connector/j/3.2.html　（英文）
http://lists.mysql.com/mysql-ja/120　（日本語）

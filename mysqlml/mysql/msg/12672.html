Subject: [mysql 12672] 特定のクエリを走らせるとMySQLが落ちる

橋本です。
お世話になっております。

CentOS4.1 , MySQL 4.1.12 なのですが、特定のクエリを
走らせると、MySQLが落ちるという現象に悩まされております。

以下のクエリでMySQLが落ちます。

  SELECT
    COUNT( * ) AS Count
  FROM
    Recipe_DateOrder
  WHERE
    MATCH( Title_s,Description_s,References_s,Ingredients_s,Tag_s )
    AGAINST( '+"ミ"' IN BOOLEAN MODE )

全く同じ構成で、レプリケーションにより並列化している、別のサーバの
MySQLも、全く同じクエリで落ちます。

原因が全く分からず、対策が立てられずに困っております。
相談に乗っていただけるとありがたいです。
よろしくお願いします。


ちなみに、次のようなクエリだと落ちません。

  SELECT
    COUNT( * ) AS Count
  FROM
    Recipe_DateOrder
  WHERE
    MATCH( Title_s )
    AGAINST( '+"ミ"' IN BOOLEAN MODE )

これも、落ちません。

  SELECT
    *
  FROM
    Recipe_DateOrder
  WHERE
    MATCH( Title_s,Description_s,References_s,Ingredients_s,Tag_s )
    AGAINST( '+"ミ"' IN BOOLEAN MODE )

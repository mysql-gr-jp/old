オープンソースのデータベースは伸びています
Bruce McCabe著
2005年４月19日

mySQLはオープンソースの世界の次の大物になるか。

2001年の11月にMicrosoftのエグゼキュテイブのBrian
Valentine は営業のスタッフにメモを送り、「Linux
は我々のコアのビジネスに対して長期に渡る脅威だ。」
と語りました。

2003年６月には Microsoft CEOの Steve Ballmerが新たなメモ
を送りました。今回は全社宛です。

市場に漏れたメモはオープンソースからの脅威を完全
に無視していた会社からの驚くべき告発でした。

次のメモはどこから出るでしょうか。Linux
は何千もあるオープンソースの１つに過ぎません。OSの
世界の秩序を破壊できるのであれば、次の爆弾は
どこで破裂するのでしょう。どの商用ソフトの部分
でこれが起こるのでしょうか。

2004年のはじめ, オーストラリアのオープンソースの
リサーチ・プロジェクトを完了しました。これの焦点はIT
で決定権を持つ人々の態度、興味とモチベーションでした。
結論の１つは次の企業や政府での大きなことはオープンソース
のデータベースだということです。特にmySQLが。。。。

私はこれにはじゃっかん、懐疑的でした。というのも
IBMがLinuxに示したような大きな動きで、mySQLを
プロモートしようというチャンピオンも現れて
いないし、大きな展開も見られないからです。

オープンソースに関する発表の中で、聴衆にmySQLかまたは
似通ったをプロダクション用に使用しているか聞いて
手を上げてもらいました。

４つの都市で平均すると約30％の手が上がりました。

これはランダムのサンプルではありません。講演は
Red HatとIntelが催したイベントの一環として開催
されました。出席者はオープンソースに関心がある組織に
属していると思って間違いないでしょう。それにしても
オーストラリアの平均的なビジネスを代表している
と思います。大企業も含まれています。

mySQLはオーストラリアの会社にあまり見えずに
大々的に報道もされずにまた上からの指示でもなく
着々と浸透しています。

mySQLを批判する人はスケーラビリティのなさと
Oracleのようなコマーシャルの製品と比較して
機能不足をあげます。

もちろん、彼らは正しいと思います。

OracleのエグゼキュティブはmySQLは脅威ではない
といいます。これはどこかで聞いたような。

IT のマネージャーは、大部分のアプリにとってOracleの機能
の１部で十分であるといいます。それだけでも、市場は大きい
です。

その上Linuxのスケーラビリティや管理性があがった
ように、mySQLにもエンタープライズの使用に耐える
機能がつくのは時間の問題に過ぎません。

Oracle はLinuxをプローモートして、オープンソース
のコミュニティの最良の友人と言って良いかもしれません。Oracle
がオープンソースのOSをプロモートしている中で、
オープンソースのデータベースがコアのデータベースの
大きな部分を食ってしまったら、皮肉としか言えません。

私が思うには次のメモの署名欄には "Larry Ellison"と
なっているのではないかと。。。。。。。

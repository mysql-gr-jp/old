Subject: [mysql 13769] show columns で NULL カラムにNOが入る (5.0.27)

成松と申します。

Fedora 5 で 5.0.27 を利用していますが、Redhat 9で 4.1.9 ではテーブル作成後の
show columns の表示結果が違います。
（Null カラムが5.0.27ではNO、4.1.9では空白となります。）
これは仕様でしょうか？Nullのカラムを空白にすることはできるのでしょうか？

-- Fedora 5 ( 5.0.27)
mysql> CREATE TABLE A(A1 VARCHAR(1) null , A2 VARCHAR(2) not null );
Query OK, 0 rows affected (0.00 sec)

mysql> show columns from A;
+-------+------------+------+-----+---------+-------+
| Field | Type       | Null | Key | Default | Extra |
+-------+------------+------+-----+---------+-------+
| A1    | varchar(1) | YES  |     | NULL    |       |
| A2    | varchar(2) | NO   |     |         |       |
+-------+------------+------+-----+---------+-------+

-- Redhat 9( 4.1.9 )
mysql> CREATE TABLE A(A1 VARCHAR(1) null , A2 VARCHAR(2) not null );
Query OK, 0 rows affected (0.12 sec)

mysql> show columns from A;
+-------+------------+------+-----+---------+-------+
| Field | Type       | Null | Key | Default | Extra |
+-------+------------+------+-----+---------+-------+
| A1    | char(1)    | YES  |     | NULL    |       |
| A2    | varchar(2) |      |     |         |       |
+-------+------------+------+-----+---------+-------+

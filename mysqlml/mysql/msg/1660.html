  吉田です。

今、MySQLをデータベースエンジンに使って実際の業務で利用するためのシステム
を開発しています。もともとオフコンで動作しているシステムをクライアント/サ
ーバ化するもので、もとのシステムはCOBOLで書かれていました。
  システムは Linux/MySQL(DBサーバ)-Linux/perl/DBI(アプリサーバ)-Windows95
/VC++/MFC(クライアント)の３階層クライアント・サーバシステムの構成です。定
型的な処理はVC++ですが、日常随時発生する非定型なデータ検索などの処理用に
は Windows環境で比較的簡単に操作できるツールが欲しいと思っていました。
  そんなわけで、できる限り汎用的に使えるGUI検索ツールを ruby-gtk+-MySQLで
作ってみました。多分これは一般的に利用できると思いますので投稿します。

  プログラム名: SearchMySQL
  動作環境    : ruby-1.4.2        ┐ http://www.softagency.co.jp/に置いて
                mysql-ruby-2.2.0  │ ある Windows版 rubyの環境です。
                ruby-gtk-022      ┘

  動作確認    : Windows NT 4.0,Windows 98にて確認
                サーバは Linux
                MySQL 3.22.30(シフトJISコード)

  インストール: SearchMySQL.rb (スクリプト本体)
                SearchMySQL.rc (リソースファイル)
                SearchMySQL.txt(使用説明ドキュメント)

                スクリプトとリソースファイルを適当なディレクトリに置いて、

                 > ruby searchmysql.rb

                のように起動します。

  補足        : 当方での典型的な検索パターン、DB定義を前提で書きましたので
                いろいろ欠けたものがあるかもしれませんが悪しからず。
                まだ日本語のキー入力ができていません。できるようにしたいの
                ですが Windows版 ruby/GTK+ で日本語を扱う方法が分かりませ
                ん。また検索結果のデータはいまのところ、コード変換などして
                いませんのでEUCで使用している場合は文字化けするものと思いま
                す。
                スクリプトの中を見ればおわかりのように大半は、GTK+用の記述
                です。

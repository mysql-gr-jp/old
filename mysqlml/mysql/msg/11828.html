その質問だと「あなたが必要とする設定」としか
答えられないような…。

何がしたくて (これは書いてあるが)、何ができて、何が問題で、
何がわかっていて、何がわからないのでしょうか?

…

別の方とのやりとりで inetd を停止していましたが、
的外れです。ご自身で inetd.conf を触ってないのであれば、
100% 関係ありません。

「iptables がうんぬん」といった指摘もありましたが、
現在の Debian は iptables による制限はデフォルトで
かからないので、ご自身で設定していない限りは問題ないかと。

telnet で MySQL サーバーに接続したときに返事が返ってこない
とのことですが、Windows XP からではなく、Debian
(MySQL サーバーが稼働しているホスト自身) からだとどうなりますか?
  $ telnet localhost mysql
TCP 接続でないソケットによる接続だとどうなりますか?
  $ mysql -u superadm

そもそも、mysqld プロセスは稼働していますか?
  $ ps -ef |grep mysqld
また、mysqld は TCP のポートで listen していますか?
  # netstat -anp |grep 3306

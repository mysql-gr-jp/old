はじめまして。

> Perl5 + DBI + Apatche + mySQLで
> 「表」などの漢字を含む文字列を挿入しようとすると
私の場合、同じ環境なんですが何の問題もありません。
クライアントはNTなのでフロントエンドとしてAccessの
手軽さは捨てられないのでSJISでやってます。

たしかMySQLは、どんなコードでもOKなんで、MySQLではなく
PerlやDBIに問題がないかしらべてみてください。

たとえば、"表"もこのまま
$sth = $dbh->prepare( qq{ INSERT INTO test_table (test_col) VALUES ('表')})
$rc = $sth->execute;
ではエラーですが

表が入った変数$hyoがあるとして
$sth = $dbh->prepare( qq{ INSERT INTO test_table (test_col) VALUES (?)})
$rc = $sth->execute($hyo);
なんてやると、ほぼコードを意識しないでインサートすることもできます。

もうやってたらすいません。

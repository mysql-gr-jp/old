とみたです。

On Sat, 12 Feb 2005 18:40:28 +0900
"T.Nakatuka" <nakatuka@xxxxxxxxxx> wrote:

> 現状は、OS　：　redhat9.0によりinstallしたlinux
>             MYSQLはrpmにより4.1.7をinstallを行いました。
> 
> うまくinstallできているいるつもりなのですが。
> rootのプロンプトより
> mysqladmin ping をtypeすると
> 
> mysqladmin: connect to server at 'localhost' failed
> error: 'Access denied for user 'root'@'localhost' (using password: NO)'

エラーメッセージを読みましょう。パスワードを使用しないで 
root@localhost から接続しようとしたらアクセスが拒否された…ということ
なので、

> mysql -uroot -p  mysql
> Enter password: *****
> 
> とTypeすると
> Welcome to the MySQL monitor.  Commands end with ; or \g.
> Your MySQL connection id is 30 to server version: 4.1.7-Max-log
> 
> Type 'help;' or '\h' for help. Type '\c' to clear the buffer.
> うまくつながっている様に思います。

…と、同じようにして、mysqladmin -uroot -p ping してみれば良いと思います。

> この状態は、LAN上のWindowsの機械から接続出来ているのか、できないのでしょうか？ 

今までの話はローカルホストのクライアントから接続するときの話なので、
「LAN上のWindowsの機械」から接続するのはまた別の話になると思います。

Subject: [mysql 08057] mysqldump が動作しない

はじめまして、makotoと申します。

FreeBSD4.8R、MySQL3.23.56上にてmysqldumpが動作しなくて困っております。
何もオプションをつけず、データベース名だけ指定しても以下の様な
メッセージが出てdumpが出来ません。

db# mysqldump test
mysqldump: unrecognized option `--~'

my.cnfは以下の通りです。
[mysqld]
default-character-set=sjis
set-variable = key_buffer=64M
set-variable = max_allowed_packet=1M
set-variable = max_connections = 255
set-variable = record_buffer=1M
set-variable = sort_buffer=1M
set-variable = table_cache=255
[mysqldump]
default-character-set=sjis

configure時オプションは以下の通りです。
./configure --with-charset=sjis --with-extra-charsets=all --prefix=/usr/local/mysql --localstatedir=/usr/local/data --with-mysqld-user=mysql --with-innodb

ご存知の方がいましたらご教授いただければ幸いです。

2005年MySQLはエンタープライズ市場でもっと大きなシェアーを目指します。
http://oetrends.com/news.php?action=view_record&idnum=390

01/20/05

Vance McCarthy著

オープンソースのデータベースのベンダーのMySQL は2005年は
エンタープライズのデータベース管理者とITのエグゼキュテイブ
を対象にして活動します。先月の4.1版のメージャーなアップグレード
で、このオープン・ソースのデータベースは新しいグラフィカルな
ビュー、管理ツールを提供し、ハイアベイラビリティのための
クラスタリングまで提供します。

MySQLのエンジニアは(広範囲なコミュニティからのサポートを受けて）これで
終わりだとは思っていません。今年はMySQL5.0版の最後の仕上げ
を行い、エンタープライズやISVが待ち望んでいた機能を追加します。
その機能とはstored procedureなどです。Open Enterprise Trends
が2005年のMySQLのエンタプライズ市場での受けとめられ方を
検証します。

MySQL 4.1版の最近のアップグレード

MySQLのマーケティング担当副社長のZack Urlocker
は「私達は開発者とDBAがMySQLでアプリを作成しやすいように
また、開発の後も維持しやすいように、いろいろとやってきました。」
と述べました。この例としてUrlockerは最新のQuery Browserと MySQL 
Administratorを
示しました。どちらもMySQL 4.1用にダウンロードできます。

MySQL Query Browserはビジュアルなデータベースのクエリツール
でグラフィカルな環境に適します。これを使用するとMySQLデーターベース
に格納されている情報を容易に、プロダクティブに
アクセスまたは解析できます。

MySQL Query Browser はウエブのような環境で統合された
ドラッグ・アンド・ドロップを使用するツールからなります。
ユーザ・フレンドリーの機能は:

    * Query ツールバーとDatabase エクスプロアーでクエリの
ためにテーブルとフィールドを簡単に選択できます。
    * テーブル・エディターでデータベーステーブルをビジュアル
に作成、変更、削除が容易にできます。
    * ユーザが複数のクエリを比較・管理できる結果ウインドウ
    * オブジェクト・ブラウザーはデータベースとブックマークを
ユーザが管理しやすいようにします。
    * スクリプト・エディターは大きなSQLのスクリプトを作成、
エディティング、デバッグする強健なインターフェースを
提供します。エディターはシンタクスのハイライトとSQLスクリプト
のデバッグを容易にするブレークポイントの設定なども行います。

MySQL AdministratorはDBAにビジュアルなコンソールを提供します。
このコンソールを使えば完全なMySQLの環境を容易に管理・モニターできます。
新しい MySQL Administrator は初めてMySQLの設計者、開発者や
その他のユーザに分かりやすいグラフィカルなUIを提供し、専門家
でない人々にも専門家のようにデータベースのオペレーションを行う
ことができるようにします。そのうちのいつくかは：

   1. サーバーのコンフィギュレーション,
   2. ユーザ管理,
   3. リプリケーション状況のモニタリング,
   4. バックアップとリストアー
   5. ログのビューなどです。

約1年前にEricssonから取得されたMySQL Clusterの技術はミッション
クリティカルなアプリに99.999%のハイアベイラビリティを
保障するものです。MySQL はこの技術に自信を持っているので
99.999%の保障を年にダウンタイムが5分以下であると定義
しています。これには保守の時間も含みます。[Ericssonが
所有していたときはこの技術はオープンではありませんでしたが、
MySQLが獲得して、オープンソースになったとUrlockerが
付け加えました。]

UrlockerはMySQL Cluster の機能は当初からエンタープライズで
使用可能なデータベースを想定して設計されており、オープンソース
版を適当に作り上げたということではないと述べています。 Urlockerは
続けて「私達は実際のハイアベイラビリティを必要とするデータベース
のニーズを重要視して、オープンソースでもそうでないソフトでも違いはありません。」 


MySQL Cluster の機能は:

    * リプリケーションができるクラスターとチェックポイントと
ログの実行をとおして99.999%のアベイラビリティを提供。
    * メインメモリをクラスターしたDBアーキテクチャを利用
した高速の自動フェイル・オーバー。
    * クラスター内でのプロセスの共有
    * 「Single Point of Failure」を避けるサーバー
の並列分散アーキテクチャ(データベース・ノード、管理サーバ
ノードとアプリケーション・ノード）

新しいMySQL Query BrowserとMySQL Administratorは
http://dev.mysql.com/　からダウンロードできます。

MySQL Clusterの詳細は
http://dev.mysql.com/doc/mysql/en/ndbcluster.html。

MySQL 5.0への準備

Urlockerはこれだけの追加では終わらずにまだ2005年の初め
にもっと追加があると言います。「MySQL 5.0に対する期待感
を感じています。特にstored procedureに対するサポートです。
そのほかアップデートができるビューも含めたビュー、簡単な
トリガーなどです。SAPはMySQLの長年のサポーターであり、
インベスターでもありますが、MySQL5.0ををエンタープライズ
環境に適するものとすることで、共同戦線を張っています。」と
Urlockerは続けました。

stored procedure (とそれに付随した機能）はMySQL 5.0で
一番特記に値するものですが、その他にも追加またはテストされる
予定のものがあります。
  o MySQL 5.0のもっとも最新のアップグレード予定の機能は
ここから。
http://dev.mysql.com/doc/mysql/en/mysql-5-0-nutshell.html
  o 最新の無料版のMySQL 5.0は
http://dev.mysql.com/downloads/mysql/5.0.html
  o 2005年のユーザコンファレンスの情報は（2005年4月18日より）
http://www.mysqluc.com/

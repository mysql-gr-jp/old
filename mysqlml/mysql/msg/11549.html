自己レスです。
cp932 の方も確認してみました。

1.MySQL Server
  (1) MySQL 4.1.12
      MySQL 4.1.1 では使えません。
  (2) configure
      ./configure --with-charset=cp932 --with-extra-charsets=complex
  (3) table
      mysql コマンドで適当に作成します。

2.ODBC
  (1) 取得元・バージョン
      http://download.softagency.net/MySQL/Downloads/MyODBC3/MyODBC-3.51.11-2-win.msi
  (2) connect options
      Initial Statement : SET CHARACTER SET cp932

3.Access2000
  作成したテーブルを odbc 経由でリンクします。

これで Windows の機種依存文字が使えます。
当然、cp932 ですので、Office2000 以上、あるいは Windows XP 以上で普通に使える
(Microsoft の) Unicode でのみ扱える文字は使えませんけど・・・。

そういえば MySQL 4.1.1 を使われてるんでしたね・・・。
前メールで書いた ujis と sjis の変換もきちんとしてくれるんでしょうか。
(MySQL 4.1.12 では可能)
どなたかご存知でしたら情報をください。

"Re: MySQL へ接続できません"のみにして返信
してください。

それから、以下はMySQLへの接続ができていない
ものとして書きますので、できていたら無視してくだ
さい。

①mydbにテーブルありますか？無ければ作成して
ください。
例）
->create table test
->(no int,name varchar(20));

②spencerにmysqlへの接続権限を与えましたか？
例）
->grant select on mydb.test to spencer@localhost;
注）grantはデフォルトではrootのみが使用できます。

③権限を与えた後、->flush privileges;　をやりました
か？まだなら、接続権限が有効になってません。やっ
てみてください。

ここまでやっても接続できない場合、考えられるのは
・PHPの書き方に誤りがある
・MySQLの初期設定の問題（TurbolinuxのCD版では
MySQLの初期設定が、PHPからの接続ができない状
態になっていました。）
・PHPかMySQLのインストール時の設定ミス
・なんらかの理由によるインストール自体の失敗
といったようなことが考えられると思います。

お世話になっております。

先述の、
mysql -e "command;" | more
これは正確ではなくて、
実際の記述は、
mysql -u root -p -e "command;" | more
です。
もっと白状しますと、
mysql -u root -p -e "show variables;" | more
mysql -u root -p -e "show columns form table_priv;" | more
この２つを実行しました。
上の例から分かる通り、パスワード認証の後、処理が実行されます。
# パスワード認証とmoreの相性の問題でしょうか？
# 素人考えですが。。。

ちなみに、Dosプロンプト経由でなくても、
直接サーバマシン上でKtermを起動しても
同様な現象が起きるのを先程確認しました。

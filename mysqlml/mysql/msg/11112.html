MySQL Network が発表されました。コミュニティ・リレーション・マネージャー
のArjen Lentzが開発者も含むコミュニティにとってMySQL Network
がどういう意味を持つか説明します。

この記事はMySQL Networkについてです。これは売らんかなの記事
ではなく、マーケティングのメッセージは入っていません。強いていえば
「Arjen流」で書いています。バズワードを使うのは他で使用されて
る場合の意味をはっきりとさせるためだけです。私の仕事は売ることでは
ありません。私の仕事は聞き、情報を流し、質問をすることです。

MySQLには大きくて様々なコミュニティがあります。そのことは大変
重要です。MySQL Networkは大きなイベントです。ユーザとしての
あなたは詳細が気になるでしょう、またこれが持つ意味も知りたいので
はないでしょうか。主なポイントについて話ます。しかし
多分もっと質問があるでしょうから、それにはいつでもお答え
します。

さてこれは一体なんなんでしょう。MySQL ABは幾つもの製品やサービスを
販売しています。それはコマーシャル（GPLでない）ライセンス、サポート、
コンサルティングなどです。簡単に言えばMySQL Network はそれらを統合して
年契約のサブスクリープション・サービスにします。今あげた他にも
２－３新しいものも含まれています。

まず最初に含まれているものから見ましょう。大事なことは
何も除かれていないということです。今までのとおり最新のMySQL
の版をダウンロードできます。そして、これが大切なのですが、コード
ベースは１つです。それでは何が無料で入手できるのでしょうか。

    *　以前無料であったもの、将来のMySQLのサーバーの版も含めて。
    * GPL ライセンスで提供されるMySQL サーバーとそれに付随する
ソフトやツール
    *　最新の開発版および製品版
    *　早期のリリースと頻度の高いリリース
    *　使い物にならない版はない。
    *　 MySQL コミュニティ版
    *　MySQLとその他による莫大な量のリソース

何も除かれておらず、ライセンスの形態も変わりません。コード
がフォークすることもありません。MySQLサーバーはいつも１つの
コードベースでヂュアル・ライセンスで無料版はGPLライセンスに
よります。こういったことは我々には基本的なことで、MySQLの
ファウンダー達(Monty Wideniusと David Axmark) が内部で新しい
製品やサービスを考えるときは常にこのベースをチェックしています。
ソフトウエアー・エンジニアリング担当副社長の Maurizio Gianolaは
「私達の仕事の80%以上は無料のサーバーのサポートです。」と
述べています。

MySQL Network

MySQL Networkは上で述べた無料サービス版の機能の他の機能も含んでいます。
主なものに触れてみましょう。

MySQL プロサーティファイド・サーバー
これはMySQLによってコンパイルされたサーバーのバイナリーです。
Red Hat Linux, Novell　SuSE Linux, Windows, HP-UX, Solarisと
そのほかのプラットフォーム用にそれぞれ特別なバイナリー
があります。もっと詳細に特別なビルド・オプション
やオプティマイゼーションを指定できます。これはビルドの
オプションのみでソースコードや機能は全く同じです。
ご存知のように、バイナリーは必ず我々が開発したテスト・スイート
でテストしています。その上これらのバイナリはその他のツール
によってもテストされます。これにはコマーシャルのツールの
Klocworkや Coverityも含みます。また良く知られているセキュリティの
攻撃やそのほかの攻撃に対してのテストも行います。これを全部
使ってテストした後で、サーティファイドに印を押します。エンタプライズの
カスタマにはこれくらいのプロセスが必要です。実際、多くの
エンタプライズのカスタマはこういうレベルの要求するような
ポリシーを設定しています。

エンタプライズのカスタマは不確定なことを嫌います。ですから
こういうカスタマはMySQLの開発者のSanja Byelkinが先週
開発したクールな新しい機能には興味がありません。選ぶ版
は安定したもので、一旦選択するとかなり長い間同じ
版を使います。クリティカルなバグやセキュリティの
フィックスに関してはうるさく言いますが新しいものを
混合することには異議を唱えます。

当然MySQL Networkをこういった会社（大抵の場合かなり大きい）
に販売しようとします。そのために特別の仕事をして、お金を
貰います。驚かれるかもしれませんが。サーティファイド
されたバイナリーはGPLでライセンスされています。これは
問題ありません。この場合カスタマーはエンドユーザですから
再配布することはありません。しかし、カスタマが望むのであれば
コマーシャル・ライセンスのバイナリーを無料で配布します。

MySQL アドバイザー
アップデート、テクニカルやセキュリティに関する通知サービス
があります。オンラインでこの情報を入手できますが、MySQL Network
はメール、SMSやページャーなどででも提供します。その他
カスタマーごとにカスタマイズできます。詳細は
http://www.mysql.com/network/advisors.html

MySQL 知識ベース
これはDBAや開発者が興味をもちそうな記事のコレクションです。
綺麗に格納されているので、検索できます。なにも差し引かれていなくて
全ての情報は他の場所から入手可能です。例えば、マニュアル、
記事、本やその他です。

MySQL アドバイザー同様、違いは焦点と便利さです。

本と比較するとこれの便利さが良く分かります。MySQLのレファレンス
マニュアルをオンラインで閲覧できます。それでもユーザはPaul DuBois
の素晴らしいMySQLの本を買いに走ります。それは説明の仕方や
情報の提供の仕方が好きだからです。

MySQLテクイカル・サポート
MySQLのサポートチームから直接サポートされます。問題があった場合
メールや様々な電話によるサポートで30分以内にリスポンス。
これは1週間7日24時間でサポートされます。基本的には前のサポートの内容を
新しいサブスクリプションのパッケージに移しかえたものです。しかし、
前は銀・金・プラチナムレベルのサポートを買う には最低限
のサーバーの数が必要でした。そのため小さな会社はこのサービス
が高すぎました。今このMySQL Networkでは最小限がありません。
もし１つのサーバーにプラチナム・レベルのサポートを買いたければ
買うことができます。$595からすきなサービスを選べます。

免責

MySQL Networkの金とプラチナム・レベルではオプションとして
IP(知的財産)の免責を提供します。カスタマーによってはこういった
免責を求めることがあります。

他にもいろいろとあります。詳細はMySQL Networkのページ
をご参照ください。
http://www.mysql.com/network/

Q&A

Q: MySQL NetworkはRed Hat EnterpriseやFedora見たいなものですか。

A: いいえ、 MySQLは１つのコード・ベースです。

サーティファイド・サーバーにこれだけテストするということは
「無料」サーバーは不安定でバグが多いということでしょうか。
もちろん違います。全てのMySQLのリリースは私達の標準テスト・スイート
によるテストをパスする必要があります。その上に行うテスト
は単に更にユーザに対する信頼性を高めるだけです。この特別余剰
のテストでセキュリティの問題やバグが発見されたら、修理
されます。コードベースは１つですから、次にダウンロードする
次期の版にこのフィックスが反映されます。最新の版は(4.1.10)
は KlocworkとCoverityのテストで修理されたフィックスを
含んでいます。もちろん、エンタプライズのユーザーはあなた
をはじめ、何百万というMySQLのユーザから恩恵を受けます。
バグが報告されると、全ての版に関して自動的に修理されていることになります。
これは非常に便利なことだと思います。

またよくある質問は、もしこれだけの人々を抱えて、
ツールや場所もあるし、無料で配布しても問題ないのでは
ないかということです。いやいや、そうではありません。現実に
そぐいません。これが出来るのはお金を払ってくださる
お客様がいらっしゃるからできるのです。

Dellからコンピュータを買う場合と比較してみましょう。
これにウエブ・ベースのサポートが含まれているとしましょう。
24x7 の電話サポートや次の日のオンサイトサービスが必要
とします。それを受けるには余計に代金を支払います。開発者、エンジニア
やサポートは魔法のように突然出現しません。

Q: ライセンスはどうですか。

実際MySQL Networkはこれをもっと簡単にします。

エンドユーザー

エンドユーザであればGPLライセンスのサーバーを使いコマーシャル
ライセンスを必要としません。これはあなたがISPでも電子コマース
のサイトでサービスを提供していても同じです。なんとなれば
あなたはコードを再配布していないからです。もし何かを
再配布するのであれば、下の開発者の項目を見てください。
MySQL Networkの恩恵を考えると年間のサブスクリプションを
考慮してください。もし、GPLのライセンスについて問題があれば、
コマーシャル・ライセンスを考えてください。MySQL Networkの
サブスクリプトを止めたくなれば、変更してGPLライセンスのサーバーを
使用できます。

開発者(含：ISV/OEM)

アプリを開発してMySQLのコードとリンクなさるのであれば、
以下を参照してください。

   1.　自社でのみこのアプリを使用していますか。

      →であればあなたはエンド・ユーザです。上記を参照してください。
   2.　あなたのアプリを配布していますか。(無料でも有料でも）GPLの
ライセンスですか。
      →GPLのライセンスであれば、GPL版のMySQLのコードとリンクして
も宜しい。これにはクライアントのライブラリーや組み込みサーバーを
含みます。あなたのアプリのエンドユーザはオプションとして
MySQL Networkをサブスクライブできます。

   3.　アプリを配布してクライアント・ライブラリーだけを
リンク(注: 組み込みサーバーでない)、GPLライセンスではないがMySQL
が認める他のオープン・ソースのライセンスのもの。（FLOSS
ライセンス例外を参照）
http://www.mysql.com/company/legal/licensing/foss-exception.html
      →上記２と同じ

   4.　アプリを配布。でもオープンソースではない。しかし、MySQL
のクライアントライブラリーとはリンクしたい。（注：組み込み
サーバーではない。）
      →MySQL Network サーティファイドのパートナーになって
http://solutions.mysql.com/program/
ください。年間2-3百ドル程度です。これになるとMySQL Networkに
アクセスできます。知識ベースへのアクセスも含みます。パートナー
として、我々のウエブサイトのパートナーの項目に載ります。あなたの
アプリはMySQLによって認定されているので、競合会社に対して
優位となります。

 それではあなたが必要なライセンスはどうでしょう。オプションで
パートナーであればコマーシャル版のMySQLのクライアント・ライブラリ
をリクエストすることができます。少し書類が必要となりますが、それ以外は
ロイアリティなどはありません。もちろん、MySQLのクライアント・ライブラリー
を含み配布するというライセンスはMySQL　Networkサーティファイド
パートナーでなければなりません。
      あなたのカスタマもMySQL Networkをサブスクライブしたいかもしれません。
もうそれぞれの個別のライセンスに関して気にかけることはありません。
これはMySQLをサポートしたいクローズ・ソースの開発者にとっては
非常に目新しいことです。上記で述べたことは非常に簡単です。前
の様なライセンスに関しての不安や不確実性もあります。

   5.　オープンソースでなくアプリを配布するが、MySQLの
組み込みサーバーとリンクしたい場合。
      →MySQL ABにコンタクトしてOEM用のコマーシャルライセンス
の契約書を求めてください。 MySQL Networkがあればあなた
の開発者は知識ベース、アドバイザーやサポートにアクセスできます。

   6.  ６番目のケースはありません。全部カバーしました。
もちろん、上記４で述べたようなケースでなくても、MySQL Network
サーティファイド・パートナーとなることもできます。かなり
お手ごろな価格でMySQL Networkの恩恵を享受できます。

ここでは「リンク」とか「配布」の定義には触れません。
GNU General Public ライセンスの文書と
http://www.fsf.org/licensing/licenses/gpl.html
FSF (Free Software Foundation, GPLのライセンスを
開発した人々）を参照してください。
(Frequently Asked Questions about the GNU GPL).
http://www.fsf.org/licensing/licenses/gpl-faq.html

弁護士は色々な意見を持っています。ライセンスに
関して質問があるが弁護士を雇いたくない場合は
自分で決定するか適当なフォーラム（(MySQL Licensing Forum)
http://forums.mysql.com/list.php?4
を参照してください。MySQLのファウンダーはこれを
どう思っているのでしょうか。

David Axmarkは先週メールで次のように彼のコメントを
発表しました。

「最初のMySQLのビジネスモデル（ヂュエル・ライセンス）
は組み込み市場では非常にうまく機能しました。もっと
大きな市場であるエンタープライズではうまく機能しません。
それでここ2年間ばかり、MySQLのマネージメント（Monty
と私を含む）はどうすれば企業ないのユーザ数を増加できる
か考えてきました。問題はコマーシャルのカスタマが
何を望んでいるかということです。余分なサービスや便利
さに対してお金を支払うか。それと同時に会社の設立
ベースであるフリーソフトやオープンソースのスピリット
を同時に保たなければなりません。MySQL Networkではとうとう
この質問に対する答えが見つかったと思います。

エンタープライズのユーザにカスタマイズした価値を
追加しているのです。自動サービスなどで提供しています。
同時に普通のGPLユーザから何も取り上げません。この
件に関しては社内で大いに議論して他社の失敗や
成功から学びました。

MySQL Networkは問題がおこる前に処理するサービスや
サーティファイド・カスタマイズされたバイナリやサポート
を提供します。アップデートやアドバイザーは非常に
エンタープライズのカスタマには重要です。私が会った
カスタマは全て興味を示しました。これは非常に大きな
古い会社から非常にハイテックな会社の区別はありませんでした。

あまりお金を持っていない、小さな会社にとっては
何も前と比べてなくなっていません。開発途上国
のユーザは前と同様全てを使えます。

もう１つ。MySQL Network に関して非常に気にいって
いることはコマーシャルライセンスを売るときに
良く使われるFUD (Fear, Uncertainty
 and Doubt)と取り除くことができるからです。企業でMySQL Community
版を無料で使用したい場合は、問題なくできます。MySQL Network
の余分な機能を使いたい場合は（サポート、サーティファイド
バイナリーやアドバイズや知識ベース）、お金を払います。

Montyは

「MySQL Network はエンタープライズのカスタマ向けです。こういった
カスタマの要求は自分でなんでもするというオープンソース
の方式では満足させられません。これは以前よりもっと
うることが多いサポートオプションです。」

結論

MySQL Networkを始める前にコミュニティの多くの
人々と話をしました。また同時に以前MySQLのライセンス
やビジネス・モデルに批判的な人々にも話をしました。
この記事はそういった議論に一部基づいています。これは
本音の答えを得るために素晴らしい（かなり
突っ込んだ）質問をしました。コミュニティから
の入力は非常に有益でした。MySQL Network　FAQには
もっと詳細があります。
http://www.mysql.com/network/faq.html

この記事でNySQL Networkをかなり良くわかって頂けたかと思います。
これは皆にとって良いことです。この新しいビジネスモデルで現在も
将来も開発に投資できる力を得ます。まだ質問がある方や
意見を述べたい方は、この記事にあるリンクから
フォーラムで意見を述べてください。
http://forums.mysql.com/list.php?95

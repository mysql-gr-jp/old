MySQLは5.0に到達、「十分パワフルですか」

MySQL 5.0はQ2にプロダクション・クオリティに達します。
メージャーな機能はstored procedures, triggersとviewsです。
競合のデータベースのベンダーにとってはMySQLのような
良い話がなくて頭が痛いでしょう。コマーシャル版のデータベース
ベンダーの大手は値段を再考慮する必要に迫られるでしょう。

Nancy Cohen著
2005年3月14日

神と極刑と毛皮の枕の話はいろいろと議論を呼びます。
データベースの設計者にリレーショナル
データベースの定義を尋ねて見てください。じゃっかん
血の雨が降るのが構わないのであれば、MySQLはリレーショナル
DBどころか、DBでさえないと主張してみてください。
大体話しの大筋はこうです。「笑わさないでよ。MySQLは
トランザクションDBじゃないよ。速いのは良いし、情報を
クエリできるし、まとめることもできる。でもそれだけじゃ
ないか。」  Stored procedures, viewsと他の機能がMySQLに
ないのがエンタープライズ市場に入り込めない理由だと
言われてきました。MySQLのマーケティング担当副社長のZack Urlocker
は「一般的にMySQLへの印象は前の版の印象のままです。」と
述べています。MySQLの無料版を使うユーザベースは500万人で
料金を払うコマーシャル版を使用するカスタマは5,000件位です。
MySQLはまだまだだという印象を持たれているからといって
MySQLの開発者は休まずMySQLの改善とツールに勤めています。

MySQL 5.0のリリースが近づいており（(Urlockerによれば
5.0 はQ2にプロダクション・リリースとのこと）、古い印象
はもはや時代遅れです。

Noel YuhannaはForrester Researchのデータベースのシニア
アナリストですが、明らかに市場が変わってきていることを
感じています。「MySQLの展開の殆どはまだミッション・クリティカルで
ないところですが、コマーシャルのDBMS ベンダーの
Oracle, MicrosoftやIBMにとっては圧力です。どうしてかって。
それはLinuxが世界中に広がったのと同じ理由です。エンタープライズ
はオープンソースのデータベースを使用することでコストの削減に気づき
始めました。」と語りました。

MySQLは昨年10月に4.1版をリリースしました。これには
エンタープライズへの展開を可能にする製品の改善が
含まれており、以下を含みます。

*簡単に効率良く複雑なデータセットを検索できるSubqueries とderived テーブル
*より速いクライアント・サーバープロトコルでprepared statementsの
サポート、最適化されたクエリ実行と改善された警告の情報。
*Linuxと Windowsの新しいGUIのインストラーとコンフィギュレーション・ウイザード 

これでMySQLデータベースのセットアップと最適化が簡単に。
*OpenSSL (GPL)を使った暗号化されたクライアント・サーバー交信。
許可されないアクセスに備えるため。
*更に最適化され、使用メモリーを削減し、サードパーティの
ソフトに組み込まれた際により高速となったサーバー・ライブラリ
*ローカル言語をサポートするためのUnicode サポート
*地理的データを格納するための業界標準のGIS spatialタイプのサポート
*全文検索とヘルプ機能

MySQLの開発者はエンタープライズにMySQLデータベースを
更に展開するために他の戦略も検討しています。

昨年の8月にMySQLはIBMのPOWERベースのコンピュータ
上のLinuxへのサポートを発表しました。MySQL ABの
戦略家達はエンタープライズのIBM Power5アーキテクチャへ
の興味へ反応したのです。このアーキテクチャは同一のマシンで複数のOS
をサポートするロジカルなパティション（32と64ビット環境をサポート）
をサポートします。

そのほかにMySQL の戦略は昨年12月の新しいツールの発表です。
 MySQL Query　BrowserとMySQL Administratorです。
Query　Browserの結果ウインドウを使うと複数のクエリを
管理および比較できます。 MySQL Administrator は
ビジュアルなコンソールでユーザ管理、backup/restore、ログの閲覧
などをサポートします。

Yuhanna はエンタープライズはスタッフの数を削減するため
と同時に複雑なデータベースを管理しやすすくする
ツールを求めていると述べました。続けて「MySQL のツールは
エンタープライズには大変重要です。特にOracle, DB2, or SQL Server
からマイグレートするユーザには。これらのデータベースに
はツールが揃っていますから。」と語りました。

しかし、なんと言っても一番大きいのはMySQL 5.0です。
新しい機能はこれがなかったためにMySQLがデータベースの機能比較で上位に
位置できなかったものです。つまり、Triggers, Stored proceduresとViews
です。Urlockerによれば、リリースは「MySQL 5.0は一月には
ベータです。プロダクションはQ2の予定です。」

Sybase, IBM, Microsoftや Oracle は心配するべきでしょうか。Yuhanna
は「そりゃそうです。オープンソースのデータベースはコマーシャル
のエンタープライズにとって代わりはしませんが、シェアーは奪う
でしょうから。」と述べました。

Yuhannaの予想の続きで、彼は全部のオープンソースのデータベース
は全体としてコマーシャルのベンダーの価格を12-24ヶ月で
低下させるとのことです。

現在オープンソースのデータベースのインパクトはローエンド
に限られていますが3-5年後にはミッドからハイエンド
へ達するでしょう。現在のオープンソースのデータベース
の市場は（ライセンス、サポートやサービス）は$120M程度
ですが、2008年には$1Bになると予想されます。

Gartnerの2003年のデータによればIBM, Oracleと Microsoft
はRDBMS の市場を支配しています。3社合わせると87%のシェアです。
確かにMySQL 5.0はMySQL ABにとってはキーとなるリリース
でエンタプライズ市場に押し上げますが、YuhannaはMySQL
だけが唯一のオープンソースのデータベース会社でないと言います。
そして「MySQLは他の製品に比べると新しいDBMSの機能を提供
する速度が遅すぎました。これは改めなければなりません。MySQL
はIngresやPostgreSQLなどからチャレンジを受けてきました。これに
対応するためには新しい機能を提供する頻度を上げなければなりません。
そうでなければリーダーの地位を保てません。」と続けました。

Yuhanna が忠告するようにMySQL AB はMySQL Cluster をアプリケーション
を99.999%のアベイラビリティで提供する手法としてプロモートしています。
テレコムの範疇を超えての適用は良いことです。 MySQL ABのある
マネージャはこの製品がたくさんのアプリに対するハイアベイラビリティ
を提供すると考えたいようです。

PostgreSQLの8.0がリリースされました。MySQLと同様注目を
集めたいでしょう。PostgreSQL はコマーシャル・データベース
にある大部分の機能を網羅していることを強調しています。
例えば、inheritance, rulesそして複数版の concurrencyコントロールなど
です。また複雑なクエリやread/writeクエリ負荷に対しては
MySQLより速いと語っています。

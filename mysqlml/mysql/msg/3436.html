Subject: [mysql 03436] root ユーザでアクセスできるようにする方法は？

長谷＠八戸と申します。

RedHat 7にMySQLのパッケージを導入して、フロントエンドの'mysql'
を使用しユーザ設定と新規Database、tableの作成を行おうとしていますが、
うまくいきません。

>mysql -uroot

とすると

ERROR 1045: Access denied for user: 'root@localhost' (Using password: NO)
[estate estate]$

このようなエラーで動作しません。

>mysql

とユーザを指定しない場合は

Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 10 to server version: 3.22.32

Type 'help' for help.

mysql>

となり一応起動はするのですが

mysql> use mysql

とmysqlデータベースを指定すると

ERROR 1044: Access denied for user: '@localhost' to database 'mysql'
mysql>

となりやはり操作できません。そこでrootパスワードに変な物が
入っているのかとも思い、DBを初期化すれば良いのかなと
'mysql_install_db'を実行しましたが、

>mysql_install_db
010303 19:28:09  /usr/sbin/mysqld: Can't create/write to file '/var/lib/mysql/ww
w.pid' (Errcode: 13)

To start mysqld at boot time you have to copy support-files/mysql.server
to the right place for your system

PLEASE REMEMBER TO SET A PASSWORD FOR THE MySQL root USER !
This is done with:
/usr/bin/mysqladmin -u root password 'new-password'
See the manual for more instructions.

このようになってやはり操作できません。
どのようにしたらマニュアル通り'root'ユーザで操作できるよう
になるのでしょうか？

何か初歩的なところで間違っているのかもしれないのですが、
何処から調べて行けばよいのか見当がつきません。
どなたか取っかかりをご教授くださればと思います。

こちらには久しぶりになります。

> ACCESSにテーブルをおくのはやめて、すべてデータはMySQL側におく
> ACCESSではMySQLのテーブルをOBDC接続して参照するようにする。
> MySQLからACCESSの方向では参照しないという作りにするのがよいような。

同感です。テーブルを MySQL に持っていくだけで速度改善が期待
できますし。

> しかし、フロントエンドがACCESSならばマイクロソフトのSQL SERVERとか
> 無料でダウンロードできる同社のMSDEなんかを使った方が無難なような
> 気がします。

私ならばもう一歩進んで、フロントエンドに Access を使うのを
やめた方が良いかと。

ちょっと本を読めば、Visual Basic で簡単にフロントエンドアプ
リケーションができます。

あと、工数や時間の制限がきついようであれば、Web インターフェ
ースは JDBC を使って JAVA Servlet か JSP にした方がいいと思い
ます。ボーランドの JBiluber で楽に開発できます。

セキュリティ

SEC 1. MySQL 5.0はSSLをネイティブでサポートしますか？
はい。ほとんどの5.0バイナリ版で、クライアント/サーバー間のSSLで保護/認証
され接続をサポートしています。新しいYaSSLライブラリについては非常に新し
く、全てのプラットフォームでコンパイルできないため、まだビルドすることが
できません。詳細はこちらをご覧下さい
http://dev.mysql.com/doc/mysql/en/secure-connections.html 。

クライアントアプリケーションがSSL接続をサポートしていない場合などには、
SSHトンネルを使用した接続もできます。この例についてはこちらをご覧下さい
http://dev.mysql.com/doc/mysql/en/windows-and-ssh.html 。

SEC 2. SSLサポートはバイナリ版に含まれますか、それとも再度コンパイルする
必要がありますか？
全ての5.0バイナリ版は、クライアント/サーバー間のSSLで保護/認証された接続
をサポートしています。詳細はこちらをご覧下さい
http://dev.mysql.com/doc/mysql/en/secure-connections.html 。

SEC 3. MySQL 5.0はLDAPディレクトリを使用した認証機能を備えていますか？
いいえ。外部認証方式のサポートは"rolling feature"としてMySQLのロードマッ
プでは位置づけられています。つまり、主要な機能ではないが将来的に時間ある
ときに実装されるといったものです。特定のお客様からの要望により、このスケ
ジュールが変わることもあります。

SEC 4. MySQL 5.0はRBAC（Roles Based Access Control）備えていますか？
いいえ。ロールは"rolling feature"としてMySQLのロードマップでは位置づけら
れています。つまり、主要な機能ではないが将来的に時間あるときに実装される
といったものです。特定のお客様からの要望により、このスケジュールが変わる
こともあります。

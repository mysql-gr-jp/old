　初めて質問をさせていただきます。
　kouki＠cinomixと申します。

　MySQL 3.23.23 を Solaris 2.6(sparc) 上で使用しています。
　かつて稼動させていたのですが、トラブルがありまして環境を作り直してみよ
うかと思っておりまして、いきなり壁にぶち当たってしまいました。

　ひとまず、/usr/local/mysql-data の中にあるデータを一度保管した後に、こ
の中のデータをすべて空にして、mysql_install_db を実行後、mysqld を再起動
させました。

　このあと mysql データベース内の user テーブルで権限の変更とユーザーの追
加を行った後に、root で


cinomix# /usr/local/bin/mysqladmin -p create kpriv
Enter password:********


　とやってみました。この結果として、何のエラーもメッセージも表示されなかっ
たのですが、続けてこのデータベースを起動しようとすると、


cinomix# /usr/local/bin/mysql -p kpriv
Enter password:********
ERROR 1049: Unknown database 'kpriv'


　となって、データベースが起動できませんでした。ためしにmysqlコンソール上
から"SHOW DATABASES" を試してみたのですが、やはりこのデータベースがなく、
また /usr/local/mysql-data にも kpriv というディレクトリが見当たりません。

　これって何が原因であるかわかる方、対処方法かもしくは参照すべきドキュメ
ントがありましたら教えていただきたいのです。

# 3.23日本語版のドキュメントサイトを読んでるのですが、何せまだ該当する
# もしくは関連しそうな項目に行き着いておりません (^_^;

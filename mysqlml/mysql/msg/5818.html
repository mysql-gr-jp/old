お助け下さい。

以下 長文で失礼します。 

WindowsNT4.0 SP6a の 環境で Ver 3.23.49 を Dドライブ に
インストールしました。
自動起動させようとサービスに登録しましたが自動起動
できませんでした。
過去ログを参照したところ Cドライブ 以外にインストールした
場合は my.iniファイルの [WinMySQLAdmin] に Server=D:/mysql/bin/mysqld-nt.exe
と指定しなければならないようなので指定しました。

その後も自動起動はできませんし、
(サービスには登録されています。)
それどころか d:\mysql\bin\mysqld でも起動できません。

MySQLを起動するには winmysqladmin を起動し、
アイコンの右クリックでメニューを表示して
 Win NT→Remove The Service 
再びアイコンの右クリックでメニューを表示して
 Win NT→Start The Server Standalone 
を行うと MySQL が起動して使用できるようになります。

Ver 3.23.49 をアンインストールして Ver 3.23.51 を
インストールしても結果は同じでした。 

Cドライブの空き容量が少ないのでDドライブにインストール
したのですが、やはりCドライブにインストールしないと
いけないのでしょうか?
(ちなみに、CドライブにMySQLをインストールした別のパソコンでは、問題なく自動起動できています。)

初めてインストールした時に自動起動なんか必要ないと思い
 winmysqladmin.exe の Start The Server Standalone
で起動して使っていたのが原因かとも思っています。
(自動起動できている別のパソコンは最初から自動起動で使って
 います。)

解決策をご存じの方がいらっしゃれば、ご回答のほどよろしく
お願いいたします。

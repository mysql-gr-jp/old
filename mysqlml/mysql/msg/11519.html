>MyODBC3.51を使ってAccessからMySQLサーバーに接続しています。
>MySQLサーバーのバージョンが3.23や4.01のときは問題なかったのですが、
>4.1.1になったところvarcharやｔextのフィールドにマルチバイト文字が入らなくな
>ってしまいました。

最近MySQL4.1を本格的にテストしはじめたので、正確なところは書けま
せんが、MyODBCのConnectOptionsのInitial Statementに以下のような
キャラクタセットの指定を書くと、だいたいうまく行くと思います。

    SET CHARACTER SET SJIS

これはMySQL4.1からサーバ側に文字コードの自動変換機能が入ったため
に、latin1以外のクライアントは自分のキャラクタセットを明示する必
要があるからだろうと推測しています。

こんにちは。いけだです。

--defaults-fileを使用する場合には、先頭のオプションとして使う必要が
あったと思います。

○　./scripts/mysql_install_db --defaults-file=/etc/my5.cnf --user=mysql 
×　./scripts/mysql_install_db --user=mysql --defaults-file=/etc/my5.cnf

> YuGoさま
> 
> 有り難うございます。
> 
> ./scripts/mysql_install_db --user=mysql --defaults-file=/etc/my5.cnf
> 
> Installing MySQL system tables...
> 070601  9:34:57 [ERROR] ./bin/mysqld: unknown variable 'defaults-file\=\/etc\/my5.cnf'
> と--defaults-file=の設定が知らないよといわれています。
> 
> こちらでインストール
> ./scripts/mysql_install_db --user=mysql
> 
> ./bin/mysqld_safe --user=mysql --defaults-file=/etc/my5.cnf&
> として起動しようにも
> 
> Starting mysqld daemon with databases from /usr/local/mysql/data
> STOPPING server from pid file /usr/local/mysql/data/INTRA.FUJIWA-NET.COM.pid
> 070601 09:36:09  mysqld ended
> となり起動が出来ない状態です。
> 
> my5.cnfの抜粋
> [mysqld]
> basedir         = /usr/local/mysql
> datadir         = /usr/local/mysql/data
> pid-file        = /usr/local/mysql/data/mysql.pid
> port            = 23306
> socket          = /tmp/mysql.sock
> 
> これに設定しているpidファイル名も違うことから読んでいないと判断できそう
> です。
> 
> ソースからconfigureオプションで指定し構築したいと思います。
> 
> > my5.cnf[mysqld]に
> > basedir=
> > datadir=
> > pid-file=
> > port=
> > socket=
> > を古いやつとぶつからないように指定して、初期化からやってみてください。
> > 
> > #/usr/local/bin/mysql_install_db --user=mysql --defaults-file=/etc/my5.cnf
> > #/usr/local/bin/mysqld_safe --user=mysql --defaults-file=/etc/my5.cnf
> > 
>

Subject: [mysql 16587] ST_Union(g1, g2)などの空間データ処理関数の緯度経度系データへの対応について

いつもお世話になります。

SRIDが4612（測地成果2000の緯度経度）のg1, g2を使ってST_Union(g1, 
g2)を実行したところ、下記のエラーが発生しました。
Error Code: 3618 st_union(POLYGON, POLYGON) has not been implemented for 
geographic spatial reference systems.

なお、SRIDが2451（測地成果2000の平面直座標系）のg3, g4を使ってST_Union(g3, 
g4)を実行した場合は、エラーは発生せず、期待通りunionされたpolygonが得られました。

そこで、g1, g2のSRID(4612)をST_Transform(g, 
target_srid)で2451に変換しようとしたところ、以下のエラーが発生しました。
Error Code: 3742. Transformation to SRID 2451 is not supported.

MySQL8.0.19は緯度経度系のgeometryを取り込めるが、st_union()やST_Transform()は緯度経度系のgeometryはサポートしていないということでしょうか？
緯度経度系のgeometryに対してst_union()やST_Transform()を適用する方法についてお分かりの方がいましたら、教えてください。

実行環境：
OS:Windows10 Pro 64ビット
MySQL:8.0.19 Community Server(GPL)

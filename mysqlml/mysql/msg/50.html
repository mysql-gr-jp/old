こちらも make してみました。
民斗さんの日本語パッチをあてて、再び Linux + libc6 (Debian GNU/Linux)で
ためしてみました。
動きました。一応、bench テストも通りました。

ただ、configure でできる DEF の定義が多少あわないのと、
どうも libc5 と libc6 では、mysql のソースが見ている thread の
インクルードファイルが違うようなので、以下、修正しました。

1.configure で ncurses がないといわれるので、--with-named-* を指定して回避
2.コンパイル時に F_TO_EOF がないといわれるところは追加
  #define F_TO_EOF        0L
3.sigwait が linux thread の定義と重なっているので、はずす
4.各 Makefile の -static -lpthread を /usr/lib/libpthread.a に変更

場当たり対処なので、いいのかどうかはわからないのですが、ベンチは動きました。

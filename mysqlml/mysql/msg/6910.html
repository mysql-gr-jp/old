Subject: [mysql 06910] Linux 版インストールについて

こんばんわ。近藤と申します。
勉強をかねて、自宅にLinuxをインストールしました。
普段使用しているWindows2000マシンにはMySQLをインストール
しているのですが、このPCにMySQLをインストールしたいと思っています。
ところが、インストール途中でわからなくなりましたので、みなさんのお知恵を
貸して下さい。
ちなみに、インストール方法については、
○Soft Agency HPの実践MySQL
○MySQL徹底攻略ガイド(技術評論社)
を参考にしています。


現在迄の経緯です。

1.HPよりパッケージをダウンロードしました。
<DL先>
http://www.softagency.co.jp/MySQL/Downloads/MySQL-Max-3.23/mysql-max-3.23.55-pc-linux-i686.tar.gz

2.ダウンロードパッケージを解凍し、それを /usr/local/mysql に格納しました。
  (DLしたものは、既にバイナリ版のようで、make は必要ありませんでした。)

ここまでは問題ありませんでした。

この後、MySQLの初期化ということで、mysql_install_dbを実行しました。
<コマンドパス>
/usr/local/mysql/script/mysql_install_db

そこでこんなメッセージが表示されました。

メッセージ===================================================
./mysql_install_db: my_print_defaults: command not found 
Didn't find ./bin/mysqld
You should do a 'make install' before executing this script
=============================================================

どうも「./bin/mysqldがないよ」といわれているように思えます。
これがでてくるということは、環境の作り方がおかしいと思っています。

この後、InnoDBの使用設定を行おうと思ったのですが、my.cnf が見当たり
ません。徹底攻略ガイドによると、 /etc/my.cnf なのですがファイルが
存在しません。

次に起動確認のために、mysql.server start を実行してみましが、

Can't connect to local MySQL server through socket '/tmp/mysql.sock'(111)

とでてきました。「MySQLサーバーに繋げられなかったよ」といわれます。

おそらくわたしができているのは、「1.ダウンロード」「2.解凍」しかできていないと
思います。この後、どうしたらよいのでしょうか?

最終的な目標としては次のとおりです。
○DBD および InnoDBを使用できる環境にする。
○MySQLを常駐させる

みなさん、アドバイスをお願いします。

<環境>
CPU：Celeron 633MHz
Mem：128MB
OS ：Vine Linux 2.5CR1
インストール予定：MySQL-Max 3.23.55 (できれば4.0.9)

※Linux初めてまだ1ヶ月にも満たない初心者です。
  基本的なところが間だわかってないところもあるかと思いますが、よろしくお願いします。

Subject: [mysql 04783] Msql-Mysql-modules の make について

こんにちは、初心者の岩永です。


最近TURBO LINUX 6.1を導入しました。
そこでTURBO LINAXに入っている、MySQLを起動させました。
MySQLは上手く起動できたので、今度Perlを使ってMySQLを使おうと思い、
以下の3つをダウンロードしました。


DBI                     DBI-1.14.tar.gz
Data-ShowTable          Data-ShowTable-3.3.tar.gz
MySQLモジュール         Msql-Mysql-modules-1.2215.tar.gz



そしてインストールしようと思ったのですが、エラーで引っかかってしまいました。
DBIとData-ShowTableは上手くできたのですが、MySQLモジュールが上手くインストー
ルできません。


以下、MySQLモジュールのインストール手順
($:一般ユーザー   #:スーパーユーザー)


$ tar xvfz Msql-Mysql-modules-1.2215.tar.gz
$ cd Msql-Mysql-modules-1.2215
$ perl Makefile.PL
        いくつか質問されるので、初めは1を選択。
        後はデフォルト。


$ make


make: *** No targets specified and no makefile found.  Stop.



上記のように、makeをするとエラーになってしまいました。
このエラーは、make にパッチをあてて再コンパイルしていないか、
古いバージョンの make がシステムのどこかに残っているということでしょうか？



$ make install
をすると
make: *** No rule to make target `install'.  Stop.


というエラーが出ます・・



以前UNIXにインストールしたときは上手くいったのですが・・
以前のmakeのversion     GNU Make 3.79.1
今回のmakeのversion     GNU Make 3.79


どなたかおわかりでしたら、教えてください。
よろしくお願いします。



環境
TURBO LINUX 6.1
MySQL 3.22.32
perl 5.005_03
GNU Make 3.79

とみた＠今日は品川 です。

PHP は詳しくないので外しているかもしれませんが、エラーを見る限りしょうが
ないような気もします。

Apache にリンクしようとしている PHP のライブラリは modules/php3/libphp3.a
となってるので、スタティックライブラリですよね。

スタティックライブラリには、そのライブラリが必要としている別のライブラリの
情報を持つことができないので、そのライブラリのリンク時に指定する必要が
あるのですが、Apache は libphp3.a をリンクするときに libmysqlclient が
必要であることは知りようがないので…。

libphp3.a ではなくて共有ライブラリを指定することができれば、うまくいく
ような気がします。

または、PHP 自体を Apache の動的モジュールにするように、PHP の configure 
時に --with-apache ではなく --with-apxs でやってみてはいかがでしょう？

# Apache に mod_so が入ってないとダメかな…。

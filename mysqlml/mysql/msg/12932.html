honmaさま wrote...
> hanbaiテーブルと nyukinテーブルがありまして、
> hanbaiテーブルには
> 販売日、商品メーカー、商品名、規格、個数、単価、消費税、納品日
> 
> nyukinテーブルには
> 現金入金日、入金金額、ローン会社、ローン承認日、ローン入金日

> とありましてこれを時系列で並べ替えたいのですが日付に関する
> フィールドが複数しかも多テーブルに渡って存在していてどうして
> いいかわかりません。
何をどのように時系列に並べたいのかわからないので想像を含んでしまいますが。

以下のようなSQLを作ったらいかがでしょうか。

select 販売日 as 基準日 from hanbai
union all
select 納品日 as 基準日 from hanbai
union all
select 現金入金日 as 基準日 from nyukin
union all
select ローン承認日 as 基準日 from nyukin
union all
select ローン入金日 as 基準日 from nyukin
order by 基準日

必要に応じて、見せたい項目をselect句に書いてくださいませ。

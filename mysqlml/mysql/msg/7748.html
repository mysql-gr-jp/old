金宏と申します。

> 自宅と職場での環境はソフトのバージョンも、OSも全て同じで、
> Fドライブ直下に置いた『data.txt』ファイルをテーブルにインポートしたいのです
> が、
>   load data infile 'F:\data.txt' into table register_datas
> 
> というSQL文を発行したところ、自宅環境では問題なくインポートができたのに
> 職場の環境では、
> 
> 　ERROR 1105: File 'F:data.txt' not found(Errcode 2)
load data文では'F:\data.txt'となっており，エラーメッセージでは，
'F:data.txt'となっています。この違いは認識しておられますか？
試しにload data infile 'F:\\data.txt'としてみて下さい。

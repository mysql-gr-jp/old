ActivePerlの場合、PPMというモジュール管理のためのモジュール＋スクリプトが
ついてきますから、インターネットにつながっているマシンであれば、
ActivePerlをインストールした後、DOS窓から

ppm install DBI
ppm install DBD-mysql

とすると、ビルド済みのモジュールがインストールされます。
これだけでPerl側は準備万端なはず、後は自分のスクリプトを書きましょう。

PPMについての詳細は
Using PPM to install modules
  http://velocity.activestate.com/docs/ActivePerl/faq/ActivePerl-faq2.html
をどうぞ。
とりあえず、こんなのとかもありますが
  http://member.nifty.ne.jp/hippo2000/perltips/ppm.htm

またDBIを使ったことがあるとのことですが、利用するDBDによって機能が
違ってくることがあります。データベース毎のクセみたいなものがありますから。
いずれにせよDBD::mysqlのドキュメントは一通り目を通しておいた方がよい
でしょう。PPMでインストールすると
C:\Perl\html\site\lib\DBD\mysql.htmlにHTML形式のドキュメントが入ります。
（c:\perlは実際にはPerlがインストールされたディレクトリ）
DOS窓でperldoc DBD::mysqlとしても表示されたりもします。

とりあえず、こんなんもありますが...
  http://member.nifty.ne.jp/hippo2000/perltips/DBD/mysql.htm

可能性があること、インストールされたモジュールに対応するのは
一緒に入ってくるドキュメントであることから英語の方を重視する
べきだと思っています。

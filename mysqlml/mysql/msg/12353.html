Subject: [mysql 12353] エラーメッセージ中のメタデータはUTF8

藤原です

mysqlクライアントで、メタデータに日本語を使用してエラーが出たとき、エ
ラーメッセージ中のメタデータ部分は、設定しているキャラクタセットに関係
なくUTF8になってしまうようです。
VERSION: 5.0.15-standard-log

たとえば、'倉庫'というデータベースがない状態で、

mysql> SET NAMES utf8;
Query OK, 0 rows affected (0.00 sec)

mysql> use `倉庫`;
ERROR 1049 (42000): Unknown database '倉庫'
mysql>

という感じに、ちゃんとエラーメッセージも読めるんですが、


mysql> SET NAMES eucjpms;
Query OK, 0 rows affected (0.00 sec)
 
mysql> use `倉庫`;
ERROR 1049 (42000): Unknown database '?綺?'
mysql>

としてしまうと、文字化けが発生してしまいます。

このバグ、既に出ているんでしたっけ？

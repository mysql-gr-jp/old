myodbc euc のテーブル名が日本語の件ですが、

> 
> テーブル名が日本語だと、受け取り(SELECT) はできて、
> DELETE, UPDATE, INSERT ができない、ってことでしょうか？

おそらく、SELECT と INSERT はうまくできます。

　　　　SELECT -> テーブルの(内容)表示ができたことで確認
　　　　INSERT -> レコードの追加ができたことで確認

DELETE と UPDATE はうまくいかないようです。

　　　　DELETE -> レコードの削除で警告メッセージが表示され動作完了しない
　　　　UPDATE -> 同様に警告メッセージ。動作完了しない 。この時の警告メッセージは...
　　　　　　　　　　「このレコードは他のユーザーによって変更されています。
　　　　　　　　　　　[レコードの保存]を選択すると、他のユーザーによる変更
　　　　　　　　　　　を無視し、自分が行った変更を反映します。
　　　　　　　　　　　[クリップボードにコピー]を選択すると、変更したデータは
　　　　　　　　　　　クリップボードにコピーされ、他のユーザーによる変更が
　　　　　　　　　　　反映されます。必要に応じて、クリップボードのデータを
　　　　　　　　　　　貼り付け、自分が変更したデータに戻すこともできます。」
　　　　　　　　　　と表示されます。この時、[レコードの保存]は選べない状態で、
　　　　　　　　　　結局目的の動作(UPDATE)を行えない。　　　　　　　　　　　

> テーブル名、データベース名の SJIS->EUC は、まだ抜けがあると思います。
> これらは 英数でお願いします(EUC版)

了解いたしました。

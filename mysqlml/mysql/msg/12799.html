田畑さん、山村さんありがとうございます。

1Byte文字のみなら、検索できるようなので、
前提で、すべて２Byte文字だとして、その文字を全部16進表記に
換えてみました。
例えば：× → 817e
それで、
select count(*) from DTtest where match(textdata) against('817e');
と行うと、ヒットされず
select count(*) from DTtest where textdata like '%817e%';
と行うと、ヒットされます。

全部1byte文字にし（今回は数字・アルファベット）すれば、
検索可だと思っていたのですが、何か考え方が違っているのでしょうか？

とりあえず、フルテキストインデックスの勉強も兼ねて、今回はこの方法で
やってみようと思います。
他の検索エンジンが、利用可なのか、今回の案件はわからないので^^;;

よろしくお願いします。

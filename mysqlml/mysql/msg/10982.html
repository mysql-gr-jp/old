こんばんは

> WebDAVを使うとか他の手段を使うとかってことは置いておいて...

そりゃそうなんですが、ファイルサーバーなのに更に DB を使って
ファイルリストを作るということ自体に何か疑問が。ファイルを
更新したりするたびに、DB も自動更新しておかないと双方の内容
が合致しません。

まぁそれから検索対象ではないでしょうが、Windows では OS が
勝手に更新するファイルが沢山あります。

サーバーの負荷が大きくなるだけで、あまり良い方法ではないんで
はないのかな、と。

さて、方法ですが、namazu で用が済むならその方がいいかも。

もちろん Dir リストを BAT で取ってもいいでしょうし、Windows
Script というのもあります。

ただ私だったらこうします。

クライアントから検索要求があったら、検索プログラムをサーバー
で実行します。これは何も BAT とか WS ではなくて、何の言語で
もいいので、SendMessage で [F3} キーを押したことと同じ状態
を作り出して［ファイルとフォルダ検索］ダイアログを呼び出します。

あとはダイアログのテキストフィールドに検索データを突っ込んで
検索開始ボタンを押して(自動で)検索させて、結果をやはりフィール
ドから取り出します。ただ、この場合テキストボックス、リストボッ
クスなどの　ID が分からないといけないので、VB  とか VC++ を
使うのがベターかと思います。

> 海老原です。
>
> 僕の理解が正しければ、Repeatable Readというのは常にトランザクション開始時
> 点の状態を返すものではなく、一旦トランザクション内で読み込んだレコードを
> 再度読む場合に、最初の読み込みのときの状態が返される、というものです。
>
> >      term1                           | term2
> >     ---------------------------------+--------------------
> >      set autocommit=0;               |
> >                                      | set autocommit=0;	<--(1)
> >      select * from BBB;              |
> >      update BBB set ID=2 where ID=1; |
> >                                      | commit;		<--(2)
> >      commit;                         |
> >                                      | select * from BBB;	<--(3)
> >      select * from BBB;              |
> >
> > この場合のterm2を見てみると、
> > (1)でトランザクションが開始され、
> > (2)でトランザクションが終了されると同時に新しいトランザクションが開始
> > される。
> > よって(3)ではID=1と表示される、ということでしょうか。
>
> トランザクションにおける最初の読み込みですから、最新の状態、つまりID=2が
> 表示されるのではないでしょうか。

MySQL環境を作って早速テストしてみましたところ、
結果はID=2が得られました。
トランザクションの最初の読み込み（＝(3)）の時点のデータが得られたということ
ですね。

そこで疑問が出てきたので、もう１ケース実験をしてみました。
今度はテーブルを２つ用意しました。

 term1                           | term2
---------------------------------+--------------------
 set autocommit=0;               |
                                 | set autocommit=0;    <--(0)
 select * from AAA;              |                      <--(1-1)
 select * from BBB;              |                      <--(1-2)
 update BBB set ID=2 where ID=1; |
 commit;                         |
                                 | select * from AAA;	<--(2-1)
 update BBB set ID=3 where ID=2; |
 commit;                         |
                                 | select * from AAA;	<--(3-1)
                                 | select * from BBB;	<--(3-2)

テーブルAAAとBBBはIDというフィールド１つのみをもち、値＝１が双方ともセットさ
れているとします。
(1-1)の結果はID=1、(1-2)の結果もID=1です。
(2-1)の結果はID=1です。
(3-1)の結果はID=1、あたりまえですね。
さて、(3-2)の結果はID=2となりました。

ですから、
> 点の状態を返すものではなく、一旦トランザクション内で読み込んだレコードを
> 再度読む場合に、最初の読み込みのときの状態が返される、というものです。
「一旦トランザクション内で読み込んだレコードを再度読む場合」
ということではないようです。なぜなら、一旦トランザクション内で読み込んで無く
ても、「ある時点」におけるデータの読み取りができたからです。ここでいう「ある
時点」というのは(2-1)を指すといえるでしょう。

Repeatable Readは、「ある時点」におけるデータの読み取りを保証するものだとし
たとき、私は「ある時点」＝トランザクション開始時点だと考えていましたが、
MySQLの場合は
A．トランザクション開始時点＝(0)
B．「ある時点」＝(2-1) <--あるトランザクション内で最初にSQLが実行された時点
？
のようです。

このようなものなのでしょうか？
その他のDBMSなどについても調べてみたいと思いますが、
直感的に違和感を感じてしまったものですから…。

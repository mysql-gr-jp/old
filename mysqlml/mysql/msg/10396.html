異なるサーバへのmysql接続要求について

以前に投稿させていただいた浅倉と申します。
早速ですが、仮に
同じLAN上にあるAサーバのMySQLを、Bサーバで読み出す方法についてご教授願いま
す。
（長文ですみません。）

前提：Aサーバ、Bサーバともに、MYSQL 4.0.21-ntをインストール
ともに ユーザー='root' 、パスワード='空白'にて設定。

Aサーバの、mysqlコマンドプロンプトから、以下の2行を実行。

GRANT ALL PRIVILEGES ON *.* TO 'root'@'%'
FLUSH PRIVILEGES;

いったん QUIT、PCを再起動して、Aサーバのコマンドプロンプトから、以下を確認。

mysql --user=root --host=ホスト名(Aサーバの)
-->成功
mysql --user=root --host=IPアドレス(Aサーバの)
-->成功
次は Bサーバの、mysqlコマンドプロンプトから、以下を実行、確認。
mysql --user=root --host=ホスト名(Aサーバの)
-->失敗
mysql --user=root --host=IPアドレス(Aサーバの)
-->成功
となり、BサーバからAサーバのホスト名にての接続がうまくいきません。

LANがドメインにて管理されているのが気にはなるところですが、
どなたか原因及び対策法をご教授くださるよう、宜しくお願いいたします。

以下コマンドプロンプトの表示内容を記させていただきます。
↓---------------------------------*
C:\mysql\bin>mysql --user=root --host=ホスト名.ドメイン名
ERROR 2003: Can't connect to MySQL server on 'ホスト名.ドメイン名' (10060)
↑---------------------------------*

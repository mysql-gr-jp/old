いろいろ調べていると、以下のような事がわかり、
これは 3.22.15, 3.23 にて順次 fix する事となりました。

1.Pentium 上での話だが、DATETIMEをCHARに変換するルーチンが
　思ったほど効率が上がっていない。
　Pentium で 64bit演算を行わせると極度に結果が悪い。
　3.22.15でこれは fix する予定で、DATETIME 項を扱う場合、4-8倍速くなるはず。

2. 64 bit の int, string も少しもたついているかんがある。
　MySQLは現在、internal DATETIME type を持っていないが、
　3.23でこれを追加する。そうすれば、DATETIMEの比較において、
　2-3倍速くなるはず。

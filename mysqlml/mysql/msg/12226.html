MySQL はデータベースのIkea になりたい。
2005年10月7日
Matthew Aslett著 (Business Review)

MySQL ABの近日のMySQL 5.0 データベースのリリースはエンタプライズ
用の機能を含みOracle, IBMや Microsoftともっと互角に
戦うことができるようになります。MySQLのCEO はそれよりも
スエーデンの小売業のIkeaの成功に例えられる方に
興味があるようです。

MySQL CEOのMarten MickosはBusiness Reviewに
stored procedures,　triggersとviewsに付いて
語りました。「5.0版は明らかに最初で最後の
機能の同等性を語る場です。我々は競争するものの
数をセットするような人たちとは競争したくありませんし
興味もありません。」

新機能はスエーデンのUppsalaに本社があるデータベース管理
システムこの会社は今までのウエブ主体のマーケットからそれ
を超える市場にも進出できるようになります。Mickos は
それでも新しいウエブ・ベースのアプリケーションの
カスタマに興味があり、既にインストールされたデータベース
を置き換えるということにはあまり興味がないと語っています。

EnterpriseDB とPervasive SoftwareなどはOracleと競合しよう
としています。Mickosはそんな競争は他の人に任せておけば
良いとしています。

Mickosは「他のベンダーが市場を規定してくれることに
感謝しています。ベンダーが１つだと製品は良く規定
されません。Pervasiveと EnterpriseDBはOracleと競合
するつもりです。我々はそうしたくありません。
Oracleを怒らせたくありません。動物園にいたら、ライオン
にちょっかいを出すものにはなりたくないでしょう。」と
続けました。

Mickos は付け加えて、スカンジナビアの小売の大手の
Ikea、割引のRyanairやVirgin Mobile のビジネスから学びたいと言いました。
これはサービスやイノベーションに焦点を当てること
を強調しているからです。「ビジネスの真髄は新しいもの
を作ることです。新しい市場に出て行って我々の
製品の方が良いとはいえません。」

「シンプルであることは底が浅いということではありません。」
他の競合はセットできるデータベースのパラメーターが3,500
あることをひけらかすでしょう。MySQLはそれよりも35の
パラメータでエンドユーザから煩雑さを取り除く方が良いです。

2004年のショウでOracleの社長のCharles PhillipsはMySQLとを比べてこう
言いました。「どちらも同じ運送業についている。我々
は747であちらは、Toyotaだ。」こういった比較はMicosが好む
ところです。というのは747の販売台数とTpyotaを比較すると
比べ物にならないからです。Toyota は非常に良く利益の
出ている会社だからです。

MySQLがstored procedures, triggersとviewsをMySQL5.0に加えて
先月にrelease candidate版が出たばかりですが後数週間で
最終版がでます。Mickos は既存や将来のカスタマはこういった
ものよりも更に新しい機能に興味があると言っています。例えば
federated storage　engineです。

stored procedures やその他は長い間待ち焦がれていたものです。
本当に長い待ちだったとMicosは認めています。これは
しかたのないことでした。新しいエンタプライズ用の機能
がデータベースの性能を落とさないことに留意しました。

2004年１月にアルファ・フェーズに入り、その年の末に
ベータに入りました。GAのステージには2005年のQ1かQ2に
入る予定でした。

Mickos は出荷の時期を設定するのを誤ったと認めました。
しかし、それ以後もっと信頼できるマネージメントを導入し
て問題を解決しました。社員数も80から235にしました。
「私達はプランが甘かったんです。以前は予想が困難でした。」と
述べました。

5.0版がなかなか出ませんが、MySQLのビジネスはうまく行っています。
これは現在の版の4.1とMySQL Network サポートです。Q2は
会社１０年の中で一番業績が良かったです。「5.0が出るのが
遅れています。それでエンジニアにこれで、売り上げが落ちる
と言いましたが、実際には落ちず、あまり良い発破には
ならなかったです。」と冗談を言いました。

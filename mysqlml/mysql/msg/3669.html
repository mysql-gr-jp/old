On Wed, 23 May 2001 09:57:44 +0900
Hiroyuki Hasegawa <admin@xxxxxxxxxx> wrote:

> 非常に高負荷のかかるサーバにおいて、エラーが発生するため、
> mysql_error() 
> でメッセージを表示したところ、
> Can't connect to MySQL server 'IPアドレス'(11)
> といったエラーが頻発します。
> 
> (11)というのはmysqld_error.hの中のエラー番号と等価なんでしょうか・・？

多分、OS の errno だと思います。ちなみに手元の VineLinux では errno=11 は
Resource temporarily unavailable でした。

何のシステムコールで発生しているのかわかれば、解決の糸口になると思う
ので、システムコールトレースを取ってみてはいかがでしょうか。Linux では
strace, Solaris では truss コマンドで取ることができます。 

> あと、マニュアルにある
> shell> perror
> で調べるエラーコードは、mysql_errno()で吐き出されたエラー番号
> でいいのでしょうか。
> mysqld_error.hと一致する結果を返すのでしょうか。

perror は man で読めるマニュアルも用意されてるので、読んでみるのが
よろしいかと。

       perror  can  be used to display a description for a system
       error code, or an MyISAM/ISAM table  handler  error  code.
       The error messages are mostly system dependent.

…ということらしいです。

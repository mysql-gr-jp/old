MySQL とは何か？

MySQL は SQL (Structured Query Language) データベースサーバです。 SQL は世界でもっともポピュラーなデータベース言語です。MySQL は、 サーバデーモン mysqld と多くの様々なクライアントプログラム/ライブ ラリからなるクライアントサーバ実装です。

MySQL の主な目的は高速で頑丈なことです。

MySQL が作成される上での基本は、何年もの間、製品環境の高い要求 で使用されていたルーチンのセットです。MySQL はまだ開発中ですが、 豊富でとても有用な関数セットを既に提供しています。

MySQL の公式な発音は 'My Ess Que Ell [訳注:マイエスキューエル？]' (MY-SEQUEL ではない)です。

MySQL の歴史

我々自身の速くて低レベルな (ISAM) テーブルへ接続するために、 我々は一旦 mSQL の使用を開始しました。しかし、いくつかのテストの後、我々は mSQL は速くなく、そして我々に必要な十分な柔軟度を持っていないとの 結論に至りました。この結果、mSQL とほとんど同じ API インタフェー スを持つ新しい SQL データベースインタフェースに帰結しました。この API は サードパーティコードの移植を簡単にするために選択されました。

MySQL の名前の由来は完全には明らかではありません。我々のベース ディレクトリと多くのライブラリとツールは接頭辞 'my' を 10 年以上も使って います。しかし、Monty の娘(何年か若い)も My という名前です。そのため、2 つのどちらが MySQL に名前を与えたかは、我々にとっても未だ謎です。

MySQLの主な機能

  • カーネルスレッドを使用する完全なマルチスレッド。これは有効なら簡単に複数 の CPU を使用できることを意味します。
  • C, C++, JAVA, Perl, Python そして TCL API。 「MySQL クライアントツールと API」節参照
  • 様々な種類のプラットフォームで働きます。 「MySQL がサポートする OS は？」節参照 。
  • 多くの項目の型: 1,2,3,4,8 バイト長の符号付き/符号無し整数, FLOAT, DOUBLE, CHAR, VARCHAR, TEXT, BLOB, DATE, DATETIME, YEAR, SET そして ENUM 型。 「項目型」節参照
  • 最適化された one-sweep multi-join を使用するとても速い結合。
  • SELECT と WHERE 部内での全ての関数のサポート。例: select column1 + column2 from table where column1/column2 > 0
  • SQL 関数はとても最適化されたクラスライブラリを通して実装されていて、それ が得ることのできるのと同じくらい速いのです！ 普通、クエリの初期化の後に は、全くメモリ割り当てはありません。
  • SQL GROUP BY と ORDER BY の完全なサポート。グループ関数 (COUNT, AVG, STD, SUM, MAX そして MIN) のサポート。
  • ANSI SQL と ODBC 構文での LEFT OUTER JOIN のサポート。
  • 同一クエリ内に異なるデータベースからのテーブルを混在できます (in 3.22)。
  • とても柔軟で安全な特権とパスワードシステム。ホストベースの証明を許可しま す。
  • ネット上の全てのパスワードトラフィックは暗号化されているためパスワードは 安全です。
  • Windows95 の ODBC Open-DataBase-Connectivity (ソースつき)。全ての ODBC 2.5 関数と他の多く。そのためあなたの MySQL サーバに接続するため に、例えば Access を使用できます。 「ODBC」節参照
  • インデックス圧縮つきのとても速い B-tree ディスクテーブル。
  • 16 インデックス/テーブル。各インデックスは 1 から 15 個の項目/項目部から なります。最大インデックス長は 256 バイトです (しかし、MySQL の コンパイル時に変更できます)。インデックスは CHAR 項目の前に置かれます。
  • 固定長と可変長のレコード。
  • 一時テーブルとして使用されるメモリ内ハッシュテーブル。
  • 大きなデータベースの操作 (我々は、50,000,000 レコードを含むいくつかのデー タベースで MySQL を使用しています)。
  • 全ての項目はデフォルト値を持ちます。任意の項目のサブセットで INSERT を使 用できます。
  • 可搬性のために GNU autoconf を使用します。
  • C と C++ で書かれています。広範な種類のコンパイラでテストされてます。
  • とても速いスレッドベースのメモリ割り当てシステム。
  • メモリリークがありません。市販のメモリリーク検出製品(purify)でテ ストされています。
  • とても速いテーブルチェック、最適化、修復ユーティリティ。 「MySQL テーブルチェック, 最適化そして修復プログラム」節参照
  • 全てのデータは ISO8859_1 形式で保存されています。通常文字列項目の全ての 比較はケース非依存です。
  • 完全な ISO8859_1 (Latin1) サポート。例えば、スカンジナビア語 -A繖� $)Bはテー ブルと項目の名前として許されます。
  • ISO8859_1 Latin1 (当時のスウェーデン語の方法) によるソート。新しいソート 順配列の追加によりこれはソース内で変更可能です。とても先進的なソートの例 を見るには、Czeck ソートコードを見てください。
  • SQL92 標準にあるようなテーブルと構文の別名。
  • DELETE, INSERT, REPLACE, そして UPDATE は影 響を受けた行の数を返します。
  • 関数名はテーブルや項目の名前と衝突しません。例えば ABS は正しい項目名で す。制限は、関数使用時に、関数名と '(' の間に空白が許されないことだけで す。 「MySQL は予約語について厳密か？」節参照
  • 全ての MySQL コマンドはヘルプのために --help または -? を持っています。
  • サーバは現在クライアントに対して多くの言語のエラーメッセージをサポートし ます。 「MySQL がサポートしている言語は？」節参照 。
  • クライアントは、MySQL サーバとの接続時に TCP 接続または Unix ソ ケットを使用します。
  • 次のユーザコマンド。show tables, SHOW INDEX FROM table そ して show columns from table

ストアドプロシジャやトリガ等とは何か？

ストアドプロシジャは、サーバに格納されコンパイルされる SQL コマンドのセッ トです。この後、クライアントはクエリ全体を発行する必要なく、ストアドプロ シジャを参照することができます。これによりさらにいくらかの速度が得られま す。クエリは一回解釈されるだけで、サーバとクライアントの間で送られるデー タが少なくなるためです。サーバに関数のライブラリを持つことで、概念レベル をあげることもできます。

トリガは何かが起こった時に呼び出されるストアドプロシジャです。例えば、取 り引きテーブルのすべての削除をチェックして、取り引きが削除された時に対応 する客を自動的に削除するというようなストアドプロシジャをインストールする ことができます。

MySQL がいつこれらの機能を得るかを知るには 「List of things we want to add to MySQL in the future (The TODO).」節参照 。
